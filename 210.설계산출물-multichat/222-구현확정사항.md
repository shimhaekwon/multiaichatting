function genId(): string {                          // "YYYYMMDD-HHmm-xxxxxx" (214 §1)

## 7. 세션 스냅샷 결선 (`session-store.ts`) [B-2 / loadRoundSnapshot id 갭]

- **스냅샷에 RoundSession 전체를 저장**(214 §5의 부분 필드 → 전체로 확정. 복원이 trivial).
```ts
export async function persistRoundSnapshot(session: RoundSession): Promise<void> {
  await chrome.storage.session.set({ [`lastRound_${session.id}`]: session })
  await chrome.storage.local.set({ lastSessionId: session.id })          // ← loadRoundSnapshot이 찾을 키
}
export async function loadRoundSnapshot(): Promise<RoundSession | null> {
  const { lastSessionId } = await chrome.storage.local.get('lastSessionId')
  if (!lastSessionId) return null
  const got = await chrome.storage.session.get(`lastRound_${lastSessionId}`)
  return (got[`lastRound_${lastSessionId}`] as RoundSession) ?? null
}
export async function clearRoundSnapshot(sessionId: string): Promise<void> {
  await chrome.storage.session.remove(`lastRound_${sessionId}`)
  await chrome.storage.local.remove('lastSessionId')
}
```
- `chrome.storage.session`은 SW 재시작엔 살아있고 브라우저 종료 시 소멸 → best-effort 재개(214 [G5] 그대로). 비밀채팅은 여기에 **저장 금지**.
- **[HIGH-3] 재수화 정규화**: `loadRoundSnapshot()` 결과의 **`status==='WAITING'`(in-flight, 재개 불가)이면 복원 시 `STOPPED`로 강제** — §10 버튼이 "계속하기/종료"로 뜸(WAITING 복원 시 생성도 없는데 STOP만 뜨는 교착 방지). **`ROUND_ACTIVE`는 그대로 복원**(in-flight 없음 → 바로 전송 가능). `ROUND_COMPLETE`/`DONE`/`STOPPED`/`IDLE`도 그대로.
- 스냅샷 **필드 목록**(214 §5 [G5]의 partial 형태)은 본 §7(전체 `RoundSession` 저장)로 **대체**된다. **단 [G5]의 `chatFamily` 메시지 재구성(history→셀 메시지 복원)은 재오픈 시 셀 렌더에 여전히 필요**하므로 그대로 적용(저장 형태만 바뀜, 재구성 단계는 유지).

## 8. content script 본문 확정 (`dom-adapter.ts`) [B-3/B-5/B-6]

> **221 §2 실증 스니펫(chatHubProbe/chatHubRoundtrip)이 검증된 참조 구현**이다. 아래는 그것을 content script로 옮긴 v1 — 그대로 구현. `AdapterSelectors`에 **`loginIndicator?: string`** 필드 추가.

```ts
const $ = (sel?: string) => (sel ? document.querySelector<HTMLElement>(sel) : null)
const $all = (sel?: string) => (sel ? [...document.querySelectorAll<HTMLElement>(sel)] : [])

function isLoginPage(s: AdapterSelectors): boolean {           // [B-6] 휴리스틱
  if (s.loginIndicator) return !!$(s.loginIndicator)           // config에 명시된 로그인 폼 선택자(옵션)
  return !$(s.promptArea)                                      // 미정의 시: 입력창 부재 = 미준비/미로그인으로 간주
}

function fillPrompt(s: AdapterSelectors, prompt: string) {     // 221 §2/§5 검증 경로
  const box = $(s.promptArea); if (!box) return
  box.focus()
  if (s.promptIsContentEditable) {
    box.textContent = prompt
    box.dispatchEvent(new InputEvent('input', { bubbles: true, inputType: 'insertText', data: prompt }))
    if (!box.textContent) document.execCommand('insertText', false, prompt)   // ProseMirror/Lexical/ql-editor 폴백
  } else {
    const setter = Object.getOwnPropertyDescriptor(window.HTMLTextAreaElement.prototype, 'value')!.set!
    setter.call(box, prompt)
    box.dispatchEvent(new Event('input', { bubbles: true }))
  }
}

function pressEnterOrClick(s: AdapterSelectors) {              // sendButton 있으면 click, 없으면 Enter
  const btn = s.sendButton ? $(s.sendButton) : null
  if (btn) btn.click()
  else $(s.promptArea)?.dispatchEvent(new KeyboardEvent('keydown',
    { key: 'Enter', code: 'Enter', keyCode: 13, which: 13, bubbles: true }))
}

// stopButton 조회: perplexity는 :has(use[*|href]) 가 querySelector에서 무효 → closest() 경로 사용 [B-4]
function findStop(s: AdapterSelectors): HTMLElement | null {
  if (!s.stopButton) return null
  if (s.stopButton.includes('*|href'))                        // perplexity 전용 분기
    return document.querySelector<HTMLElement>("use[*|href$='pplx-icon-player-stop-filled']")?.closest('button') ?? null
  return $(s.stopButton)
}

function observeResponse(s: AdapterSelectors, onUpdate: (text: string, done: boolean) => void) {
  let lastText = '', lastChange = Date.now()
  const debounce = s.doneDebounceMs ?? 1500
  const timer = setInterval(() => {
    const els = $all(s.responseArea)
    const cur = els.length ? (els[els.length - 1].innerText || '') : ''   // 마지막 요소 = 최신 응답
    if (cur !== lastText) { lastText = cur; lastChange = Date.now(); onUpdate(cur, false) }
    const flag = s.generatingFlag ? $(s.generatingFlag) : null
    const stop = findStop(s)
    let done = false
    if (s.generatingFlag) done = !flag && !!lastText                       // 1순위(현재 6종 미사용)
    else if (s.stopButton) done = !stop && !!lastText                       // 2순위: stop 소멸
    else done = !!lastText && Date.now() - lastChange > debounce            // 3순위: 텍스트 안정화(deepseek)
    if (done) { clearInterval(timer); onUpdate(lastText, true) }
  }, 300)
  return () => clearInterval(timer)        // STOP/abort 시 호출
}
```
- **STOP 처리**: `STOP` 메시지 수신 시 `findStop(selectors)?.click()` + observe timer 중단 + 즉시 `DONE` 통보. 자동 타임아웃 없음(FR-11).
- done 판정 우선순위/`doneDebounceMs`(기본 1500)는 217 §3.3과 동일.

## 9. background 릴레이 `ensureTab` 확정 (217 §3.3 [G1]) [A-7]

```ts
async function ensureTab(llm: string): Promise<number | null> {
  const url = LLM_URL[llm]
  const found = await chrome.tabs.query({ url: url + '/*' })
  if (found[0]?.id != null) return found[0].id
  const tab = await chrome.tabs.create({ url, active: false })
  return await new Promise<number | null>((resolve) => {
    const to = setTimeout(() => { chrome.tabs.onUpdated.removeListener(li); resolve(tab.id ?? null) }, 15000)  // 타임아웃 후에도 tabId 반환(미준비는 CS가 ERR_002)
    const li = (id: number, info: chrome.tabs.TabChangeInfo) => {
      if (id === tab.id && info.status === 'complete') { clearTimeout(to); chrome.tabs.onUpdated.removeListener(li); resolve(tab.id ?? null) }
    }
    chrome.tabs.onUpdated.addListener(li)
  })
}
```
- chatgpt 탭 재사용: `LLM_URL.chatgpt='https://chatgpt.com'` 쿼리에 `chat.openai.com` 탭은 안 잡히나, host_permissions에 둘 다 있어 신규 chatgpt.com 탭을 열면 동작(허용). 별칭 통합은 follow-up.

function RoundActionButton({ engine, onOpenSave }: { engine: RoundEngine; onOpenSave: () => void }) {
  const { t } = useTranslation()
  const st = engine.session?.status ?? 'IDLE'
  switch (st) {
    case 'WAITING':  return <Button type="button" color="flat"    text={t('stop')}   onClick={engine.stop} />          // 빨강
    case 'STOPPED':  return <><Button type="button" text={t('resume')} onClick={engine.resume}/><Button type="button" color="flat" text={t('end')} onClick={engine.endSession}/></>
    case 'DONE':     return <Button type="button" color="primary" text={t('save')}   onClick={onOpenSave} />
    case 'ROUND_COMPLETE': return <><Button type="submit" color="primary" text={t('nextRound')}/><Button type="button" text={t('endSave')} onClick={engine.endSession}/></>
    default:         return <Button type="submit" color="primary" text={t('send')} />   // IDLE / ROUND_ACTIVE
  }
}

## 12. MD 산출 결정성 (214 §4) [E1-F14/E3-A5/A6]

- **per-LLM `<botId>.md`의 "+ 타 LLM 응답" 출처** = `buildContextForLlm(botId, prevRound, round, '', participants).others`(216) — **prompt에 주입한 것과 동일 소스**(완료분만·자기 제외). 표시 라벨은 **가독성 위해 `CHATBOTS[k].name`** 사용(키는 botId지만 MD는 사람이 읽음). **1라운드 per-LLM MD에는 "+타 LLM" 블록 없음**(빈 값은 MD에서 생략).
- **summary.md / SaveSessionDialog 봇 집합 = `session.slots`** (정본). 순서 = `session.slots` 순. swap(N-3)으로 동기 유지되므로 history 합집합 불요. **(판단: swap으로 교체돼 나간 봇의 이전 라운드 응답은 파일에서 제외 — swap=best-effort, 단순성 우선. 다른 동작 원하면 알려주세요.)**
- `_(응답 없음)_` 표기: 헤더(`#### {name}`) **다음 줄에** 본문 대신 `_(응답 없음)_`. (status!=='completed' 봇.)
- `download()`는 objectURL 누수 방지 위해 `URL.revokeObjectURL(url)`을 다운로드 후 호출(214 §4 [G6] 보강).

## 14. 비밀채팅 확정 (218 §6 / 217 §6) [E4-A1/A2]

- **mount = `components/Dialog`**(재사용). fixed overlay 포크 종결.
- **토큰 = 누적**(`SecretChat.tokenCount` = 메시지별 `approxTokens` 합). 입력 차단 = `tokenCount + approxTokens(draft) > 1000`이면 전송 비활성 + "(N/1000)" 표시.
- **스트림 소비(useChat 안 씀, 전용 인스턴스)**:
```ts
const botRef = useRef<AbstractBot | null>(null)   // 오버레이 오픈 시 createBotInstance(targetLlm) 1회
async function sendSecret(text: string) {
  const ac = new AbortController()
  setSecret(d => { d.messages.push({role:'user',content:text}, {role:'assistant',content:''}); d.tokenCount += approxTokens(text) })
  const full = text + tokenLimitInstruction(1000)              // 216
  try {
    for await (const ans of botRef.current!.sendMessage({ prompt: full, signal: ac.signal })) {
      setSecret(d => { d.messages[d.messages.length-1].content = ans.text })   // bot.sendMessage = AsyncIterable<{text}>
    }
  } finally {
    setSecret(d => { d.tokenCount += approxTokens(d.messages[d.messages.length-1].content) })
  }
}
```
- 닫기/세션종료 → `setSecretChat(null)`. **저장 절대 금지**(FR-09). §7 스냅샷에도 미포함.
- ⚠️ §18 한계: 같은 LLM 탭/웹 대화 공유(217 §6 명시) — 메인 세션 진행 중 같은 LLM 비밀채팅은 탭 경합 가능 → UI에서 해당 LLM이 WAITING이면 비밀채팅 전송 비활성 권장.
- **마운트**: `MultiBotChatPanel`이 `const secret = useAtomValue(secretChatAtom)`로 열림 감지 → 패널 하단에 `{secret && <SecretChatOverlay />}` 조건 렌더. `SecretChatOverlay`는 `secretChatAtom`을 자체 구독(별도 prop 불요), 헤더 닫기→`useSetAtom(secretChatAtom)(null)`. 🔒 클릭(ConversationPanel 헤더, §16/[G3])이 `secretChatAtom`을 §14 페이로드로 set → 오버레이 오픈.

## 15. 사이드바 가드 + toast (218 §3) [E4-A3]

```ts
function toggleLlm(botId: BotId) {     // Sidebar 체크박스 onChange
  setSelectedLLMs(prev => {
    if (prev.includes(botId)) {
      if (prev.length <= 1) { toast(t('llmMinOne')); return prev }     // 최소 1
      return prev.filter(b => b !== botId)
    }
    if (prev.length >= 6) { toast(t('llmMaxSix')); return prev }       // 최대 6 (중복은 includes로 차단)
    return [...prev, botId]
  })
}
```
- `toast`는 `react-hot-toast`(기존 의존성). 세션 시작(`status!=='IDLE'`) 후엔 사이드바 토글 자체를 disabled(잠금, 218 §0.2).

## 20. 검증-안전 (변경 불요, 구현자 안심용)
- `completedNaturally`: `stop()`이 STOPPED를 **동기 set** 후 abort → 슬롯 resolve 시점엔 이미 STOPPED → 중단 슬롯이 completed로 오분류되지 않음(215 §4).
- `padTo6` 더미: `slots`에서 제외되어 렌더·전송 안 함. 더미가 slot0과 `chatFamily` 공유해도 무해(읽기 동일 atom).
- swap 중 in-flight `slots`: swap은 WAITING 차단, ROUND_COMPLETE swap 시 atom 변경→리렌더로 다음 클릭 전 최신 `slots` 반영.
- `approxTokens`: `gpt3-tokenizer` 생성/인코딩 실패는 try/catch로 글자수 근사 폴백(216) — 표시용이라 무해.
