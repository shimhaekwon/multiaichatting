## 2. 메인 패널 개조 (`MultiBotChatPanel.tsx`)

기존 `GeneralChatPanel`(`MultiBotChatPanel.tsx:35`)이 이미 그리드+`sendAllMessage`+`onSwitchBot`를 갖춤. 이를 **라운드 엔진 연동**으로 개조:

```tsx
const engine = useRoundEngine()
const maxRounds = useAtomValue(maxRoundsAtom)               // 214 §2 (SettingsBar와 공유)
const tokenLimit = useAtomValue(tokenLimitAtom)
const [showSave, setShowSave] = useState(false)            // [222 §10] SaveSessionDialog 오픈 소유(MultiBotChatPanel)
useEffect(() => { if (engine.session?.status === 'DONE') setShowSave(true) }, [engine.session?.status])  // DONE 진입 시 자동 오픈(목업 0.4)

// [C-2] 첫 전송 처리 — start()는 void 반환 + jotai set은 store에 "동기" 반영.
//   기존 `engine.start(...) && engine.sendPrompt(text)`는 (a) void && → 단락평가로 sendPrompt 미호출,
//   (b) 같은 tick의 closure 세션이 stale 인 두 버그가 있었다.
//   sendPrompt가 store.get(roundSessionAtom)로 세션을 읽으므로(215 §4), start 직후 호출해도 안전하다.
const handleSubmit = useCallback((text: string) => {
  const s = engine.session
  if (!s || s.status === 'IDLE') engine.start(maxRounds, tokenLimit)   // store에 세션 동기 기록
  engine.sendPrompt(text)              // [H-1] ROUND_ACTIVE(R1)·ROUND_COMPLETE(R2+) 모두 진행 — sendPrompt 가드가 둘 다 허용(215 §4). R2+는 새 입력 텍스트, 교차맥락은 자동 주입(216)
}, [engine, maxRounds, tokenLimit])

// 그리드: 활성 슬롯 수에 따른 열 규칙 (레이아웃 규칙)
function gridCols(n: number): string {   // [222 §11] 동적 `grid-cols-${n}` 금지(Tailwind JIT가 안 만듦) — 리터럴만
  switch (n) { case 1: return 'grid-cols-1'; case 2: return 'grid-cols-2'; case 3: return 'grid-cols-3'; case 4: return 'grid-cols-2'; default: return 'grid-cols-3' }  // 1 / 2 / 3 / 2×2 / 2×3
}

return (
  <div className="flex flex-col h-full">
    <SettingsBar engine={engine} />                         {/* 라운드/토큰 select, 세션종료, 진행바 */}
    <div className={cx('grid grow auto-rows-fr gap-2', gridCols(engine.slots.length))}>
      {engine.slots.map((s, i) => (
        <ConversationPanel key={`${s.botId}-${i}`} botId={s.botId} bot={s.chat.bot}
          messages={s.chat.messages} generating={s.chat.generating}
          stopGenerating={s.chat.stopGenerating} resetConversation={s.chat.resetConversation}
          mode="compact" roundMode onSwitchBot={(b) => engine.swap(i, b)} />
          {/* [G3] roundMode=표시전용(자체 입력창·WebAccess 미렌더)+헤더 🔒. FR-14★(VER-002 확장) swap=onSwitchBot. onUserSendMessage 불필요 */}
      ))}
    </div>
    <ChatMessageInput mode="full" disabled={engine.generating || engine.session?.status==='WAITING'}
      onSubmit={handleSubmit}                                 {/* [C-2] start→sendPrompt 안전 처리 */}
      actionButton={<RoundActionButton engine={engine} onOpenSave={() => setShowSave(true)} />} />   {/* [222 §10] onOpenSave 필수 */}
    {showSave && <SaveSessionDialog session={engine.session!} onClose={() => setShowSave(false)} />}
  </div>
)
```
> **[G3] `ConversationPanel` 개조 필요 (그대로 재사용 불가)**. 실제 props = `{ botId, bot, messages, onUserSendMessage(필수), resetConversation, generating, stopGenerating, mode?, onSwitchBot? }`. 셀마다 **자체 `ChatMessageInput`("Send to X")과 `WebAccessCheckbox`를 항상 렌더**하며, 공유/지우기/기록은 **헤더 아이콘**에 이미 존재(🔒 없음). 라운드 모델은 "셀=표시전용 + 하단 단일 입력"이므로 다음을 추가:
> - 신규 prop **`roundMode?: boolean`** — true면 하단 자체 `ChatMessageInput` 렌더 **생략**. 이때 props의 **`onUserSendMessage`를 optional(`onUserSendMessage?`)로 변경**해야 strict TS 통과(roundMode에선 미전달) [M-2].
> - **`WebAccessCheckbox` 렌더 제거**(웹접근 미사용, G11).
> - 헤더 아이콘에 **🔒(비밀채팅)** 추가 → `secretChatAtom` 오픈.
> - `trackEvent`(plausible) 호출 **제거**(G10).
> - 공유=`ShareDialog` / 지우기=`resetConversation` / 기록=`HistoryDialog` / swap=`onSwitchBot`(compact)은 **기존 그대로**.

## 5. STOP / 진행 / 저장 액션 (`RoundActionButton`)

| status | 하단 버튼 |
|--------|-----------|
| IDLE/ROUND_ACTIVE | "전송"(`sendPrompt`) |
| WAITING | "STOP"(`engine.stop`) — 빨강 #FF4444 (FR-11) |
| ROUND_COMPLETE | "다음 라운드"(현재 입력창 텍스트로 `nextRound(text)`, handleSubmit과 동일 경로) + "종료/저장" |
| STOPPED | "계속하기"(`resume`) + "종료"(`endSession`) |
| DONE | "저장"(SaveSessionDialog 오픈) |

진행바: `●●●○○` 식 라운드 표시(색상: 진행 #4488FF, 완료 #44BB44, 대기 #ccc).

> **[H-1] 다음 라운드 결선**: `ROUND_COMPLETE`에서 입력창은 활성(`disabled`는 WAITING/generating만, §2)이라 라운드2+는 **새 사용자 입력**을 받는다. "다음 라운드" 버튼·Enter 모두 현재 입력 텍스트를 `engine.sendPrompt`(=`nextRound`)로 전달 → 가드가 `ROUND_COMPLETE`를 허용(215 §4)해 `roundNo=currentRound+1` 진행. 교차맥락(타 LLM 직전 응답 YAML)은 `buildContextForLlm`가 자동 주입(216). → "전송"과 "다음 라운드"는 **라벨만 다르고 동일 경로**.
