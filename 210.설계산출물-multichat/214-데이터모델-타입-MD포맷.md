## 1. 핵심 TS 타입 (`src/types/round.ts` 신규)

```ts
import { BotId } from '~app/bots'

export type RoundStatus =
  | 'IDLE' | 'ROUND_ACTIVE' | 'WAITING' | 'ROUND_COMPLETE' | 'STOPPED' | 'DONE'

export type ResponseStatus = 'waiting' | 'completed' | 'stopped' | 'error'

export interface RoundResponse {
  content: string
  tokens: number
  status: ResponseStatus
  timestamp: number | null
}

export interface RoundRecord {
  round: number                       // 1-based
  userPrompt: string
  responses: Partial<Record<BotId, RoundResponse>>
}

export interface Slot {
  slot: number                        // 0..5 (2×3 그리드 위치)
  botId: BotId
}

export interface RoundSession {
  id: string                          // "YYYYMMDD-HHmm-xxxxxx"
  createdAt: number
  status: RoundStatus
  currentRound: number                // 1-based, IDLE이면 0
  maxRounds: number                   // {3,5,7,9} (기본 5) [ver.002]
  tokenLimit: number                  // 100|500|1000|3000|5000|10000 (기본 1000) [ver.002]
  slots: Slot[]                       // 활성 LLM (최대 6)
  history: RoundRecord[]              // 완료된 라운드 누적
}

export interface SecretChat {
  targetLlm: BotId
  tokenLimit: number                  // 1000 고정
  messages: { role: 'user' | 'assistant'; content: string }[]
  tokenCount: number
}
```

> [ver.002] `TOKEN_LIMIT_OPTIONS = [100,500,1000,3000,5000,10000] as const`, `MAX_ROUNDS_OPTIONS = [3,5,7,9] as const`, `MAX_CONCURRENT_LLMS = 6` 는 `consts.ts`에 추가.

## 2. Jotai atom (`src/app/state/round.ts` 신규)

```ts
import { atom } from 'jotai'
import { atomWithImmer } from 'jotai-immer'
import { atomWithStorage } from 'jotai/utils'
import { BotId } from '~app/bots'
import { RoundSession, SecretChat } from '~types/round'

// 선택된 LLM 풀 + 그리드 슬롯 (지속)
export const selectedLLMsAtom = atomWithStorage<BotId[]>('selectedLLMs', ['chatgpt', 'claude', 'gemini'])

// 라운드/토큰 설정 (지속) — SettingsBar(218 §4)와 입력 핸들러(218 §2)·엔진 start(215 §4)가 공유 [C-2]
export const maxRoundsAtom = atomWithStorage<number>('maxRounds', 5)      // {3,5,7,9} [ver.002]
export const tokenLimitAtom = atomWithStorage<number>('tokenLimit', 1000) // {100,500,1000,3000,5000,10000} [ver.002]

// 진행 중 라운드 세션 (가변 드래프트)
export const roundSessionAtom = atomWithImmer<RoundSession | null>(null)

// 비밀채팅 (휘발성 — atomWithStorage 금지!)
export const secretChatAtom = atom<SecretChat | null>(null)
```

> **[C-2] 설정 atom 공유 이유**: `start()`/입력 핸들러가 같은 `maxRoundsAtom`/`tokenLimitAtom`을 읽어야 "첫 전송 시 start→sendPrompt"가 일관된 값으로 동작한다(218 §2). 세션 시작 후에는 SettingsBar에서 잠금(218 §4).

> **비밀채팅은 절대 `atomWithStorage`/`chrome.storage` 사용 금지** — 메모리 전용(FR-09 휘발성). 세션 종료/패널 닫힘 시 소멸.

```ts
import { CHATBOTS } from '~app/consts'
import { BotId } from '~app/bots'

// LLM별 파일: "<botId>.md"
export function buildLlmMarkdown(session: RoundSession, botId: BotId): string
// summary.md
export function buildSummaryMarkdown(session: RoundSession): string
// 저장 [G6]: chrome.downloads로 파일별 다운로드 (다중 파일 안정적, 'downloads' 권한 필요)
export async function saveSessionMarkdown(session: RoundSession, fileBotIds: BotId[], includeSummary: boolean): Promise<void>
```

**LLM별 `chatgpt.md` 템플릿** (FR-12 — Request/Response/Request(+타LLM) 흐름):

```markdown
# {CHATBOTS[botId].name} — 브레인스토밍 기록
> 세션: {createdAt} | 라운드: {history.length}/{maxRounds}

## Round 1 — Request
{userPrompt}

## Round 1 — Response
{responses[botId].content}

---

## Round 2 — Request (+ 타 LLM 응답)
{userPrompt}

[이전 라운드 다른 AI 의견]
- {otherName}: {otherContent}
...

## Round 2 — Response
{responses[botId].content}
```

> **LLM별 md vs summary.md 차이**: **LLM별 `<botId>.md`** = 해당 봇 **단일 관점**의 흐름(자기 Request/Response + 다음 라운드 Request에 타 LLM 의견 주입). **`summary.md`** = **전체 LLM 합본** — 라운드 순서대로 Request + 참여 LLM **전원**의 응답을 한 문서에 모음.

**`summary.md` 템플릿** (FR-13 — 라운드순 전체 합본):

```markdown
# 브레인스토밍 결과 정리
> 세션: {createdAt} | 참여 LLM: {slots.length}종 | 라운드: {history.length}/{maxRounds}

## 주제 (Round 1 Request)
{history[0].userPrompt}

## Round 1
### Request
{history[0].userPrompt}

### Responses
#### ChatGPT
{responses['chatgpt'].content}

#### Claude
{responses['claude'].content}

#### Gemini
{responses['gemini'].content}
<!-- status !== 'completed' 인 봇은 본문 대신: #### Grok 줄 아래 _(응답 없음)_ -->

## Round 2
### Request
{history[1].userPrompt}

### Responses
#### ChatGPT
{responses['chatgpt'].content}
...
```

> **[G7] summary 생성 (결정적 · LLM 호출 없음 · 라운드순 전체 합본)** [ver.002 사용자 확정 2026-05-29]:
> - 문서 최상단 `## 주제 (Round 1 Request)` = `history[0].userPrompt` 1줄(제목 표기용).
> - `session.history`를 **라운드 순서대로** 순회하며, 각 라운드마다 **Request(`userPrompt`)** + 참여 LLM **전원의 응답 원문 전체**(`responses[botId].content`)를 등록. **요약/첫문장 추출/절삭 없음**.
> - 봇 출력 순서 = `session.slots` 순. 헤더는 `CHATBOTS[botId].name`.
> - 응답이 `status !== 'completed'`(중단/오류/미응답)인 봇은 `_(응답 없음)_` 로 표기(자리 유지).
> - 별도 "LLM별 핵심 의견"·"최종 종합" 절 및 **LLM 추가 호출 없음**(이전 첫문장 추출/종합 알고리즘 폐기).
>

## 5. `chrome.storage.session` 스냅샷 (SW 복구, `src/services/session-store.ts` 신규)

```ts
// 매 ROUND_COMPLETE 시 저장
export async function persistRoundSnapshot(session: RoundSession): Promise<void> {
  await chrome.storage.session.set({ [`lastRound_${session.id}`]: {
    sessionId: session.id, completedRound: session.currentRound,
    slots: session.slots, history: session.history,
    maxRounds: session.maxRounds, tokenLimit: session.tokenLimit,
  }})
}
export async function loadRoundSnapshot(): Promise<RoundSession | null>   // 재오픈 시 재개 [G5]
export async function clearRoundSnapshot(sessionId: string): Promise<void>
```

> **[G5] 재수화(재오픈 시)**: 앱(app.html) mount 시 `loadRoundSnapshot()` 호출 → 스냅샷 있으면 (1) `roundSessionAtom`에 복원, (2) 각 슬롯의 `chatFamily({botId, page:'singleton'})` 메시지를 `history`로부터 **재구성** — 라운드별로 `{id, text: userPrompt, author:'user'}` 와 `{id, text: responses[botId].content, author: botId}` 2개씩 push(`use-chat`의 메시지 구조와 동일). 이후 `status`에 따라 UI 분기(ROUND_COMPLETE→"다음 라운드" 노출 등). `chatFamily`는 메모리라 재오픈 시 비어 있으므로 이 재구성이 필수.
>
> `chrome.storage.session`은 `manifest`에서 `setAccessLevel`이 이미 ChatHub background에 설정됨(`TRUSTED_AND_UNTRUSTED_CONTEXTS`). 비밀채팅은 여기에도 저장 금지.
