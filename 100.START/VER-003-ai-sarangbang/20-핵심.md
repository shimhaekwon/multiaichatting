# ai-sarangbang — 핵심 요구사항 (전문가용)

> 정본: `220.설계산출물-ai-sarangbang/221`(제품·규칙), `222`(coordinator 상태머신). 충돌 시 규칙은 221 §3 우선.

## 1. 시스템 구성

### 1.1 기술 스택
- **Type**: 로컬 웹앱 (브라우저, 크롬 확장 아님)
- **Backend**: AgentDriver 추상화 — `speak(ctx, signal): AsyncIterable<token>` 단일 seam
  - P0 = MockDriver(스크립트/지연) · P1 = OllamaDriver(localhost:11434) · P2 = ApiDriver
- **Coordinator**: 프레임워크 무관 순수 TS 모듈 (UI/드라이버 분리, 멀티유저 이식 대비)
- **Storage**: 공개 대화 MD 파일 (요청 시). 귓속말 미저장

### 1.2 아키텍처 계층
```
[UI 레이어 — 90s 채팅 렌더·타이핑 효과]
        |
[Coordinator — floor 루프 (단일 권위, race-free)]
        |
[AgentDriver — Mock / Ollama / API]
```

### 1.3 단계 (범위 · 제약)

| 축 | P0 (지금) | P1 | P2 (최후) |
|----|-----------|----|-----------|
| AI 백엔드 | Mock 드라이버 | 로컬 Ollama(무료) | API(여유 시) |
| 참가자 | 사람 1 + AI N(목) | + 실 AI | 동일 |
| 사람 | 솔로(나 혼자) | 동일 | (멀티유저 = follow-up) |
| 형태 | 로컬 웹앱 | 동일 | 동일 |
| 비용 | 0 | 0 | 토큰 과금 |

- [제약] **무료 최우선** — 유료는 메커닉 무결 후 최후. P0/P1 외부 유료 의존 0
- [범위 밖 → follow-up] 멀티유저 네트워크, 모바일, 영구 계정, 음성

## 2. 기능 요구사항

| ID | 분류 | 요구사항 | 단계 |
|----|------|----------|------|
| FR-01 | R1 턴제 | 한 턴 동안 각 참가자(사람·AI)는 **최대 1회** 공개 발언. 귓속말은 미포함 | P0 |
| FR-02 | R2 floor 직렬화 | 동시 출력(floor)은 **한 명만**. 발언 의사는 도착 순 FIFO 큐. floor 보유자가 스트림 끝까지 출력 후 반납 → 다음 큐 | P0 |
| FR-03 | R2 바지인 의미 | AI 간 바지인 = 다음 차례 **선점/대기**이지, 진행 중 출력 중간 끊기 아님 | P0 |
| FR-04 | R3 큐 소진 | floor 양도를 발언 큐가 빌 때까지 반복. 더 발언할 참가자 없으면 **턴 종료** → 사람 입력 대기 | P0 |
| FR-05 | R4 귓속말 | 특정 참가자와 1:1 사적 채널. floor·턴·공개 로그 거치지 않음. **휘발성**(저장·MD·스냅샷 미기록) | P0 |
| FR-06 | 턴 흐름 | 사람 입력이 새 턴을 연다([제안] D1) = 사람의 1회 공개 발언 + 턴 시작 | P0 |
| FR-07 | 참가자 | 사람 1(나) + AI N. P0 = 전원 1회 발언, opt-out 훅 자리만([제안] D2) | P0 |
| FR-08 | 사람 인터럽트 | 사람이 AI 출력 중 전송 → 현재 floor 보유자 abort(부분 보존·`stopped`) 후 새 턴([제안] D3) | P0 |
| FR-09 | 큐 우선순위 | 의사 도착 FIFO, 동시면 좌석순. 사람은 D3대로 우선([제안] D4) | P0 |
| FR-10 | 발언 컨텍스트 | 발언 시점까지의 공개 대화(**자기 포함**, done만 — stateless 드라이버 기억). 화자 `[이름]` 라벨로 자타 구분([제안] D5, [C-2] 정정) | P0 |
| FR-11 | done 감지 | 드라이버 **stream-end = done**(추상화 계약). 안전망 = 하드 상한 무응답 타임아웃 → `stopped`(드라이버 래퍼). [H-3] DOM용 debounce는 제외 | P0 |
| FR-12 | abort 가드 | abort 시 상태를 동기적으로 `stopped` 확정 → 중단 발언이 `done`으로 오분류 안 됨(completedNaturally 가드, v1 계승) | P0 |
| FR-13 | 90s UI | monospace 폰트, 글자별 타이핑 효과, `[이름]` 대괄호 발화, 입·퇴장 메시지, 미니멀 | P0 |
| FR-14 | MD 로그 | 공개 대화 MD 저장(요청 시). **귓속말 제외(휘발)** | P0 |
| FR-15 | 백엔드 교체 | AgentDriver로 mock→Ollama→API 무중단 교체. abort는 `signal`로 드라이버에 전파 | P0/P1/P2 |
| FR-16 | AI 발언 판정 | P1+ = LLM이 "말할지" 판정 또는 설정 (R3 일부 미발언 허용) | P1 |

## 3. Coordinator 상태 모델

Room: `idle`(사람 입력 대기) / `turn_active`(큐 처리 중, floorHolder = 1인 또는 전이 순간 null).

Participant(turn-state): `idle` / `queued`(의사 제출, floor 대기) / `speaking`(floor 보유·스트리밍) / `done`(자연 종료) / `stopped`(인터럽트·STOP, 부분 보존).

- [중요] 공유 boolean 플래그 제거 — floor를 **단일 소비자 루프**가 소유. 참가자는 `enqueue`만, 출력 권한은 루프만 부여 → 경쟁 자체가 없음(TOCTOU 레이스 회피).

## 4. 엣지케이스
| 케이스 | 처리 |
|--------|------|
| 빈 큐(아무도 발언 의사 없음) | drainFloor 즉시 종료 → `idle` |
| 전원 opt-out(D2) | 위와 동일, 사람 발언만 남음 |
| 스트림 중 abort(D3) | 부분 보존 · `stopped` · completedNaturally 가드 |
| whisper 도중 턴 시작 | 독립 — whisper 유지, 턴 floor와 무관 |
| 드라이버 에러 | 해당 발언 `stopped` + 에러, 큐 계속 |
| 같은 참가자 중복 intent | spokeThisTurn + 큐 중복 차단으로 1회 제한 |
