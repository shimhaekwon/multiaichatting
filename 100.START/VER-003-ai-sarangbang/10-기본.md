# ai-sarangbang — 기본 개요

## 한 줄 요약
로컬 웹앱. AI 여러 명과 사람(나)이 함께 들어가는 라이브 턴제 채팅방. 90년대 PC통신 감성의 "사랑방". v1(multichat)의 라운드제 병렬 → 라이브 턴제로 대체.

## 핵심 동작
- **라이브 턴제** — 한 명씩 순서대로 발언 + 끼어들기 + 귓속말
- **핵심 규칙 4개**
  - R1: 한 턴에 누구나 1회만 발언 (귓속말 제외)
  - R2: 먼저 잡은 화자가 끝까지 출력, 나머지는 대기 (바지인 = 다음 차례 선점, 중간 끊기 아님)
  - R3: 발언할 화자가 혼자/없을 때까지 R2 반복 → 턴 종료
  - R4: 귓속말(whisper) — 특정 참가자와 1:1 사적 채널, 휘발성(저장 안 함)
- **턴 흐름** — 사람 입력이 새 턴 시작([제안] D1) → AI 발언 의사 큐 → floor 한 명씩 양도 → 큐 소진 시 종료
- **참가자** — 사람 1(나) + AI N. P0는 전원 1회 발언([제안] D2)
- **사람 인터럽트** — 사람이 AI 출력 중 전송 시 사람 우선, 현재 출력 abort·부분 보존([제안] D3)
- **90s 감성 UI** — monospace 폰트, 글자별 타이핑 효과, `[이름]` 대괄호 발화, 입·퇴장 메시지
- **AI 백엔드 단계** — Mock 드라이버 → 로컬 Ollama(무료) → API(최후). AgentDriver 인터페이스로 무중단 교체

## 저장/산출물
- 공개 대화 MD 로그 (요청 시) — **귓속말은 제외(휘발성)**
- 휘발: 귓속말은 history·로그·MD·스냅샷 미기록 (v1 비밀채팅 불변식 계승)

## 기술 제약
- **무료 최우선** — 유료(API)는 메커닉 완전 무결 후 최후. 외부 유료 의존 0 (P0/P1)
- **솔로(나 혼자)** — 인증·서버·네트워크 동기화 불요
- **로컬 웹앱** (브라우저, 크롬 확장 아님)
- coordinator는 프레임워크 무관 순수 모듈로 분리 (멀티유저 이식 대비)
- 범위 밖(follow-up): 멀티유저 네트워크, 모바일, 영구 계정, 음성

## 미해결
턴 시작 트리거 / AI 발언 여부 / 사람 하드 인터럽트 / 큐 우선순위 / 발언 컨텍스트 (D1~D5, 제안값으로 P0 진행) · 90s 감성 구체 스펙 · P1 Ollama 모델·하드웨어
