# 다자 AI 브레인스토밍 — 핵심 요구사항 (전문가용)

## 1. 시스템 구성

### 1.1 기술 스택
- **Type**: Chrome Extension (Manifest V3)
- **Auth**: Browser session cookie 기반, API key 미사용
- **Storage**: 파일 시스템 (MD 저장), IndexedDB 불필요
- **Base**: ChatHub 방식 참조

### 1.2 아키텍처 계층
```
[Popup / Side Panel UI]
        |
[Session Manager] [Round Engine] [Secret Chat]
        |               |             |
[LLM Adapter Layer — 10종]
        |
[Content Script]
```

### 1.3 LLM Adapter Layer
- 대상 10종: chatgpt, gemini, claude, deepseek, mistral, grok, kimi, perplexity, pi, qwen
- 각 LLM content script로 session 탐지 + DOM 조작
- 미로그인 시 로그인 페이지 안내
- Top 8까지만 노출
- 공통 인터페이스: `checkSession()`, `sendMessage()`, `extractResponse()`

## 2. LLM 선택

| 항목 | 사양 |
|---|---|
| 최소 선택 | 2개 (미만 시 전송 버튼 비활성화) |
| 최대 노출 | Top 8 |
| 상태 유지 | chrome.storage.local |
| 라운드 중 변경 | 차단, 라운드 경계에서만 허용 |

## 3. 라운드 엔진

### 3.1 라운드 흐름
```
R1: 사용자 prompt → 각 LLM 독립 응답
R2: 사용자 prompt + R1 응답 전체 → 각 LLM 응답
R3~R9: 동일 패턴 반복
```

### 3.2 응답 전달 구조 (FR-03)
```
{
  "roundNo": 2,
  "maxRounds": 5,
  "userPrompt": "...",
  "otherResponses": {
    "chatgpt": "응답1",
    "claude": "응답2"
  },
  "tokenLimit": 1000
}
```
포맷: json/md/yaml/텍스트 중 선택.

### 3.3 동기화
- 응답 순서 제한 없음
- 모든 LLM 응답 완료 시 라운드 종료
- 자동 타임아웃 없음, STOP 버튼으로만 중단
- STOP 시: 현재 대기 중인 응답 폐기

## 4. Request Payload
```
system: "응답은 1000토큰 이하. 브레인스토밍 형식으로 자유롭게 의견 제시"
user: {원본 prompt}
meta: {roundNo, maxRounds, otherResponses}
```

## 5. 사용자 개입

| 시점 | 가능 |
|---|---|
| 응답 대기 중 | STOP 버튼 |
| 라운드 완료 후 | 입력, LLM 변경, 비밀채팅 시작 |
| 비밀채팅 중 | 1:1 대화 (전체 맥락과 분리) |

## 6. 저장 / 산출물

### 6.1 산출물
- `{llm}.md`: Request / Response / Request(+타응답) 흐름 기록
- `summary.md`: 브레인스토밍 결과 종합

### 6.2 저장 정책
- 자동 저장 없음, 사용자 요청 시 MD 생성
- 비밀채팅: 저장 안 함 (휘발성)

## 7. 비밀채팅 (FR-09)

| 항목 | 사양 |
|---|---|
| 대상 | 특정 LLM 1:1 |
| 컨텍스트 | 전체 맥락과 분리 |
| 토큰 | 1000 제한 |
| 저장 | 휘발성 |
| 실행 시점 | 라운드 사이 |

## 8. STOP (FR-11)
- 수동 버튼 클릭
- 응답 대기 중단 및 폐기
- STOP 후 세션 종료

## 9. 기술 제약
Chrome 최신 / Manifest V3 / Windows·macOS·Linux / 단일 사용자 / 로컬 전용 / API 키 불필요

## 10. 미해결 설계 항목
1. STOP 후 재시작 가능 여부
2. 비밀채팅 시작 조건 구체화
3. 토큰 카운트 근사 방식 (글자수 vs 토크나이저)
4. 10종 LLM DOM selector 관리 전략
5. 10종 LLM 중 일부 미로그인 시 처리 정책
