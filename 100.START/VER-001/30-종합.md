# 다자 AI 브레인스토밍 — 종합 가이드

> 개발 비전문가(기획자, PM, 테스터)를 위한 설명서

---

## 이게 뭐 하는 건가요?

10명의 AI 전문가를 불러 모아 브레인스토밍하는 크롬 확장 프로그램입니다.

```
비유: 회의실에 AI 10명을 앉혔습니다.

당신: "시장 조사 아이디어를 내줘"

  [1라운드 — 각자 생각]
  ChatGPT: "온라인 설문조사가 좋겠습니다"
  Claude: "경쟁사 분석부터 시작하는 게..."
  Gemini: "두 의견을 합쳐서..."
  (나머지 7개 AI도 각자 의견)

  [2라운드 — 서로 의견 공유]
  ChatGPT: "Claude의 경쟁사 분석에 동의합니다. 추가로..."
  Claude: "Gemini의 통합 접근법이 현실적입니다"
  (각자 다른 AI 의견을 보고 추가 생각)

  [3라운드 — 정리]
  ChatGPT: "최종 제안은..."
  Claude: "종합하면..."
```

---

## 어떻게 접속하나요?

**API 키가 필요 없습니다.** 평소처럼 크롬에서 chatgpt.com, claude.ai, gemini.google.com 등에 로그인만 해두면 확장 프로그램이 그 세션을 그대로 활용합니다.

---

## AI는 어떤 것들이 있나요?

총 10종을 지원합니다. 로그인된 것 중 상위 8개까지만 표시됩니다.

chatgpt / gemini / claude / deepseek / mistral / grok / kimi / perplexity / pi / qwen

**2개 이상** 골라야 시작할 수 있습니다.

---

## 라운드는 어떻게 진행되나요?

1. **질문 입력** → 모든 선택 AI에 동시 전송
2. **각 AI 응답** (1000토큰 이하, 순서 무관)
3. **모든 AI 응답 완료** → 사용자 확인
4. **다음 라운드** → AI들에게 "지난번 다른 AI들은 이렇게 답했어" 정보 포함
5. 설정한 라운드(3~9)까지 반복
6. **최종 정리본(summary.md)** 자동 생성

---

## 특별 기능

### STOP 버튼
AI 응답이 너무 길거나 느릴 때 **수동으로 클릭**하여 중단합니다. 자동으로 멈추지 않습니다.

### 비밀채팅
특정 AI와 **1:1로만** 대화할 수 있습니다. 전체 브레인스토밍과 별개로 진행되며, **저장되지 않습니다** (휘발성).

---

## 저장은 어떻게 하나요?

- **자동 저장 없음** — 세션 중에는 데이터가 휘발됩니다
- **MD 파일 저장** — 토론 종료 후 요청 시:
  - AI별 파일 (chatgpt.md, claude.md 등) — 대화 흐름 기록
  - summary.md — 브레인스토밍 결과 종합 정리
- 비밀채팅 내용은 **저장되지 않습니다**

---

## 안 되는 것들

- ❌ 크롬 외 브라우저
- ❌ 모바일
- ❌ 여러 사용자 공유
- ❌ 서버 저장
- ❌ 자동 타임아웃 (STOP은 수동만)
- ❌ 데이터 암호화

**가능**: Windows / macOS / Linux + Chrome 최신 버전

---

## 구현 순서 (참고)

```
1단계: ChatHub 코드 분석
2단계: LLM 10종 로그인 세션 인식 모듈
3단계: LLM 선택 UI (Top 8, 2개 이상 필수)
4단계: 라운드 엔진 (질문 → 응답 → 전달)
5단계: 응답 전달 구조 (타 LLM 응답 공유)
6단계: MD 파일 저장 (LLM별 + summary)
7단계: STOP 버튼 기능
8단계: 비밀채팅 기능
9단계: 10종 LLM adapter 최적화
10단계: 에러 처리 및 예외 대응
```
