# 多者 AI Cowork 브레인스토밍 모드 — 요구사항 정의

## 목적
- LLM 여러 개와 동시에 대화하며 **브레인스토밍**을 수행
- 머릿속에 정리되지 않은 **아이템/프로젝트를 정리**
- **둘 이상의 LLM**이 참여

## 기능 요구사항

| No | 분류 | 요구사항 |
|----|------|----------|
| 1 | 응답 제약 | LLM은 라운드당 응답 **1회만** 가능 |
| 2 | 토큰 제약 | 응답 **1000토큰 초과 금지** (라운드마다 request에 자동 추가) |
| 3 | 라운드 간 전달 | 라운드 종료 후, 사용자 다음 request와 함께 각 LLM의 응답을 **타 LLM에게 전달**(자기 응답 제외). 전달 포맷은 개발 편의대로 json/md/yaml/텍스트 |
| 4 | 라운드 제어 | 라운드 수 제한: **시작 시 3~9 지정**, 중간 stop 가능 |
| 5 | 접속 방식 | **chathub 방식** — 크롬 브라우저에 로그인된 google 계정으로 접근 가능한 LLM과 대화 (**API키 없음**) |
| 6 | LLM 노출 | LLM 목록은 **top 8까지만** 노출 |
| 7 | 응답 순서 | LLM 응답 **순서 제한 없음** |
| 8 | 구현 형태 | **크롬 확장 프로그램**으로 개발 |
| 9 | 비밀채팅 | **비밀채팅** — 특정 LLM과 1:1 별도 대화 (전체 맥락과 분리, 1000토큰 제한, **저장 안 함 / 휘발성**) |
| 10 | 대상 LLM | 대상 LLM: chatgpt, gemini, claude, deepseek, mistral, grok, kimi, perplexity, pi, qwen (**10종**) |
| 11 | STOP | **STOP 기능** — 응답이 너무 길거나 느릴 때 사용자가 STOP 버튼 **수동 클릭** (자동 stop 없음) |
| 12 | 산출물 | 산출물: **LLM별 md 파일**(LLM명.md). 구조 = Request / Response / Request(+다른 LLM 응답) 흐름 |
| 13 | 산출물 | **최종 정리본 md** 생성 (브레인스토밍 결과 종합) |

## 산출물

| 산출물 | 설명 | 저장 |
|--------|------|------|
| `chatgpt.md`, `gemini.md`, `claude.md` … | 참여 LLM별 대화 흐름 (Request / Response / Request(+타 LLM 응답)) | 저장 |
| `summary.md` | 최종 정리본 (아이템/프로젝트 종합) | 저장 |
| 비밀채팅 | 특정 LLM과 1:1 별도 대화 | **저장 안 함 (휘발성)** |

## 참고 — 토큰 환산
- 1000토큰 ≈ 한글 700~1200자 (LLM별 토크나이저 상이, 브라우저 방식은 글자수 근사 가능)
- 100토큰 ≈ 한글 70~120자

## 미해결 / 후속 검토
- **크롬 확장 기술검토**
  - content script 주입
  - 각 LLM 사이트 DOM 대응
  - 토큰 카운트 근사
- 설계 / 유사코드는 **추후 단계**에서 진행
