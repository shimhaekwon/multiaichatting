# VER-230 — 클라우드 LLM 참여자 : 요구사항 정의

> 전제: `10-검토결과-및-방향.md`의 결론(무료 API 키 + Vite proxy/.env). 본 문서 = 기능 요구사항(FR) + 비기능(NFR).
> 대상 코드베이스: `E:\workspace\ai-sarangbang` (Vite + React + TS, core 순수성 유지).

---

## 1. 목적·범위

ai-sarangbang(로컬 Ollama 턴제 채팅)에 **클라우드 LLM을 무료·합법으로 추가 참여자로 등록**한다. 로컬 Ollama와 클라우드 AI가 **한 방에 혼합** 참여할 수 있다.

- **In**: provider 추상화(ollama/gemini/openrouter), CloudDriver, 무료 키 프록시, 로비 선택 UI, 영속, i18n
- **Out**: 세션 재사용/스크래핑/확장, prod 키 백엔드, 멀티유저, 유료 직접 연동(설계상 열어두되 미구현)

## 2. 용어

| 용어 | 정의 |
|---|---|
| provider | 발화 백엔드 종류 — `ollama`(로컬) / `gemini` / `openrouter` |
| CloudDriver | OpenAI 호환 `chat/completions` 스트리밍을 `AgentDriver`로 어댑트한 드라이버 |
| 프록시 라우트 | Vite dev/preview의 `/gemini`·`/openrouter` → 실제 API로 중계 + `.env` 키 헤더 주입 |

## 3. 기능 요구사항 (FR)

| ID | 요구사항 | 우선순위 |
|---|---|---|
| **FR-C01** | 참가자(AiSlot)는 `provider` 속성을 가진다(ollama/gemini/openrouter). 기존 슬롯은 `ollama`로 폴백(영속 호환). | 必 |
| **FR-C02** | 로비에서 슬롯별 **단일 통합 모델 드롭다운**(optgroup: `로컬(Ollama 동적)`·`Gemini`·`OpenRouter`(정적)을 **한 목록에 함께** 표시) + 선택 모델 옆 **provider/키 상태 배지**(키 유효 ✓ / 키 필요 🔒 — 키 상태는 **키를 실제 요구하는 헬스 프로브** 결과, FR-C11). provider는 별도 컨트롤 없이 **옵션 value `provider::model`에서 명시**(추론 아님 — 로컬·클라우드 ID 충돌 회피, 검토 R2-4)해 `AiSlot.provider` 설정. (사용자 확정 2026-06-08) | 必 |
| **FR-C03** | **CloudDriver**: provider별 baseUrl로 OpenAI 호환 `chat/completions`(stream=true) 호출, SSE(`data:` 라인, `[DONE]`)를 토큰 단위 `AsyncIterable<string>`로 yield. signal abort 시 fetch 취소. 빈 응답·HTTP 오류는 throw(coordinator가 "응답 없음" 처리). | 必 |
| **FR-C04** | **키 보관**: 키는 `.env`(gitignore)에만. dev/preview Vite proxy가 `Authorization` 헤더 주입. 클라이언트 번들·localStorage·history·MD에 키 노출 0. | 必 |
| **FR-C05** | **프록시 라우트**: `/gemini`→`generativelanguage.googleapis.com`, `/openrouter`→`openrouter.ai`. dev(server.proxy)·preview(preview.proxy) 동형. prod는 키 부재 시 명확한 안내(백엔드 follow-up). | 必 |
| **FR-C06** | **영속·마이그레이션**: `SCHEMA_V` 1→2, `isValidSlot`에 provider 검증. **`loadConfig`에 v1→v2 변환 단계 신규**(provider 없는 구슬롯 → `provider:'ollama'` 주입 후 v2로 통과) = 기존 방 보존. ※ `loadConfig`는 현재 변환 없이 버전 불일치를 null 폴백하므로 변환 미추가 시 기존 구성 전부 폐기됨(검토 H2). `reconcileModels`는 **ollama 슬롯만** 대조(클라우드 모델 missing 오탐 방지). | 必 |
| **FR-C07** | **혼합 방**: 로컬 Ollama + 클라우드 AI가 동일 방에서 턴제·자동대화·귓속말 정상 동작. coordinator/core 무수정. | 必 |
| **FR-C08** | **런타임 실패**: 발화 중 429·네트워크 오류는 기존 표식("[이름] 응답 없음", status=error)로 처리(coordinator가 throw를 흡수 — `coordinator.ts:286-290`). hardTimeout(idle) 적용. ※ 흡수되므로 **키 누락 진단은 런타임이 아닌 로비에서**(FR-C11). | 必 |
| **FR-C09** | **i18n**: provider/모델/키 안내 라벨 ko·en 동시. UI 한글 리터럴 게이트 준수(i18n 경유). | 必 |
| FR-C10 | system 프롬프트는 기존 `ollamaSystem` 재사용(기본). **단 클라우드는 `[이름]` 라벨 환각 위험이 큼**(프런티어 모델, 메모리 B1 계열) → S3에 환각 회귀 테스트 필수, 필요 시 클라우드용 system 분리(검토 H3). | 中 |
| FR-C11 | **로비 사전 헬스체크 — 키를 실제 요구하는 프로브**: OpenRouter `GET /api/v1/key`, Gemini `GET /v1beta/openai/models`(프록시가 .env 키 주입; `400/401/403`→키 필요). ⚠️ OpenRouter `/models`는 **공개**라 키 없어도 200(거짓 ✓) → **금지**(검토 R2-1, 라이브 확인). 런타임 throw는 coordinator가 흡수(무음)되므로 진단은 로비에서. | 中 |
| FR-C12 | OpenRouter는 Gemini와 동일 패턴 fast-follow(같은 CloudDriver, baseUrl/모델목록만 추가). | 中 |
| FR-C13 | **메시지 호환**: 다자 대화에서 발생하는 연속 same-role 메시지를 클라우드가 수용하도록 처리. Gemini OpenAI-호환은 연속 same-role 허용 확인됨(위험↓) — 방어적으로 연속 same-role 병합 옵션 보유(검토 M-role). | 中 |
| **FR-C14** | **Ollama-off 클라우드-only 입장**(사용자 확정): Ollama 미연결이어도 로비 동작 — 로컬 그룹만 비고 클라우드 정적 목록 선택·입장 가능. `canEnter`=`loadState!=='loading' && slots>=1 && 모든 슬롯 model이 (로컬목록 ∪ cloud config)에 존재`(**유효=목록 존재이지 헬스 아님** — 🔒여도 입장 허용). error/empty 분기(`:116-136`)는 **인라인 알림으로 격하**(슬롯 UI 항상 렌더). 기본 슬롯은 load settle 후 로컬 우선·없으면 첫 클라우드(로딩 중 churn 방지, 검토 R2-6). | 必 |

## 5. 수용 기준 (검증 시나리오)

1. `.env`에 Gemini 무료 키 설정 → `npm run dev` → 로비에서 provider=gemini·모델 선택 → 입장 → 사람 입력 시 **클라우드 AI 실제 발화**(스트리밍 타이핑).
2. 로컬 Ollama AI + Gemini AI를 **같은 방**에 두고 턴제·자동대화 정상.
3. 키 미설정 시 **로비에서 진단 안내**(어떤 .env 키 필요) — 런타임 무음 "응답 없음"이 아니라 입장 전 안내. 앱 크래시 0.
4. 키가 브라우저 devtools(Network 헤더는 프록시 뒤, Sources 번들)·localStorage에 노출되지 않음 확인.
5. 게이트: tsc 0 / vitest(신규+회귀) / build OK.
