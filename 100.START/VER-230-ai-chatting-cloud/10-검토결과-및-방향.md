## 0. 요청 요약

> "ai sarangbang 정상 작동 … 이 기능에 더하여 **cloud LLM들을 대화 참여자로 추가 등록** 기능 요청 / **api key 없이 해당 LLM 채팅 계정으로 처리 가능한지** 검토 요청"

즉 두 갈래:
1. (기능) 클라우드 LLM(ChatGPT·Claude·Gemini 등)을 ai-sarangbang의 대화 참여자로 추가
2. (방식) **API 키 없이, 각 LLM의 로그인된 채팅 계정/세션을 재사용**해서 구동 가능한가?

## 1. 결론 (먼저)

| | 판정 |
|---|---|
| 클라우드 LLM을 참여자로 추가 (일반) | **가능** — 드라이버 seam이 깨끗함 |
| **"API 키 없이 채팅 계정으로"** | **불가** (이 앱 구조에서 기술적 차단) + **약관 위반** + **계정 정지 위험** + **v1 폐기 방식으로의 회귀** |
| 채택 방향 | **무료 API 키** (Gemini 무료 티어 + OpenRouter 무료 모델), 키는 **Vite proxy + .env**로 보관(클라이언트 노출 0) |

"무키"는 막혔지만 "무료"는 열려 있다 — 비용 0이라는 **실질 목적은 무료 키로 달성**한다.

## 2. 근거 ① — 코드 seam은 깨끗하다 (어디 꽂을지는 문제 아님)

- `AgentDriver.speak(ctx, signal): AsyncIterable<string>` 단일 계약(`drivers/AgentDriver.ts:6-8`). `OllamaDriver`가 1:1 템플릿이라 **core/coordinator 무수정**으로 새 드라이버 추가 가능.
- 단, 로비가 Ollama `/api/tags`에 강결합(`Lobby.tsx:4,38`, `ollamaApi.ts:32`) → 클라우드는 **별도 모델 목록 + provider 구분 + 영속 스키마**(`persist.ts`) 수정이 불가피.
- 즉 문제는 **"무엇을 호출하느냐"**.

## 3. 근거 ② — "무키 = 로그인 세션 호출"은 로컬 웹앱(확장 아님)에서 구조적으로 차단

순수 브라우저 웹페이지의 JS는 `chatgpt.com`·`claude.ai`·`gemini.google.com`의 로그인 세션 엔드포인트를 호출·판독할 수 없다. **4겹 차단**:

| 차단막 | 내용 |
|---|---|
| httpOnly 인증쿠키 | 세션 쿠키를 JS가 못 읽음. **Vite proxy로도 우회 불가** — 프록시는 단순 HTTP 중계라 *사용자 브라우저의 해당 도메인 쿠키 jar*에 접근 못 함 |
| CORS | 대상 도메인이 `localhost`를 허용 origin으로 반환하지 않음(자격증명 포함 요청은 와일드카드 불가) |
| SameSite 쿠키 | `Lax/Strict`면 cross-site 요청에 인증쿠키 자동 첨부 안 됨 |
| Cloudflare 봇탐지 + preflight | OPTIONS preflight엔 쿠키 미첨부 → `cf_clearance` 챌린지 통과 불가, `__cf_bm` 봇 스코어 |

→ 기술적 우회는 **브라우저 확장(extension)**이라야 열리는데, 이는 ai-sarangbang의 핵심 제약 **"로컬 웹앱(확장 아님)"**(`221 §3` 형태 제약)과 정면 충돌.

## 4. 근거 ③ — 약관(ToS)·계정 안전 : 3사 모두 명시 금지 + 2026 실제 정지

- **OpenAI** Terms "What you cannot do": *"except as permitted through the API, use any automated or programmatic method to extract data or output from the Services, including scraping, web harvesting, or web data extraction"* — API 외 프로그램적 접근 금지.
- **Anthropic** Consumer Terms §3: *"Except when you are accessing our Services via an Anthropic API Key … to access the Services through automated or non-human means, whether through a bot, script, or otherwise"* + *"To crawl, scrape, or otherwise harvest data or information from our Services …"* — Claude.ai의 OAuth 토큰은 Claude.ai/Claude Code 전용.
- **Google**: 소비자용 Gemini 앱(`gemini.google.com`)은 자동화/구성요소 추출 금지. Gemini **API**는 공식 키 경로(개발·상업 목적)라 정당 — 단 이건 "키 있는" 경로.
- **실제 정지 사례**: 2026년 초 **Anthropic "OpenClaw" 단속** — 비정상 트래픽/비브라우저 user-agent/자동화 패턴을 탐지해 세션 거부·레이트리밋, **제3자 하네스로 구독 재사용한 계정 실제 정지**(일부는 부수 피해로 인정·복구). (VentureBeat, The Register 보도)
- 비공식 라이브러리(revChatGPT 등)는 **2023년 이미 archived(사실상 사망)**, Cloudflare 우회 의존.

→ 단기간 동작하더라도 차단·정지는 시간문제이며, **사용자의 실제 ChatGPT/Claude/Gemini 계정이 위험**.

## 5. 근거 ④ — 이건 v1(multichat)이 버린 바로 그 방식 + 우리 설계 결정과 충돌

- **v1 폐기 사유(문서 근거)**: 백그라운드 탭 throttle(rAF 정지·setInterval clamp)→done 미발화 hang, staleness(라운드 밀림), 사이트별 DOM selector/fill 깨짐, CSP의 인라인 스크립트 차단 → **"DOM 스크래핑 fragility 내재 → network-first/API로"** 결론내고 ai-sarangbang으로 pivot한 것. (`900.AICOWORK/20260602-114351-처리내역.md`, `210.설계산출물-multichat/224-네트워크인터셉트-task7.md`)
- **설계 충돌**: `220.설계산출물-ai-sarangbang/224 §3`은 P2를 **"공식 API + 키 보호 백엔드 프록시", "무료 최우선·유료는 최후"**로 명시. VER-230의 "무키 채팅계정"은 이 결정의 반대 방향.

## 8. 채택 방향 (사용자 결정 2026-06-08)

1. **방식 = 무료 API 키** (Gemini 무료 티어 + OpenRouter 무료 모델). 로컬 Ollama와 **혼합 방** 구성.
2. **키 보관 = Vite proxy + `.env`(gitignore)** — dev/preview 프록시가 헤더로 키 주입, 클라이언트 번들·localStorage 노출 0(코드생성원칙 §3.4). **prod 백엔드 프록시는 follow-up**.
3. **작업 범위 = 설계문서 먼저**(본 폴더 + `230.설계산출물-ai-chatting-cloud/`) → 사용자 확인 후 구현.
