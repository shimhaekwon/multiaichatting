## 2. CloudDriver 설계 (`src/drivers/CloudDriver.ts` 신규)

`OllamaDriver`를 1:1 템플릿으로 한다. 차이 = 엔드포인트/포맷(OpenAI 호환)과 인증(프록시가 주입).

```ts
export interface CloudDriverConfig {
  baseUrl: string                  // 클라이언트 기준 상대경로: '/gemini' | '/openrouter' (동일출처→프록시)
  model: string | ((p: Participant) => string)
  system?: (p: Participant) => string   // 기존 ollamaSystem 재사용(FR-C10)
  fetchImpl?: typeof fetch         // 테스트 주입
}
```

- **호출**: `POST ${baseUrl}/chat/completions`, body `{ model, messages, stream:true }`.
  - `messages` = `buildChatMessages(ctx, system)` — OllamaDriver의 `buildOllamaMessages`와 동일 규칙(self=assistant·타자=user, `[이름]` 라벨, roster로 id→이름). **공용 헬퍼로 추출** 권장(`drivers/openaiMessages.ts`) → 두 드라이버 공유.
  - **Authorization 헤더는 클라이언트가 넣지 않는다** — 프록시가 주입(§5 보안).
- **SSE 파싱**: 응답은 `text/event-stream`. 라인 단위로 `data: ` 접두 제거 → `JSON.parse` → `choices[0].delta.content` yield. `data: [DONE]`이면 종료. 비-JSON/빈 라인 무시(견고). (Ollama의 NDJSON 파서와 형태만 다름.)
- **abort**: `signal`을 fetch에 전달, 루프 중 `signal.aborted` 검사 → `AbortError` throw. (OllamaDriver와 동일.)
- **오류/빈응답**: `!res.ok` → `throw Error('HTTP ' + status)`; 토큰 0개로 정상 종료 시 coordinator가 `status='error'`("응답 없음") 처리(`coordinator.ts:283`). 429(레이트리밋)도 동일 경로. ※ throw는 `coordinator.ts:286-290`이 **흡수**해 메시지가 소실되므로, 키/연결 진단은 런타임이 아니라 **로비 헬스체크**로(FR-C11, 검토 M-diag).
- **연속 same-role 처리**(검토 M-role): 다자 대화 context는 타자 발언이 전부 `user`라 user 연속이 생긴다. Gemini OpenAI-호환은 연속 same-role을 **허용**(웹 확인)하나, 다른 백엔드 호환 위해 **연속 same-role content를 개행 병합**하는 옵션을 둔다.
- **로비 헬스체크**(FR-C11, 검토 M-diag/R2-1): 클라우드 provider는 입장 전 **키를 실제 요구하는 프로브**로 점검 — OpenRouter `GET /api/v1/key`, Gemini `GET /v1beta/openai/models`(§4.2 매핑). ⚠️ OpenRouter `/models`는 공개(키 없어도 200)라 키 신호로 **부적합**(라이브 확인). 실패 시 필요한 .env 키 안내, 런타임 무음 회피.
- **stripLeadingSelfLabel** 래핑 유지(이름 메아리 제거) — buildSession에서 OllamaDriver와 동일하게 감쌈. ※ 단 클라우드는 라벨이 **문장 중간**에 올 수 있어 선두-스트립만으론 부족(검토 H3) → S3 환각 회귀 테스트로 검증, 필요 시 클라우드 system 강화/분리.

### provider별 엔드포인트(프록시 target)

| provider | client baseUrl | 프록시 target | rewrite | 인증 |
|---|---|---|---|---|
| gemini | `/gemini` | `https://generativelanguage.googleapis.com` | `^/gemini` → `/v1beta/openai` | `Authorization: Bearer ${GEMINI_API_KEY}` |
| openrouter | `/openrouter` | `https://openrouter.ai` | `^/openrouter` → `/api/v1` | `Authorization: Bearer ${OPENROUTER_API_KEY}` (+선택 `HTTP-Referer`,`X-Title`) |

→ `/gemini/chat/completions` → `…/v1beta/openai/chat/completions`. Gemini의 **OpenAI 호환 엔드포인트**라 OpenRouter와 같은 코드로 처리.

## 3. 데이터모델 변경 (`src/app/config.ts`, `src/app/persist.ts`)

```ts
export type Provider = 'ollama' | 'gemini' | 'openrouter'

export interface AiSlot {
  id: string
  name: string
  model: string
  provider: Provider     // ★신규. 구버전 로드 시 'ollama' 폴백
  think?: boolean        // ollama 전용(클라우드는 미사용 → 전송 안 함)
}

export interface RoomConfig {
  v: 2                   // ★ 1→2 (provider 추가). persist SCHEMA_V 동기
  ais: AiSlot[]
  humanName?: string
  baseUrl?: string       // ollama 직접주소 override(기존)
}
```

- **config.cloud 블록**(신규): provider별 `baseUrl`(상대경로)와 **정적 모델 목록 상수**.
  ```ts
  cloud: {
    gemini:     { baseUrl: '/gemini',     models: ['gemini-2.5-flash'] },   // ※ 모델 ID는 구현 시 현행 카탈로그/키로 재확인(미검증 단정 금지, 검토 R2-5). 추가 Flash는 확인 후
    openrouter: { baseUrl: '/openrouter', models: ['openrouter/free'] },    // 'openrouter/free'=무료 자동 라우터(라이브 확인). 개별 ':free' ID는 카탈로그 변동 → 구현 시 /models로 채움
  }
  ```
  (모델명은 provider 문서 기준 수동 상수 — 1차. 갱신 주기는 미해결 로그.)
- **persist**(`persist.ts:6` `SCHEMA_V`, `:27-43` `loadConfig`, `:56-75` `isValidConfig/isValidSlot`, `:79-82` `reconcileModels`):
  - `SCHEMA_V` 1→2, `isValidSlot`에 `provider ∈ Provider` 검증 추가.
  - **마이그레이션 — 신규 단계 필수(검토 H2)**: 현재 `loadConfig`는 변환 없이 `v !== SCHEMA_V`면 그냥 `null` → v1 구성이 **전부 폐기**된다. 따라서 `JSON.parse` 직후·`isValidConfig` 전에 **`migrate(parsed)`** 를 끼운다: `parsed.v === 1`이면 각 슬롯에 `provider:'ollama'` 주입 + `v:2` 설정 → 그 뒤 v2로 검증. = 기존 방 보존.
  - **`reconcileModels` 가드(검토 추가갭)**: 현재 모든 슬롯 model을 Ollama `/api/tags`와 대조해 missing 판정 → 클라우드 모델 전부 오탐. `provider==='ollama'` 슬롯만 대조하도록 수정.

## 4. UI 변경 (`src/ui/Lobby.tsx`, `src/i18n`) — 통합 선택 UI (사용자 확정 2026-06-08, 2차 검토 반영)

### 4.1 통합 모델 드롭다운 + provider 명시(추론 아님)
- 슬롯 행 = `[통합 모델 select] [상태 배지] [이름] [think?(ollama만)] [✕]`.
- 단일 `<select>`에 `<optgroup>`으로 **로컬+클라우드 동시 표시**:
  - `로컬 (Ollama)` — `listModels()` 동적(없으면 그룹 생략).
  - `Gemini` / `OpenRouter` — `config.cloud[*].models` 정적(항상 표시).
- **옵션 value = `provider::model`**(검토 R2-4): 로컬 태그와 클라우드 ID가 같아도 안전. `changeModel`이 value를 split해 `{provider, model}`을 **함께** 설정 — `model→provider` 맵/추론 폐기(충돌 클래스 제거). 이름 자동값/think 규칙 유지 — 클라우드면 `think=undefined`(검토 L).
- **복원 슬롯 provider 우선순위**(검토 R2-4): persist된 `slot.provider`가 **권위**. 재구성 시 다시 추론하지 않음(사용자가 명시 재선택할 때만 변경).
- **`gone`(존재) 판정 수정**(검토 R2-3): 현재 `Lobby.tsx:155` `gone = !models.some(...)`는 Ollama 목록만 봐 **클라우드 슬롯을 전부 깨진 것으로** 표시. → `slot.provider==='ollama' ? models : config.cloud[slot.provider].models` 기준 존재 판정으로 변경.

### 4.2 상태 배지 + 헬스체크 (키를 실제 요구하는 프로브)
- 배지: 로컬=provider 표시. 클라우드=`✓`(키 유효) / `🔒`(키 없음·무효).
- **헬스 엔드포인트 — provider별로 키를 실제 요구하는 것**(검토 R2-1, 라이브 확인):
  - **OpenRouter**: `GET /api/v1/key`(인증 필수). ⚠️ `/api/v1/models`는 **공개(키 없어도 200)**라 거짓 ✓ → **쓰면 안 됨**.
  - **Gemini**: `GET /v1beta/openai/models` — 프록시가 .env 키 주입. 매핑: `200`=✓, `400/401/403`("valid API key" 류)=🔒, 그 외/네트워크=미연결.
- 키는 클라이언트가 못 읽으므로(서버 .env) 위 프로브 **결과로만** 판정. 로비 진입/재연결 시 provider별 1회. 선택 슬롯이 🔒면 인라인 경고(입장은 허용 — FR-C14, 런타임 무음 회피).

### 4.3 게이트 완화 — Ollama-off 클라우드-only 입장 (FR-C14)
- `canEnter`(`:83`): `loadState !== 'loading' && slots.length>=1 && 모든 슬롯 model이 (로컬목록 ∪ cloud config)에 존재`. **유효=목록 존재(헬스 아님)** — 🔒여도 입장 허용(FR-C14). `models.length>0`(Ollama) 의존 제거.
- error/empty 분기(`:116-136`)가 슬롯 UI 전체를 가리던 것을 **인라인 알림**("Ollama 미연결 — 클라우드만 사용 가능")으로 격하. **슬롯 UI는 cloud config가 있어 항상 렌더**. retry/데모는 보조.
- **loading 중 안정성**(검토 R2-6): `loading`이면 입장 비활성(`canEnter` false), 기본 슬롯은 load settle 후 결정(로컬 있으면 첫 로컬, 없으면 첫 클라우드) — 로딩 중 cloud 선택→로컬 도착 churn 방지.
- D-7(자동입장 안 함)·StrictMode-safe 세션 생성(App.tsx)은 **무변경**.

## 5. 보안 설계 (코드생성원칙 §3.4) — 핵심

- **키는 `.env`에만**(repo 루트). **`.gitignore`에 `.env`·`.env.* 추가(신규 — 검토 H1)**: 현재 `.gitignore`는 `*.local`만 있어 `.env`는 추적 대상이다. 안 고치면 첫 `git add`에 키 유출. `.env.example`(키 없이 변수명만) 커밋.
- **변수명에 `VITE_` 접두 금지** — Vite는 `VITE_*`만 클라이언트 번들에 인라인한다. `GEMINI_API_KEY`/`OPENROUTER_API_KEY`처럼 접두 없이 두면 **클라이언트 번들에 절대 포함되지 않음**.
- `vite.config.ts`(Node 컨텍스트)에서 `loadEnv(mode, cwd, '')`로 키 로드 → **proxy `configure`의 `proxyReq` 훅에서 `Authorization` 헤더 주입**. 클라이언트 코드·localStorage·history·MD 어디에도 키 없음.
- 결과: devtools Network는 프록시(`/gemini`) 뒤라 키 미노출, Sources 번들에도 키 문자열 0.
- **prod 한계 명시**: Vite preview/dev 프록시는 개발용. 실배포 시 동일 역할의 경량 백엔드 프록시 필요(=follow-up). prod에서 키 없으면 CloudDriver는 친절한 오류.
