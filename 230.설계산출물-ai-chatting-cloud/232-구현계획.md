- **기존 테스트 마이그레이션 필수(검토 R2-2 — '회귀 0' 공짜 아님)**: `persist.test:36` 불일치 단언 `{v:2}`→`{v:3}`(SCHEMA_V=2와 충돌 회피), `persist.test`의 `cfg`/`dup` 및 `buildSession.test:48/56/75` 리터럴에 `provider:'ollama'` 추가·`v:2`. 이 수정 없이는 tsc/vitest 실패(회귀가 아니라 의도된 스키마 변경의 동반 수정).
- `Lobby`: ollama 경로(`/api/tags`)는 로컬 그룹으로 유지. **게이트 완화로 기존 'Ollama 필수' 동작이 의도적으로 바뀜**(회귀 아님 — 클라우드-only 입장 테스트로 명시). missing/reconcile은 ollama 슬롯만.
- `coordinator`/core: **변경 0** 확인(diff에 core 미포함).
- 기존 게이트 150 테스트 전건 green = 회귀 0 실증.

## 4. 라이브 실증 (코드생성원칙 §6.3)

1. Gemini 무료 키 발급(AI Studio, 카드 불필요) → `.env`에 `GEMINI_API_KEY=…`.
2. `npm run dev` → 로비에서 슬롯 provider=`gemini`·모델 선택 → 입장.
3. 사람 입력 → **클라우드 AI 스트리밍 발화** 확인(타이핑 효과).
4. 로컬 Ollama AI + Gemini AI **혼합 방** 턴제·자동대화 정상. **Ollama 끈 상태에서 클라우드-only 입장·발화**도 확인(게이트 완화). 통합 드롭다운 optgroup·배지 `✓`/`🔒` 표시 확인.
5. 키 미설정 시 **로비 헬스체크 진단**(필요 .env 키 안내)·배지 `🔒`. ⚠️ **OpenRouter는 키-요구 프로브(`/api/v1/key`)라야** 거짓 ✓ 안 뜸(검토 R2-1 — 공개 `/models` 사용 금지). 런타임 무음 아님, 크래시 0.
6. devtools: Network 헤더에 키 노출 0(프록시 뒤), Sources 번들·localStorage에 키 문자열 0.
7. (사용자 실 환경 = 원격 PC면 git pull 후 재현.)
