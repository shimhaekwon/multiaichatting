## 1. 핵심 TS 타입 (`core/types.ts` 신규)

> Coordinator가 프레임워크 무관 순수 모듈([221] §6)이므로 타입도 UI/드라이버와 분리해 `core/types.ts`에 단독 배치. [222] §4·§8의 `Intent`/`Message`/`Participant`/`SpeakContext`/`AgentDriver`가 모두 여기서 파생.

```ts
export type ParticipantId = string
export type MessageId = string
export type RoomSessionId = string   // "YYYYMMDD-HHmm-xxxxxx" (§1.3)

// 참가자 — 사람·AI 일반화 (v1 Slot 계승·확장)
export interface Participant {
  id: ParticipantId
  name: string
  kind: 'human' | 'ai'
  seat: number              // [H4] 좌석 순서(0-based). D4 tie-break(동시 도착 시 좌석순) = collectIntents enqueue 순서([222] §4)
  persona?: string          // AI 캐릭터 프롬프트(목/Ollama/API 공통). human은 미사용
  color?: string            // 90s 감성 발화 색([221] §1). 미지정 시 UI 기본 팔레트
}

// 발언 상태 — v1 ResponseStatus 계승. [H1] [222] Participant turn-state의 부분집합 아님:
// error=Message 전용(드라이버 실패), idle/queued=Participant 전용. 대응은 [222] §3 표.
export type MessageStatus = 'streaming' | 'done' | 'stopped' | 'error'

// 공개 발언 1건
export interface Message {
  id: MessageId
  turnNo: number            // 이 발언이 속한 턴(1-based). 사람 입력이 턴을 연다([221] D1)
  by: ParticipantId         // 발화자
  role: 'human' | 'ai'      // by.kind 미러. [M-3] beginMessage가 단독 채움(외부 설정 금지) → 동기화 책임 단일화
  text: string              // 원문 전체(요약·절삭 없음, §3)
  status: MessageStatus     // [C-1] 사람 메시지는 생성 즉시 'done'(스트리밍 없음) → §4 done 필터에 포함되어 AI가 봄
  ts: number                // 발언 시작 시각(epoch ms)
}

// 발언 의사 — floor 큐의 원소([222] §4). 출력 권한이 아니라 "말하겠다"는 신청
export interface Intent {
  by: ParticipantId
  turnNo: number
}

// 방 세션 — 라이브 턴제 1회분(v1 RoundSession 계승)
export interface RoomSession {
  id: RoomSessionId
  createdAt: number
  status: 'idle' | 'turn_active'      // [222] §3 Room status와 동일
  turnNo: number                      // 현재/최근 턴(1-based). idle 직전 값 유지
  floorHolder: ParticipantId | null   // [L3] 현재 출력 중 참가자(UI 하이라이트, [222] §3). idle이면 null
  participants: Participant[]         // 사람 1 + AI N (seat 순)
  history: Message[]                  // 공개 대화 누적(whisper 미포함 — §2)
}
```

> **v1 → v3 매핑** [결정]

| v1 (multichat, [214]) | v3 (ai-sarangbang) | 변경 사유 |
|------------------------|--------------------|-----------|
| `Slot{slot,botId}` | `Participant{id,name,kind,seat,persona?,color?}` | 사람+AI 일반화 — 라이브 턴제는 사람도 1급 참가자([221] R1). `slot`→`seat`(그리드 좌석 → 발언 좌석순, D4 tie-break으로 용도 변경) |
| `RoundResponse.status: ResponseStatus`(`'waiting'\|'completed'\|'stopped'\|'error'`) | `MessageStatus`(`'streaming'\|'done'\|'stopped'\|'error'`) | `waiting`→`streaming`(라이브는 대기 즉시 출력), `completed`→`done`([222] turn-state 용어 통일) |
| `RoundRecord{round,userPrompt,responses}` | `Message[]`(평면 history) | 라운드(병렬 묶음)가 턴(직렬 발언열)로 바뀌어 라운드 단위 묶음이 불요. `turnNo`로 턴 경계 표현 |
| `RoundSession.maxRounds/tokenLimit/currentRound` | (없음) | 라운드 상한·토큰 예산은 v1 병렬 브레인스토밍 개념. 라이브 턴제는 사람 입력이 턴을 열고 큐 소진으로 닫힘([221] D1/R3) — 상한 불요 |
| `SecretChat` | `Whisper`(§2, 휘발) | 비밀채팅 불변식 계승, 타입명만 [221] R4 용어로 |

### 1.3 id 체계 — v1 계승 [결정]

- `nanoid`의 `customAlphabet(alphabet, 6)`로 6자리 무충돌 ID 생성(v1 계승). `RoomSession.id` = `"YYYYMMDD-HHmm-" + nanoid6` (예 `"20260602-1430-a1b2c3"`).
- `Message.id`/`ParticipantId`도 동일 nanoid6(접두 불요 — 타입으로 구분). 결정적 정렬은 `ts`/`turnNo`로(ID는 동등성·키 용도).

```ts
import { customAlphabet } from 'nanoid'
const nano6 = customAlphabet('0123456789abcdefghijklmnopqrstuvwxyz', 6)
export function newSessionId(now = new Date()): RoomSessionId {
  const p = (n: number, w = 2) => String(n).padStart(w, '0')
  const stamp = `${now.getFullYear()}${p(now.getMonth() + 1)}${p(now.getDate())}-${p(now.getHours())}${p(now.getMinutes())}`
  return `${stamp}-${nano6()}`
}
export const newMessageId = (): MessageId => nano6()
```

## 2. Whisper 휘발 불변식 (R4 계승) [결정·중요]

> v1 비밀채팅(`214` `SecretChat`, "atomWithStorage 금지")의 **휘발 불변식**을 그대로 계승. [221] R4 · [222] §7.

```ts
// 휘발 — 저장·MD로그·스냅샷 어디에도 들어가지 않는다
export interface Whisper {
  target: ParticipantId                                  // 1:1 상대(AI)
  messages: { by: 'human' | ParticipantId; text: string }[]   // by='human' 또는 target
}
```

- **불변식**: `Whisper`는 **in-memory 전용**. `RoomSession.history`(§1) · MD 로그(§3) · localStorage 스냅샷(§5) **어디에도 미포함**.
  1. `Message`로 승격되지 않는다(floor 루프 밖, [222] §7).
  2. MD 생성기(§3)는 `Message[]`만 입력받으므로 구조적으로 whisper에 닿지 못한다(타입 차단).
  3. 스냅샷 직렬화(§5)는 `RoomSession`만 직렬화 → `Whisper`는 별도 변수라 자동 제외.
- 소멸 시점: 세션 종료 · 대상 퇴장 · 패널 닫힘 시 참조 폐기(GC). 영속 계층 없음.
- [중요] whisper를 `RoomSession`이나 스냅샷 객체에 **필드로 달지 말 것** — 달면 §5 직렬화에 새어 들어간다. 별도 보관(예 coordinator 내부 `Map<ParticipantId, Whisper>`)이 불변식의 구조적 보증.

## 3. MD 로그 포맷 (v1 `214` §4 계승)

> **결정적 생성** — LLM 요약/절삭 **없음**, 원문 전체. **공개 대화만**(whisper 제외, §2). v1 `214` `[G7]`(결정적·LLM 호출 없음) 계승. 웹앱이므로 v1 `chrome.downloads`를 **Blob 다운로드**로 치환(§3.3).

### 3.1 생성 함수 (`src/log/md-export.ts` 신규)

```ts
import { RoomSession, ParticipantId } from '../core/types'   // [M-D] md-export는 src/log → core는 ../

// 참가자별 단일 관점 파일: "<name-slug>.md" (자기 발언 흐름 + 맥락)
export function buildParticipantMarkdown(session: RoomSession, participantId: ParticipantId): string
// 전체 합본: "summary.md" (턴 순서대로 전원 발언)
export function buildSummaryMarkdown(session: RoomSession): string
```

- **입력은 `RoomSession`(=`Message[]`)뿐** — whisper에 구조적으로 닿지 못함(§2 불변식 2).
- 발언 순서 = `history`의 `turnNo` → `ts` 순(결정적). 헤더 표시명 = `Participant.name`.
- `status !== 'done'`(streaming 중 종료·stopped·error)인 발언은 본문 대신 **`_(중단/응답없음)_`** 으로 표기(자리 유지). v1 `_(응답 없음)_` 계승, 라이브 중단(`stopped`) 포함이라 문구 확장.

### 3.2 템플릿

**참가자별 `<name>.md`** (단일 관점 — 자기 발언 + 직전 맥락):

```markdown
# {participant.name} — 사랑방 기록
> 세션: {createdAt} | 참가자: {participants.length}명 | 턴: {maxTurnNo}

## Turn 1
[{직전 발화자}] {직전 발언}
...
[{participant.name}] {자기 발언 원문 전체}
<!-- status !== 'done' 이면 본문 대신: _(중단/응답없음)_ -->

## Turn 2
...
```

**`summary.md`** (전체 합본 — 턴 순서대로 전원 발언):

```markdown
# 사랑방 대화 기록
> 세션: {createdAt} | 참가자: {participants.length}명 | 턴: {maxTurnNo}

## Turn 1
[{human.name}] {사람 발언}
[{aiName}] {AI 발언 원문 전체}
[{aiName2}] {AI 발언 원문 전체}
<!-- status !== 'done' 인 발언은: [{name}] _(중단/응답없음)_ -->

## Turn 2
[{human.name}] {사람 발언}
...
```

> **참가자별 vs summary 차이**: **참가자별 `<name>.md`** = 그 참가자 **단일 관점**(자기 발언 + 발언 시점까지 본 공개 맥락). [H3] 주의: §4 `buildSpeakContext`는 드라이버 주입용이라 self를 **제외**하나, 본 MD는 사람이 읽는 기록이라 **자기 발언을 포함**(용도가 달라 §4와 동일하지 않음). 표시명은 모두 `Participant.name` — 사람 발화도 `[나]` 하드코딩 금지(한글 하드코딩·i18n 회피, [221] §6). **`summary.md`** = **전원 합본** — 턴 순서대로 그 턴의 모든 공개 발언을 `[이름]` 대괄호([221] §1 감성)로 나열. 둘 다 whisper 제외·원문 전체·결정적.

### 3.3 저장 — Blob 다운로드 (chrome.downloads 치환) [결정]

> v1 `[G6]`은 `chrome.downloads`(확장 권한). 본 제품은 **로컬 웹앱**([221] §2)이라 확장 API 불가 → 표준 **Blob + anchor 다운로드**로 치환. LLM 호출·요약 없음은 동일.

```ts
function downloadMarkdown(filename: string, md: string): void {
  const url = URL.createObjectURL(new Blob([md], { type: 'text/markdown;charset=utf-8' }))
  const a = document.createElement('a')
  a.href = url; a.download = filename
  document.body.appendChild(a); a.click(); a.remove()
  URL.revokeObjectURL(url)          // 누수 방지
}

// [M-E] 파일명 안전화: 한글 보존, 경로 금지문자 제거, 공백→_, 길이 제한
function slug(name: string): string {
  return (name || 'unnamed').replace(/[\\/:*?"<>|]/g, '').replace(/\s+/g, '_').slice(0, 40)
}

export function saveSessionMarkdown(
  session: RoomSession, participantIds: ParticipantId[], includeSummary: boolean,
): void {
  for (const id of participantIds) {
    const p = session.participants.find(x => x.id === id); if (!p) continue
    downloadMarkdown(`${slug(p.name)}.md`, buildParticipantMarkdown(session, id))
  }
  if (includeSummary) downloadMarkdown('summary.md', buildSummaryMarkdown(session))
}
```

- 파일명 = `slug(participant.name)`(한글/공백 안전화) + `.md`. 합본 = `summary.md`.
- 다중 파일은 anchor 클릭 연속 호출(브라우저가 일괄 다운로드 허용 — 권한 프롬프트 없음). 단 일부 브라우저는 연속 다운로드를 차단할 수 있어, P0은 **순차 호출 + 미세 지연** 또는 단일 zip 폴백을 [미정 M1]로 남김.

## 4. SpeakContext · 컨텍스트 구성 [C-2 정정]

> [222] §8 `SpeakContext` 구성 정본. **결정적 차이**: 라이브 직렬 + **stateless 드라이버**(매 `speak()` 무상태, [222] §8)라, AI가 자기 직전 발언을 기억하려면 **공개 대화 전체(자기 포함)** 를 매번 넘겨야 한다. 화자 구분은 드라이버 prompt의 `[이름]` 라벨로. → v1 `216`의 "self 제외(병렬 라운드, 각자 자기 스레드 보유)"와는 **다른 시맨틱**(라이브엔 자기 스레드가 없음 — self 제외 시 AI 치매).

```ts
import { Participant, Message, RoomSession, ParticipantId, Whisper } from './types'

export interface SpeakContext {        // [222] §8 재게재 — 본 절이 구성 정본
  participant: Participant              // 발화자(드라이버가 "당신은 [participant.name]"로 프레이밍)
  publicHistory: Message[]             // 공개 대화(자기 포함, done만). 드라이버가 [이름] 라벨로 자타 구분
}

/** [C-2] 공개 done 전사 그대로 — self 포함(stateless 드라이버의 자기 기억). whisper는 history에 없어 자동 제외(§2). */
export function buildSpeakContext(session: RoomSession, speakerId: ParticipantId): SpeakContext {
  const speaker = session.participants.find(p => p.id === speakerId)!
  const publicHistory = session.history.filter(m => m.status === 'done')   // 완료분만(streaming/stopped/error 미주입)
  return { participant: speaker, publicHistory }
}

/** [H-3] 귓속말 스레드를 SpeakContext로 정규화 → 드라이버 seam([222] §8)이 동일 시그니처로 수용. 공개 history 불참조(사적 격리). */
export function whisperContext(target: ParticipantId, w: Whisper, session: RoomSession): SpeakContext {
  const speaker = session.participants.find(p => p.id === target)!
  const publicHistory: Message[] = w.messages
    .filter(m => m.text)
    .map((m, i) => ({ id: `w${i}`, turnNo: -1, by: m.by === 'human' ? '__human__' : m.by,
                      role: m.by === 'human' ? 'human' : 'ai', text: m.text, status: 'done', ts: 0 }))
  return { participant: speaker, publicHistory }   // 휘발 — 이 합성 Message[]는 history/MD/스냅샷에 안 들어감
}
```

- **[C-2] self 포함**: v1과 결정적 차이. 라이브 직렬 + stateless 드라이버라 자기 발언을 빼면 AI가 자기 직전 주장을 잊어 다음 턴에 모순(치매). 화자 라벨로 자타 구분하므로 echo chamber는 prompt 설계로 방지(드라이버 책임).
- **공개 history만 · done만**: whisper 미포함(§2). `streaming`/`stopped`/`error`는 미완이라 제외(v1 completed-only 계승). 사람 발언은 즉시 `done`(§1 [C-1])이라 포함됨.
- **whisperContext**: `Whisper.messages`(경량)를 `SpeakContext`로 정규화 — 드라이버가 공개/귓속말을 동일 타입으로 받음(H-3 해소). 합성 Message[]는 휘발(드라이버 입력용, 영속 안 됨).
- 토큰 상한·절삭은 P1 Ollama 컨텍스트 윈도우 대응으로 [미정 M4]. prompt 합성(라벨링·직렬화)은 드라이버 위임([222] §8).

## 5. 영속성 [결정]

> P0 = in-memory. room만 옵션 localStorage 스냅샷(whisper 제외). 재수화 시 in-flight `speaking` → `stopped` 강제(v1 `HIGH-3` 계승).

- **P0 기본 = in-memory**: `RoomSession` · `Whisper` 모두 메모리. 새로고침 시 소멸(솔로·로컬 웹앱, [221] §2). 영속은 **옵션**.
- **옵션 스냅샷(room만)**: 사용자가 켜면 매 턴 종료(`idle` 전이) 시 `RoomSession`을 `localStorage`에 직렬화. **whisper는 별도 보관이라 자동 제외**(§2 불변식 3).

```ts
const SNAP_KEY = 'aiSarangbang.room'

export function persistRoom(session: RoomSession): void {
  // RoomSession만 직렬화 — Whisper는 인자에 없으므로 구조적으로 새어나가지 않는다
  localStorage.setItem(SNAP_KEY, JSON.stringify(session))
}

export function loadRoom(): RoomSession | null {
  const raw = localStorage.getItem(SNAP_KEY); if (!raw) return null
  const s = JSON.parse(raw) as RoomSession
  // [중요] 재수화 시 in-flight 강제 종결 — v1 HIGH-3 계승
  for (const m of s.history) if (m.status === 'streaming') m.status = 'stopped'   // 끊긴 발언 보존·표식
  s.status = 'idle'                                                                // 진행 중 턴은 닫고 입력 대기로
  s.floorHolder = null                                                             // [M-4] idle 불변식 완결(발언자 표시 정리)
  return s
  // 주: coordinator 런타임 상태(queue/spokeThisTurn/turnState)는 미복원 — 진행 중이던 턴 폐기(의도). 새 인스턴스가 빈 상태로 시작.
}

export function clearRoom(): void { localStorage.removeItem(SNAP_KEY) }
```
