## 0. 한 줄 요약

진입 시 **로비**에서 로컬 Ollama 모델 목록(`/api/tags`)을 보여주고, AI별 **모델·이름·사고모드 선택 + 인원 가감** 후 입장한다. 설정은 **localStorage 영속**. 로비↔방 전환은 **세션 teardown(`dispose`)** 으로 안전하게.

## 1. 요구사항
- (핵심) 진입 시 로컬 모델 목록 → 사용자가 골라 대화.
- 반영(1~5): ①Ollama 미실행/0개 ②thinking 모델 ③영속화 ④인원 가변 ⑤이름 동기화.

## 2. 핵심 전제 — `/api/tags` (CLI 아님)  ⚠️ dev 한정 주의
브라우저는 `ollama list`(CLI)를 못 돌리지만 Ollama REST가 같은 목록을 준다:
```
GET /api/tags → { models:[{ name, model, size, details:{family, parameter_size, quantization_level} }] }
```
**[C1] `/ollama` proxy는 `vite dev` 전용**이다(`vite.config.ts`의 `server.proxy`). `vite build`/`vite preview`/prod에는 **적용 안 됨** → 로비의 로드 시 `/ollama/api/tags`가 404. 따라서:
- **`vite.config.ts`에 `preview: { proxy: { '/ollama': … } }` 추가**(dev+preview 모두 지원), 그리고
- `listModels`의 `baseUrl`을 설정 가능하게(폴백: `http://localhost:11434` 직접 — 단 CORS 확인 필요).
- 228은 "**로비/Ollama 연동은 dev·preview 한정**(prod 서빙 시 별도 proxy 필요)"을 명시한다. (기존 `/api/chat`도 동일 제약이었음.)

## 3. UX 흐름
```
진입 → 로비
        · listModels()  (실패 시 §4.8)
        · 영속 config 복원(검증)
        · AI 행: [모델 ▼][이름____][사고모드 ▣tri][✕]
        · [+ AI 추가]   [입장](AI≥1 게이트)
      → 입장: buildSession(config) → 방(기존 Room)
      → 방 [설정] → dispose() 후 로비 복귀
```

## 4. 설계

### 4.1 부트 재구성 — `App` + `buildSession` + **세션 teardown [C2]**
- `app/App.tsx`(신규): `phase:'lobby'|'room'`, `session` 보관.
- **StrictMode 안전**: 현재 `main.tsx`는 **모듈 top-level**에서 조립해 StrictMode 이중렌더가 무해했다. React 안으로 옮기므로 **lazy/`useRef` 가드로 `buildSession` 1회 보장**(이중 인스턴스화 방지).
- **[C2/M4] `Coordinator.dispose()` 신설** (core 변경 — §5): `this.current?.abort()` + `autoMode=false`·`autoLeft=0` + 진행 중 whisper/`interruptibleDelay` 타이머 정리. 로비↔방 **전환·재진입마다** 이전 세션 `dispose()` 후 store 리스너 해제.
- `buildSession(config): Session` — §4.4.

### 4.2 모델 목록 — `listModels()`  [L3 방어]
- `drivers/ollamaApi.ts`(신규): `GET ${baseUrl}/api/tags` → `models[].name`(+ `details?.parameter_size`는 **문자열**, `details` **부재 가능** → 옵셔널 가드). 실패 throw → §4.8.

### 4.3 설정 모델 — `RoomConfig`
```ts
interface AiSlot { id: string; name: string; model: string; think?: boolean } // think tri-state(§4.9)
interface RoomConfig { v: 1; ais: AiSlot[]; humanName?: string }              // v=스키마 버전(§4.7)
```

### 4.4 `buildSession` — **현 `main.tsx`(L17–52) 조립을 그대로 옮기고 `config`로만 파라미터화 [H1]**
누락 없이 전부 재현한다:
1. `setLocale(config.locale)` (i18n)
2. participants = `[{id:'me',name: config.humanName ?? t('participant.me'),kind:'human',seat:0}]` + `ais.map((a,i)=>({id:a.id,name:a.name,kind:'ai',seat:i+1}))` **[L2 humanName 적용]**
3. `store = createRoomStore(room)`
4. driver = `withHardTimeout( config.driver==='ollama' ? stripLeadingSelfLabel(createOllamaDriver({ model:(p)=>map[p.id].model, think:(p)=>map[p.id].think, baseUrl, system: ollamaSystem })) : createMockDriver(demoMockConfig(typingMs)), idleMs )` — **`stripLeadingSelfLabel`·`withHardTimeout`·드라이버별 `idleMs`(ollama 60_000 / mock 5_000) 모두 유지**
5. `new Coordinator(room, driver, store, { whisperTimeoutMs, contextLimit, autoDelayMs, autoMaxTurns, rng })` — **4개 opts 전부**

### 4.5 [4] 인원 가변  [M1 게이트]
- 로비에서 슬롯 추가/삭제. seat는 `buildSession`이 순서대로 부여.
- **AI ≥ 1 을 입장 버튼에서 하드 게이트**(권고 아님 — 0명이면 빈 턴). `coordinator`/`store`/`Roster`는 participants 기반이라 N≠3 무변경 수용(검증됨). `colors`는 4팔레트 순환(5+ 색 반복 — 의도).

### 4.6 [5] 이름 동기화  [M2 단일 규칙]
- `defaultNameForModel(model)` = **org(`/` 앞) 제거 → 태그(`:` 앞) 추출 → 첫 글자만 Title-case**(버전 접미는 **유지**). 예: `exaone3.5:7.8b`→`Exaone3.5`, `qwen3.5:4b`→`Qwen3.5`, `phi4-mini:latest`→`Phi4-mini`, `ingu627/exaone4.0:latest`→`Exaone4.0`.
  - **참고(구현 정합, v1.0 critic M-B)**: 브랜드 관용 대문자(`EXAONE` 등)는 단일 규칙으로 일반화 불가(같은 규칙이 `qwen3.5`의 버전을 보존해야 하므로) → 자동값은 위 일관 규칙을 따르고, 관용 표기는 **사용자가 이름칸에서 편집**. 데모 고정 참가자(`config.ts demoParticipants`)의 `EXAONE`은 별도 하드코딩(로비 비경유).
- 이름칸이 비었거나 자동값과 같으면 갱신, 사용자 입력은 보존.

### 4.7 [3] 영속화  [H3]
- `app/persist.ts`(신규): `localStorage['ai-sarangbang.config']`.
- 저장 blob에 **`v` 스키마 버전**. 로드: **`try/catch` JSON.parse**(손상값→null), **`v` 불일치→null(기본값)**, `typeof localStorage` 가드.
- **모델 검증**: 복원 모델을 현재 `/api/tags`와 **교집합** → 사라진 모델은 표시·대체 유도, 그대로 입장 시 첫 발언서 404→"(응답 없음)"(기존 안전망).

### 4.8 [1] Ollama 미실행 / 0개  [H2 mock 고정]
- `/api/tags` 실패 → "Ollama 미실행/접근 불가" + [다시 시도] + [Mock 데모로 시작].
- **[H2] Mock 데모는 `RoomConfig`를 우회**하고 **고정 `demoParticipants()`/`demoMockConfig()`**(= id `exaone/phi4mini/qwen3`)를 쓴다. (가변 id `ai-1…`는 mock 대사 키와 불일치 → 빈 방이 되므로 금지.)
- 모델 0개 → "받은 모델이 없습니다" + pull 안내(문구만, 명령 실행 안 함).

### 4.9 [2] thinking 모델  [M3 tri-state]
- `guessThinking(model)`: `/qwen3|exaone.*4|deepseek-r1|.*think|reason/i` → 사고모드 모델 추정.
- **think는 tri-state(`undefined`|`true`|`false`)** — `OllamaDriver`는 `think!==undefined`일 때만 전송(비-thinking 모델 보호 계약). UI 토글: 미플래그 모델은 **항상 `undefined`(미전송)**, 플래그 모델은 기본 `false`(끔) + on/off. **2-state 체크박스 금지**(undefined 표현 불가 → 비-thinking에 `think:false` 오전송).

### 신규 테스트
- `listModels`: 실제 응답 fixture 파싱 / HTTP에러→throw / 빈 models→[].
- `buildSession`: human seat0 + N AIs / 드라이버에 strip+hardTimeout+4 opts·config map 모델·think 주입 확인.
- `dispose()`: in-flight stream abort·이후 publish/타이머 0(fake timer).
- `persist`: round-trip / 손상값→null(무throw) / 버전불일치→null / 모델누락 검증.
- Mock 로비 경로: 고정 demo id로 MockDriver 실제 발화.
