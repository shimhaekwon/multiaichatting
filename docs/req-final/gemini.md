합의된 모든 사항(3사 연동, IndexedDB 사용, Manifest V3 기반 크롬 확장 프로그램 구조 등)과 기술 제약 조건을 반영하여, **현재 시점(2026년 5월)을 기준으로 최종 확정된 요구사항정의서(v2.0)**를 작성했습니다.

## [요구사항정의서 v2.0] 멀티 LLM 협업 시스템 (최종 확정안)

### 1. 개요 및 목적
* **시스템명:** 멀티 LLM 협업 시스템 (가칭)
* **목적:** 사용자가 데스크톱 크롬 브라우저에 로그인한 세션을 크롬 확장 프로그램(Manifest V3)을 통해 활용하여, 별도의 API 키 비용 없이 웹 UI 대시보드에서 멀티 LLM 간 라운드 방식의 동기화 협업을 수행할 수 있도록 지원하는 단일 사용자용 로컬 데이터 기반 시스템.

### 2. 기능 요구사항

| ID | 대분류 | 상세 요구사항 | 비고 |
| :--- | :--- | :--- | :--- |
| **REQ-01** | **아키텍처** | 크롬 확장 프로그램(Chrome Extension)과 대시보드 웹 UI가 결합된 하이브리드 구조로 구현한다. | |
| **REQ-02** | **인증 및 연동** | 사용자가 크롬 브라우저에서 로그인한 세션(ChatGPT, Claude, Gemini)을 확장 프로그램 권한으로 연동하며, 별도의 API Key 입력을 요구하지 않는다. | |
| **REQ-03** | **대상 LLM** | 초기 지원 대상 LLM은 **ChatGPT, Claude, Gemini** 3종으로 한정한다. | |
| **REQ-04** | **LLM 선택** | 연동된 LLM 목록을 화면에 표시하며, 사용자는 협업에 참여할 모델을 **최소 2개 이상** 선택(일부 또는 전체)해야 한다. | |
| **REQ-05** | | 사용자가 직전에 선택한 LLM 리스트 상태는 브라우저 세션 간에 **그대로 유지(저장)**되어야 한다. | |
| **REQ-06** | **운영 환경** | 본 시스템은 단일 사용자(혼자 사용)를 위한 시스템이며, 데스크톱(PC) 크롬 브라우저 환경만 지원한다. (모바일 지원 제외) | |
| **REQ-07** | **라운드 제어** | 협업 프로세스는 **최대 3 라운드**로 제한하여 진행한다. | |
| **REQ-08** | | 모든 선택된 LLM의 응답이 완료되어야 다음 라운드로 진행하는 **동기화 방식**을 채택한다. | |
| **REQ-09** | | 특정 LLM의 응답이 지연될 경우, **최대 5분(타임아웃)** 대기 후 해당 모델을 제외하고 다음 단계로 강제 진행한다. | |
| **REQ-10** | | 3 라운드 이전이라도 사용자가 만족할 경우 **단계 조기 종료**가 가능하다. | |
| **REQ-11** | | **[R1: 독립]** 타 LLM의 답변을 보지 못하는 Blind 상태에서 독립 응답을 수집한다. | |
| **REQ-12** | | **[R2: 검토]** R1에서 취합된 타 LLM의 **전체 텍스트(원문)**를 교차 전달하여 상호 검토하도록 프로세스를 진행한다. | |
| **REQ-13** | | **[R3: 최종]** 각 LLM의 최종 수정 및 정제된 답변을 도출한다. | |
| **REQ-14** | **사용자 권한** | 사용자는 각 라운드 단계별로 수동 개입(프롬프트 추가 입력 등)을 할 수 있다. | |
| **REQ-15** | | 사용자는 특정 LLM을 지정하여 1:1로 재질의를 할 수 있으며, 이 재질의 결과는 **다음 라운드 컨텍스트에 포함**되어 협업에 반영된다. | |
| **REQ-16** | | 시스템이 제안한 결과에 대한 최종 판단 및 결정은 사용자가 수행한다. | |
| **REQ-17** | **데이터 관리** | 대화 이력 및 시스템 데이터는 전량 사용자의 로컬 저장소인 **IndexedDB**에 저장하며, 별도의 암호화는 적용하지 않는다. | |
| **REQ-18** | | 사용자가 수동으로 저장하지 않아도 **각 라운드가 완료될 때마다 로컬에 자동 저장(Auto-save)**된다. | |
| **REQ-19** | **부가 기능** | 대화 이력을 검색하고 조회할 수 있는 **최소 기능 수준의 이력 조회 기능**을 제공한다. | |
| **REQ-20** | | 특정 LLM과 주고받은 전체 대화 내용을 타임라인 형태로 확인할 수 있는 팝업 뷰 기능을 제공한다. | |
| **REQ-21** | | 최종 협업 결과를 **'라운드별 헤더 + LLM별 응답 블록'** 구조를 가진 Markdown(.md) 파일로 로컬에 저장하는 기능을 제공한다. | |

### 3. 제약사항 및 사양 환경 (Constraints)
* **개발 표준:** 크롬 확장 프로그램 최신 표준인 **Manifest V3** 필수 준수.
* **대상 브라우저:** 크롬(Chrome) 최신 안정 버전(Stable Version) 기준 지원.
