## **1. 시스템 아키텍처**
### **1.1 실행 환경**
| **요소**               | **구현 방식**                                                                 |
|-------------------------|---------------------------------------------------------------------------------|
| **Frontend (Web UI)**   | React/Vue.js + Figma 디자인 (모바일 호환)                                   |
| **Backend (Extension)** | Chrome Extension (Manifest V3) + Node.js/Express (WebSocket/HTTP)          |
| **데이터 저장**        | Chrome Storage API (IndexedDB) + 로컬 파일 (MD 파일)                        |
| **LLM 접근**            | Extension backend → Web API 호출 (OpenAI, Claude, Gemini 등)                |

---

## **2. 주요 기능 요구사항**
### **2.1 LLM 지원 및 선택**
| **항목**               | **설정**                                                                 |
|-------------------------|--------------------------------------------------------------------------|
| **지원 LLM**            | **Claude (Bedrock), ChatGPT (OpenAI), Gemini (Google AI)**               |
| **최소 선택 수**      | **2개 이상** (사용자가 2개 이상 선택한 후 진행)                         |
| **선택 상태 유지**    | ✅ **유지** (Chrome Storage API로 세션 저장)                            |
| **LLM 목록 동기화**    | ✅ **사용자 선택 → Web UI에 반영**                                        |

### **2.2 라운드 기반 협업**
| **라운드** | **요구사항**                                                                 |
|-------------|------------------------------------------------------------------------------|
| **R1 (Blind Response)** | - LLM1/2/3 → 각 독립 답변 받기.                                           |
|              | - 사용자가 답변 수정/삭제 가능.                                           |
| **R2 (Peer Review)**   | - 각 LLM → **전체 텍스트 리뷰** 전달.                                       |
|              | - 사용자가 리뷰 수정/추가 가능.                                           |
| **R3 (Final Decision)**| - 사용자가 최종 답변 저장 (MD 파일).                                     |
| **조기 종료**          | ✅ **가능** (사용자가 라운드 중 종료 가능)                                 |

### **2.3 데이터 저장 및 협업**
| **항목**               | **설정**                                                                 |
|-------------------------|--------------------------------------------------------------------------|
| **MD 파일 포맷**       | **라운드별 헤더 + LLM별 응답 블록** (예: Markdown)                      |
| **자동 저장**           | ✅ **라운드 완료 시마다** (`user_data/round_<R1/R2/R3>/output.md`)        |
| **저장 방식**          | ✅ **IndexedDB (Chrome Storage API)** + 로컬 파일 (MD 파일)             |
| **이력 조회**           | ✅ **포함** (사용자 기록 조회 가능)                                       |
