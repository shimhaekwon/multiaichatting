# 멀티 LLM 협업 시스템 요구사항정의서 v1.0

## 1. 개요

### 1.1 목적
다수의 LLM이 동일 주제에 대해 독립 응답, 상호 검토, 최종 정리를 수행할 수 있는 멀티 LLM 협업 시스템 제공.

---

## 1.2 목표

- 브라우저 로그인 세션 기반 LLM 활용
- API Key 없는 협업 환경 제공
- 다중 LLM 병렬 협업 지원
- 사용자 중심 최종 판단 구조 제공
- 대화 이력 및 Markdown 저장 지원

---

# 2. 시스템 범위

## 2.1 포함 범위

- Chrome Extension 기반 LLM 제어
- Web UI 기반 협업 orchestration
- 최대 3 Round 협업 프로세스
- LLM별 대화 조회
- Markdown 저장
- 로컬 데이터 저장
- 최소 이력 조회 기능

---

## 2.2 제외 범위

- 서버 기반 중앙 저장소
- 멀티 사용자 협업
- 모바일 앱
- API Key 기반 연결
- 음성/영상 처리
- 클라우드 동기화

---

## 3.2 실행 환경

### FR-001
시스템은 Chrome 브라우저 기반으로 동작해야 한다.

---

# 4. LLM 연결

## 4.1 지원 대상

### FR-010
초기 지원 대상 LLM은 다음과 같다.

- ["company","Anthropic","Claude"]
- ["company","OpenAI","ChatGPT"]
- ["company","Google","Gemini"]

---

## 4.2 인증 방식

### FR-011
사용자는 각 LLM 서비스에 사전 로그인된 상태여야 한다.

### FR-012
시스템은 기존 브라우저 로그인 세션을 재사용해야 한다.

### FR-013
시스템은 API Key 입력 없이 동작해야 한다.

---

## 4.3 LLM 선택

### FR-014
사용자는 사용 가능한 LLM 목록을 조회 가능해야 한다.

### FR-015
사용자는:
- 단일 선택
- 복수 선택
- 전체 선택
가능해야 한다.

### FR-016
최소 2개 이상의 LLM 선택이 가능해야 한다.

### FR-017
선택 상태는 세션 간 유지되어야 한다.

---

# 5. 협업 라운드

## 5.1 기본 규칙

### FR-020
최대 3개의 Round를 지원해야 한다.

### FR-021
각 Round는 모든 대상 LLM 응답 완료 후 다음 단계로 진행해야 한다.

### FR-022
사용자는 Round 조기 종료 가능해야 한다.

---

## 5.2 Round 정의

### FR-023
Round 1은 blind 독립 응답 단계여야 한다.

### FR-024
Round 2는 타 LLM 응답 검토 단계여야 한다.

### FR-025
Round 3은 최종 답변 단계여야 한다.

---

## 5.3 컨텍스트 전달

### FR-026
Round 2에서는 타 LLM 응답 전체 텍스트를 전달해야 한다.

### FR-027
사용자 재질의 결과는 다음 Round 컨텍스트에 포함되어야 한다.

---

## 5.4 동기화 및 타임아웃

### FR-028
Round 동기화 기본 timeout은 120초여야 한다.

### FR-029
사용자는 timeout을 최대 300초까지 변경 가능해야 한다.

---

# 6. 사용자 개입

### FR-030
사용자는 각 Round 단계별 개입 가능해야 한다.

### FR-031
사용자는 특정 LLM과 1:1 추가 질의 가능해야 한다.

### FR-032
사용자는 각 Round 결과를 검토 가능해야 한다.

### FR-033
최종 판단 권한은 사용자에게 있어야 한다.

---

# 7. 대화 및 저장

## 7.1 대화 조회

### FR-040
LLM별 전체 대화 내용을 조회 가능해야 한다.

### FR-041
대화 내용은 팝업 뷰 형태 제공 가능해야 한다.

### FR-042
최소 수준의 이력 검색/조회 기능을 제공해야 한다.

---

## 7.2 저장 정책

### FR-043
대화 데이터는 로컬에 저장되어야 한다.

### FR-044
로컬 저장 방식은 IndexedDB를 사용해야 한다.

### FR-045
응답 수신 즉시 incremental 저장해야 한다.

### FR-046
Round 완료 시 자동 저장 상태가 보장되어야 한다.

---

## 7.3 Markdown 저장

### FR-047
대화 내용을 Markdown(md) 파일로 저장 가능해야 한다.

### FR-048
Markdown 저장 포맷은 다음 구조를 사용해야 한다.

```markdown
# Session Title

## Round 1

### Claude
...

### ChatGPT
...

### Gemini
...

## Round 2
...
```

---

## 7.4 보안 정책

### FR-049
초기 MVP에서는 로컬 데이터 암호화를 적용하지 않는다.

---
