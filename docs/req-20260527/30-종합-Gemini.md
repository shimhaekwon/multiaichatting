# [종합] LLM 다중 라운드 토론 시스템 가이드라인 (v2.0)

비전문가 및 협업 부서원도 시스템의 모든 동작 프로세스와 화면 구성을 직관적으로 이해하고 검증할 수 있도록 작성된 종합 안내서입니다. 개발 원칙에 따라 임의의 추정 기능은 배제되었습니다.

## 1. 시스템 개요 및 범위
이 시스템은 사용자가 크롬 브라우저에서 실행하는 프로그램(확장 프로그램 Manifest V3)입니다. 별도의 서버나 유료 API 키 없이, 사용자가 기존에 로그인해 둔 ChatGPT, Claude, Gemini 계정을 그대로 활용합니다. 데이터는 유출 없이 사용자의 PC(IndexedDB)에만 저장됩니다.

## 2. 상세 기능 및 사용자 시나리오별 동작 방식

### 2.1 프로그램 시작 및 LLM 연결 (1단계)
* **자동 탐색:** 프로그램을 켜면 Claude, ChatGPT, Gemini의 로그인 상태를 자동으로 확인합니다.
* **로그인 실패 시 대응:**
  1. 로그인이 안 되어 있다면 "크롬에서 해당 서비스에 로그인해 주세요"라는 안내 창을 띄웁니다.
  2. 그래도 연결 가능한 LLM이 없다면, "기본(Default) LLM 계정을 생성해 주세요"라고 친절하게 안내합니다.
* **구현 기준:** 기존 `E:\\workspace\\chathub` 코드를 기반으로 이 탐색 레이아웃을 개선하여 구현합니다.

### 2.2 LLM 선택 및 채팅 준비 (2단계)
* **선택 화면:** 연결된 LLM 목록이 화면에 표시되며, 사용자는 체크박스를 통해 토론에 참여시킬 AI들을 선택합니다.
* **제약 조건:** 최소 2개 이상의 AI를 선택해야만 토론을 시작할 수 있습니다. (AI 1개 단독 실행은 불가능합니다.)
* **기억 기능:** 한 번 선택해 둔 AI 목록은 프로그램을 껐다 켜도 그대로 유지됩니다.

### 2.3 토론 진행 규칙 (3단계: 라운드 규칙)
토론은 기본적으로 3번의 라운드(회차) 동안 진행되며, 최대 9라운드까지 확장할 수 있습니다. 모든 AI가 답변을 끝내야만 다음 라운드로 넘어갑니다.
* **1라운드 (R1 - 블라인드):** AI들은 서로의 존재를 모른 채, 사용자가 처음 입력한 질문에 대해 각자 독립적으로 답변합니다.
* **2라운드 (R2 - 교차 검토):** AI들에게 "방금 전 1라운드에서 나온 다른 AI들의 답변 전체 텍스트"를 복사해서 전달합니다. 각 AI는 타사 AI의 의견을 검토한 후 자신의 의견을 다시 제시합니다.
* **3라운드 (R3 - 최종 답변):** 이전 라운드 의견들을 종합하여 최종 결론 답변을 도출합니다.
* **라운드 제어:** * 사용자는 토론 도중 언제라도 "조기 종료" 버튼을 눌러 끝낼 수 있습니다.
  * 지정된 회차가 끝나면 자동 종료되지만, 필요에 따라 회차를 "연장"할 수 있습니다. 회차는 채팅 중에 1회씩 늘리거나 줄일 수 있으며, 마지막 라운드 안에서도 회차 증설이 가능합니다.

### 2.4 사용자 개입 및 돌발 행동 통제 (4단계)
* **입력 통제:** 질문을 던진 후(Request), 선택된 모든 AI의 답변이 화면에 다 나올 때까지(Response 완료 전) 사용자는 채팅창을 조작하거나 개입할 수 없습니다.
* **라운드 사이 개입:** 라운드가 끝난 대기 상태에서는 아래의 행동이 가능합니다.
  * 토론 도중 특정 AI를 추가하거나 제외하기
  * 특정 AI 하나와 1:1로 다시 질문하기 (이 대화 내용은 다음 라운드 진행 시 전체 컨텍스트에 포함되어 반영됩니다.)
* **사전 입력 기능:** AI들이 열심히 답변을 작성하고 있는 동안, 사용자는 다음 라운드에 보낼 질문을 입력창에 미리 타이핑해 둘 수 있습니다. (단, 전송 버튼은 답변이 끝날 때까지 작동하지 않습니다.)
* **주의(구현 금지):** 먼저 답변을 끝낸 AI와 몰래 개별 채팅을 나누는 기능은 MVP(초기 버전) 이후로 미루며, 본 버전에서는 구현하지 않습니다.

### 2.5 타임아웃 및 재시도 규칙 (5단계)
* **대기 시간 제한:** 기본적으로 AI의 답변을 기다리는 시간은 5분입니다. 사용자가 분 단위로 직접 수정할 수 있습니다.
* **재시도 횟수:** AI가 응답에 실패할 경우 3회에서 9회 사이로 사용자가 지정한 만큼 자동으로 다시 시도합니다.
* **AI 전달 정보:** 질문을 보낼 때 시스템은 내부적으로 `[제한시간] + [현재 라운드 번호 / 총 제한 라운드 번호]` 정보를 질문과 함께 AI에게 전달하여 시간 관리를 인지시킵니다.

### 2.6 데이터 저장 및 이력 조회 (6단계)
* **자동 저장:** 각 라운드가 끝날 때마다 사용자의 브라우저 내부 저장소(IndexedDB)에 대화 내용이 실시간 자동 저장됩니다. 외부 서버로는 절대 전송되지 않습니다.
* **검색 기능:** 저장된 과거 대화들은 `년월일시`, `참여한 LLM 이름`, `라운드 번호` 항목을 통해 간편하게 검색할 수 있습니다.
* **파일 내보내기 (MD 저장):** 토론이 끝나면 전체 토론 내용이 합쳐진 Markdown(.md) 파일과, 각 AI별 답변만 따로 모은 개별 Markdown 파일을 PC에 저장할 수 있습니다.

### 2.7 응답 형식 제약 (7단계)
* **구조화된 출력:** AI들이 중구난방으로 답변하지 않도록 출력 형식을 강제 지정합니다.
* **글자수 제한:** 답변 내용이 너무 길어지지 않도록 응답 컨텍스트는 1,500 토큰 이내로 제한합니다. (개발 시 상세 설계 검토가 필요한 항목입니다.)
* **간소화:** AI 특유의 장황하고 불필요한 서론/결론 미사여구를 회피하도록 조치합니다.

### 2.8 환경 및 기술 제약 사항 (8단계)
* **웹 브라우저:** 구글 크롬(Chrome) 데스크톱 버전 전용입니다. 모바일 브라우저는 지원하지 않습니다.
* **확장 프로그램 규격:** 크롬의 최신 보안 표준인 Manifest V3 규격을 따릅니다.
* **사용 인원:** PC당 단일 사용자만 사용하는 로컬 프로그램입니다.
* **보안 및 서버:** 외부 서버 저장은 없으며, API 키를 쓰지 않고 로컬에만 저장되므로 별도의 데이터 암호화는 적용되지 않습니다.

## 3. 개발 행동 원칙
1. **Step by Step:** 단계별로 차근차근 개발을 진행합니다.
2. **No Guess:** 요구사항 정의서에 기재되지 않은 내용이나 사용자의 의도를 임의로 추측하여 기능을 추가하거나 구현하지 않습니다.
3. **4대 원칙:** 빠짐없이, 틀림없이, 다름없이, 더함없이 요구사항 그대로 명확하게 구현합니다.
