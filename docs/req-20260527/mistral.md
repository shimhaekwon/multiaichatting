### **10-기본-고객요구.md**
**목적**: 사용자 및 비개발자 이해를 위한 간결한 요약
**내용**:
- **시스템 이름**: ChatHub (크롬 확장 플러그인으로 실행)
- **기능 요약**:
  - 3종 LLM(Claude, ChatGPT, Gemini)를 로그인 세션 기반으로 탐색 (API 키 불필요).
  - 사용자가 **2개 이상 LLM을 선택** 후, **3~9회 라운드**로 구조화된 채팅 진행.
    - R1: 각 LLM 독립적으로 답변 (Blind).
    - R2: 다른 LLM의 R1 답변을 기반으로 의견 제시.
    - R3: 최종 답변.
  - **자동 저장** (IndexedDB) + 사용자 설정 가능 (타임아웃, 재시도 횟수).
  - **MD 저장** 기능과 이력 조회 기능 제공.
- **제약**:
  - Chrome 전용 (Manifest V3), 단일 사용자, 로컬 저장만 지원.

### **20-핵심-고객요구.md**
**목적**: 전문가용 디테일화된 요구사항
**내용**:
#### **1. LLM 접속 & 탐색**
- **접속**:
  - 크롬 브라우저로 로그인 세션을 기반으로 **3종 LLM(Claude, ChatGPT, Gemini) 자동 탐색** (API 키 미사용).
  - 탐색 실패 시:
    1. 크롬 로그인 유도 화면 표시.
    2. 유효 LLM 목록 없으면 **default 계정 생성 요청 안내** (사용자 승인 필요).
- **UI**:
  - ChatHub 플러그인 UI를 기반으로 **LLM 목록 선택 UI** 제공.
  - **세션 간 LLM 선택 상태 유지** (localStorage/IndexedDB).
- **타임아웃**:
  - 기본 5분 (분 단위 조정 가능), 재시도 횟수 3~9회 (사용자 설정).

#### **2. 라운드 규칙**
- **회차**:
  - 기본 3회, 최대 9회 (채팅 중 1회씩 증감 가능, 마지막 회차도 가능).
  - **동기화**: 모든 LLM의 응답 완료 후 다음 라운드 진행.
- **R1~R3**:
  - R1: **Blind 응답** (타 LLM 참조 미허용).
  - R2: **타 LLM의 R1 응답 전체 텍스트 검토 후 의견 제시** (개별 LLM별로).
  - R3: **최종 답변** (사용자 최종 승인 필요).
- **연장**:
  - 지정 회차 완료 후 **사용자 승인으로 연장 가능**.

#### **3. 사용자 개입**
- **제한**:
  - Request 후 **Response 완료까지 개입 불가**.
  - **재질의**:
    - **단독 LLM 재질의**: 라운드 사이에만 가능.
    - **타 LLM 재질의**: R3 이후에만 가능 (제외).
  - **재질의 결과**: 다음 라운드 컨텍스트에 포함.
- **사전 입력**:
  - 다음 회차 Request 미리 입력 가능 (Send 불가).

#### **4. 저장 & 이력**
- **자동 저장**:
  - IndexedDB로 **라운드별 저장** (회차 완료 시 자동).
- **MD 저장**:
  - 전체 내용 + **LLM별 개별 Markdown 파일** 저장.
- **이력 조회**:
  - **년월일시/LLM/라운드**로 검색 가능 (IndexedDB + 로컬 파일).

#### **5. 응답 형식**
- **강제 구조화**: JSON/XML 기반 (설계 검토 필요).
- **Token 제한**: 1500-token 제한 (LLM 응답 컨텍스트).
- **간소화**: 장황한 출력 회피 (LLM 응답 필터링).

#### **6. 기술 제약**
- **브라우저**: Chrome 100+ (Manifest V3).
- **OS**: Windows/macOS/Linux (모바일 미지원).
- **암호화**: 미적용 (로컬 데이터만).
- **서버 저장**: 미지원 (로컬 전용).

#### **9. 개발 원칙**
- **step by step**: 요구사항 순서대로 구현.
- **no guess**: 합의되지 않은 내용은 **추정 금지**.
- **재사용**: 설계 템플릿 (e.g., 라운드 로직, 저장 시스템)을 재활용.

### **30-종합-고객요구.md**
**목적**: 사용자 및 개발자 모두에게 이해 가능한 통합 문서
**내용**:

---
#### **시스템 개요**
**이름**: **ChatHub (LLM Multi-Agent Chat Plugin)**
**타겟**: 단일 사용자 (Chrome 브라우저, Windows/macOS/Linux)
**목적**: **3종 LLM(Claude, ChatGPT, Gemini)을 기반으로 구조화된 다중 LLM 채팅**를 지원.
**기능**:
1. **LLM 탐색 & 선택**
   - 로그인 세션을 통해 **자동 LLM 탐색** (API 키 불필요).
   - 사용자가 **2개 이상 LLM을 선택** 후 **3~9회 라운드**로 채팅 진행.
   - **도메인별 LLM 우선순위** (사용자 설정 가능).

2. **채팅 라운드 규칙**
   - **R1**: 각 LLM 독립적으로 **Blind 답변** (타 LLM 참조 미허용).
   - **R2**: 타 LLM의 R1 답변을 기반으로 **개별 의견 제시**.
   - **R3**: 최종 답변 (사용자 승인 필요).
   - **조기 종료**: 사용자가 **사전 입력 Request**로 라운드 중 종료 가능.
   - **연장**: 지정 회차 완료 후 **사용자 승인으로 추가 회차 가능**.

3. **타임아웃 & 재시도**
   - 기본 5분 (분 단위 조정), 재시도 횟수 3~9회 (사용자 설정).
   - **LLM 전달 시**: "제한시간 + 현재차수/제한차수" 전달.

4. **저장 & 이력 관리**
   - **자동 저장**: IndexedDB로 **라운드별 저장** (회차 완료 시).
   - **MD 저장**: 전체 내용 + **LLM별 개별 Markdown 파일** (e.g., `chat_20260527_R1_claude.md`).
   - **이력 조회**: **년월일시/LLM/라운드**로 검색 가능 (예: "2026-05-27 / ChatGPT / R2").

5. **응답 제어**
   - **강제 구조화**: JSON/XML 기반 (설계 검토 필요).
   - **Token 제한**: 1500-token 제한 (LLM 응답 컨텍스트).
   - **간소화**: 장황한 출력 회피 (LLM 응답 필터링).

6. **개발 환경**
   - **브라우저**: Chrome 100+ (Manifest V3).
   - **OS**: Windows/macOS/Linux (모바일 미지원).
   - **데이터**: 로컬 IndexedDB + Markdown 파일 (암호화 미적용).

7. **사용자 경험**
   - **로그인 유도**: 크롬 브라우저에서 **사용자 계정 선택** 화면 제공.
   - **LLM 선택 UI**: 체크박스 기반 (사용자 일부/전체 선택).
   - **재질의 UI**: 라운드 사이에만 **단독 LLM 재질의** 가능 (R3 이후에만 타 LLM 재질의).
   - **사전 입력**: 다음 회차 Request 미리 입력 가능 (Send 불가).

8. **예외 처리**
   - **LLM 탐색 실패**: default 계정 생성 요청 안내.
   - **타임아웃**: 재시도 횟수 초과 시 "LLM 응답 실패" 알림.
   - **최종 판단**: 사용자 **마지막 승인**로 시스템 종료.
