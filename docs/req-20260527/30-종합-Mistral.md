#### **시스템 개요**
**이름**: **ChatHub (LLM Multi-Agent Chat Plugin)**
**타겟**: 단일 사용자 (Chrome 브라우저, Windows/macOS/Linux)
**목적**: **3종 LLM(Claude, ChatGPT, Gemini)을 기반으로 구조화된 다중 LLM 채팅**를 지원.
**기능**:
1. **LLM 탐색 & 선택**
   - 로그인 세션을 통해 **자동 LLM 탐색** (API 키 불필요).
   - 사용자가 **2개 이상 LLM을 선택** 후 **3~9회 라운드**로 채팅 진행.
   - **도메인별 LLM 우선순위** (사용자 설정 가능).

2. **채팅 라운드 규칙**
   - **R1**: 각 LLM 독립적으로 **Blind 답변** (타 LLM 참조 미허용).
   - **R2**: 타 LLM의 R1 답변을 기반으로 **개별 의견 제시**.
   - **R3**: 최종 답변 (사용자 승인 필요).
   - **조기 종료**: 사용자가 **사전 입력 Request**로 라운드 중 종료 가능.
   - **연장**: 지정 회차 완료 후 **사용자 승인으로 추가 회차 가능**.

3. **타임아웃 & 재시도**
   - 기본 5분 (분 단위 조정), 재시도 횟수 3~9회 (사용자 설정).
   - **LLM 전달 시**: "제한시간 + 현재차수/제한차수" 전달.

4. **저장 & 이력 관리**
   - **자동 저장**: IndexedDB로 **라운드별 저장** (회차 완료 시).
   - **MD 저장**: 전체 내용 + **LLM별 개별 Markdown 파일** (e.g., `chat_20260527_R1_claude.md`).
   - **이력 조회**: **년월일시/LLM/라운드**로 검색 가능 (예: "2026-05-27 / ChatGPT / R2").

5. **응답 제어**
   - **강제 구조화**: JSON/XML 기반 (설계 검토 필요).
   - **Token 제한**: 1500-token 제한 (LLM 응답 컨텍스트).
   - **간소화**: 장황한 출력 회피 (LLM 응답 필터링).

6. **개발 환경**
   - **브라우저**: Chrome 100+ (Manifest V3).
   - **OS**: Windows/macOS/Linux (모바일 미지원).
   - **데이터**: 로컬 IndexedDB + Markdown 파일 (암호화 미적용).

7. **사용자 경험**
   - **로그인 유도**: 크롬 브라우저에서 **사용자 계정 선택** 화면 제공.
   - **LLM 선택 UI**: 체크박스 기반 (사용자 일부/전체 선택).
   - **재질의 UI**: 라운드 사이에만 **단독 LLM 재질의** 가능 (R3 이후에만 타 LLM 재질의).
   - **사전 입력**: 다음 회차 Request 미리 입력 가능 (Send 불가).

8. **예외 처리**
   - **LLM 탐색 실패**: default 계정 생성 요청 안내.
   - **타임아웃**: 재시도 횟수 초과 시 "LLM 응답 실패" 알림.
   - **최종 판단**: 사용자 **마지막 승인**로 시스템 종료.

---
**추가 명세**:
- **ChatHub 플러그인 UI**: [ChatHub GitHub](https://github.com/example/chathub) 참조 (UI wireframe).
- **LLM API 제한**: 각 LLM의 Token 제한에 맞춰 응답 필터링 (예: ChatGPT 4096-token).
- **재사용 설계**: 라운드 로직, 저장 시스템, UI 템플릿을 **모듈화** (e.g., `RoundManager`, `StorageService`).
