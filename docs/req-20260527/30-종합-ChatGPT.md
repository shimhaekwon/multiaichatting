# Multi-LLM Cowork System — 종합 고객 요구사항 v2.0

# 1. 프로젝트 개요

본 프로젝트는 Chrome Extension 기반의
다중 LLM 협업 시스템 구축을 목표로 한다.

사용자는:
- ChatGPT
- Claude
- Gemini

를 동시에 선택하여
라운드 기반 협업형 질의응답을 수행할 수 있어야 한다.

본 시스템은:
- API Key를 사용하지 않으며
- 실제 Chrome 로그인 세션을 활용하고
- 로컬 환경에서만 동작해야 한다.

---

# 2. 목표 기능

## 사용자가 할 수 있어야 하는 것

### 2.1 여러 LLM 동시 실행
사용자는 여러 LLM을 동시에 선택하여
동일 질문을 병렬 전달할 수 있어야 한다.

최소 선택 개수:
- 2개 이상

---

### 2.2 라운드 기반 협업
시스템은 여러 회차를 통해
LLM 간 의견 검토 및 개선을 수행해야 한다.

기본 흐름:

1. 독립 응답
2. 상호 검토
3. 최종 정리

---

### 2.3 응답 비교
사용자는:
- LLM별 응답
- 라운드별 변화
- 최종 결과

를 쉽게 비교할 수 있어야 한다.

# 5. 라운드 시스템

# 5.1 기본 정책

기본:
- 3회

최대:
- 9회

사용자는:
- 1회 단위 증가/감소 가능

마지막 회차에서도:
- 추가 연장 가능

---

# 5.2 Round 1

목적:
- 독립 사고 확보

전달 데이터:
- 사용자 질문만 전달

금지:
- 타 LLM 응답 공유 금지

---

# 5.3 Round 2

목적:
- 상호 검토

전달 데이터:
- 타 LLM의 Round1 전체 응답

각 LLM은:
- 장점
- 오류
- 누락
- 보완점

을 검토해야 한다.

---

# 5.4 Round 3 이상

목적:
- 수렴 및 최종 정리

전달 데이터:
- 이전 회차 결과

각 LLM은:
- 통합 의견
- 최종 권고안
- 수정 결과

를 제시해야 한다.

---

# 6. 동기화 정책

다음 라운드는:
모든 선택 LLM 응답 완료 후 시작해야 한다.

예외:
- timeout 발생
- retry 초과
- 사용자 종료
