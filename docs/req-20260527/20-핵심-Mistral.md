#### **1. LLM 접속 & 탐색**
- **접속**:
  - 크롬 브라우저로 로그인 세션을 기반으로 **3종 LLM(Claude, ChatGPT, Gemini) 자동 탐색** (API 키 미사용).
  - 탐색 실패 시:
    1. 크롬 로그인 유도 화면 표시.
    2. 유효 LLM 목록 없으면 **default 계정 생성 요청 안내** (사용자 승인 필요).
- **UI**:
  - ChatHub 플러그인 UI를 기반으로 **LLM 목록 선택 UI** 제공.
  - **세션 간 LLM 선택 상태 유지** (localStorage/IndexedDB).
- **타임아웃**:
  - 기본 5분 (분 단위 조정 가능), 재시도 횟수 3~9회 (사용자 설정).

#### **2. 라운드 규칙**
- **회차**:
  - 기본 3회, 최대 9회 (채팅 중 1회씩 증감 가능, 마지막 회차도 가능).
  - **동기화**: 모든 LLM의 응답 완료 후 다음 라운드 진행.
- **R1~R3**:
  - R1: **Blind 응답** (타 LLM 참조 미허용).
  - R2: **타 LLM의 R1 응답 전체 텍스트 검토 후 의견 제시** (개별 LLM별로).
  - R3: **최종 답변** (사용자 최종 승인 필요).
- **연장**:
  - 지정 회차 완료 후 **사용자 승인으로 연장 가능**.

#### **3. 사용자 개입**
- **제한**:
  - Request 후 **Response 완료까지 개입 불가**.
  - **재질의**:
    - **단독 LLM 재질의**: 라운드 사이에만 가능.
    - **타 LLM 재질의**: R3 이후에만 가능 (제외).
  - **재질의 결과**: 다음 라운드 컨텍스트에 포함.
- **사전 입력**:
  - 다음 회차 Request 미리 입력 가능 (Send 불가).

#### **4. 저장 & 이력**
- **자동 저장**:
  - IndexedDB로 **라운드별 저장** (회차 완료 시 자동).
- **MD 저장**:
  - 전체 내용 + **LLM별 개별 Markdown 파일** 저장.
- **이력 조회**:
  - **년월일시/LLM/라운드**로 검색 가능 (IndexedDB + 로컬 파일).

#### **5. 응답 형식**
- **강제 구조화**: JSON/XML 기반 (설계 검토 필요).
- **Token 제한**: 1500-token 제한 (LLM 응답 컨텍스트).
- **간소화**: 장황한 출력 회피 (LLM 응답 필터링).

#### **6. 기술 제약**
- **브라우저**: Chrome 100+ (Manifest V3).
- **OS**: Windows/macOS/Linux (모바일 미지원).
- **암호화**: 미적용 (로컬 데이터만).
- **서버 저장**: 미지원 (로컬 전용).

#### **9. 개발 원칙**
- **step by step**: 요구사항 순서대로 구현.
- **no guess**: 합의되지 않은 내용은 **추정 금지**.
- **재사용**: 설계 템플릿 (e.g., 라운드 로직, 저장 시스템)을 재활용.
