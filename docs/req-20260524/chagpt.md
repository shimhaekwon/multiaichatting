# 멀티 LLM 협업 시스템 요구사항정의서 v0.1

## 1. 개요

### 1.1 목적
다수의 LLM이 동일 주제에 대해 협업·검토·최종 정리를 수행할 수 있는 브라우저 기반 협업 시스템 제공.

### 1.2 범위
- 브라우저 로그인 세션 기반 LLM 연동
- 다중 LLM 선택 및 병렬 질의
- 최대 3라운드 협업 프로세스
- 사용자 개입 및 최종 판단 지원
- 대화 이력 조회 및 MD 저장

---

# 2. 기능 요구사항

## 2.1 LLM 연결

### FR-001
시스템은 브라우저 로그인 세션을 활용하여 LLM에 접속해야 한다.

### FR-002
API Key 입력 없이 사용 가능해야 한다.

### FR-003
접속 가능한 LLM 목록을 표시해야 한다.

### FR-004
사용자는:
- 단일 LLM 선택
- 복수 LLM 선택
- 전체 선택
가능해야 한다.

## 2.2 협업 라운드

### FR-010
최대 3개 라운드를 지원해야 한다.

### FR-011
모든 LLM 응답 완료 후 다음 라운드로 진행해야 한다.
(동기화 방식)

### FR-012
Round 1은 각 LLM의 blind 독립 응답이어야 한다.

### FR-013
Round 2는 타 LLM 응답 검토 단계여야 한다.

### FR-014
Round 3은 최종 답변 단계여야 한다.

### FR-015
사용자는 라운드 진행 전/중 개입 가능해야 한다.

- 응답 timeout 정책

### FR-021

### FR-022
최종 판단 권한은 사용자에게 있어야 한다.
