## **1. 개요**
### **1.1 목적**
멀티 AI 모델(LLM)을 협업하여 문제해결, 지식 생성, 또는 복잡한 작업 수행을 지원하는 시스템을 구현합니다.
- **목적달성**:
  - 사용자가 **브라우저 세션만**으로 LLM을 접근하고 협업할 수 있도록 지원합니다.
  - **동기화된 라운드 기반 협업**을 통해 최종결과를 제출할 수 있는 플랫폼을 제공합니다.

---

## **2. 시스템 요구사항**

### **2.1 기능 요구사항**

#### **2.1.1 LLM 접속 및 관리**
- **접속 방식**:
  - 브라우저 세션을 기반으로 **API 키 없이** LLM에 접근할 수 있어야 합니다.
  - 사용자가 로그인한 세션을 저장/재사용할 수 있어야 합니다.
- **접속 가능한 LLM 목록**:
  - 시스템이 **접근 가능한 LLM 목록**을 제공합니다.
  - 사용자는 **부분/전체 선택**을 가능하게 합니다.
  - 선택된 LLM별 **접속 상태**와 **사용 가능 여부**를 실시간으로 표시합니다.

#### **2.1.2 라운드 기반 협업**
- **라운드 규칙**:
  - **최대 3 라운드** 지원 (사용자가 추가 라운드 요청 시 **TBD**로 처리).
  - **동기화 방식**: 모든 LLM이 응답 완료 후 다음 라운드 진행 (동기화 완료 시 다음 라운드 시작).
  - **라운드별 역할**:
    - **R1 (Blind Independent Response)**:
      - 각 LLM이 **독립적으로** 문제/작업에 대해 답변을 생성합니다.
      - 사용자는 **각 라운드별 개입 가능**합니다 (예: 특정 LLM 답변 수정/삭제).
    - **R2 (Peer Review)**:
      - 각 LLM은 **R1 답변을 검토/리뷰**합니다.
      - 사용자는 **리뷰 결과 수정/추가** 가능합니다.
    - **R3 (Final Decision)**:
      - 사용자가 **최종 답변을 결정**합니다 (LLM별 가중치 적용 가능).
- **LLM 재질의**:
  - 사용자는 **특정 LLM 1:1 재질의**를 지원합니다 (예: "LLM1의 답변을 수정할 수 있도록").
  - 재질의 내용은 **사용자 정의**로 저장 가능합니다.

#### **2.1.3 사용자 권한 및 제어**
- **개인화 가능 기능**:
  - **LLM별 대화 뷰** (전체 대화 내용 조회 가능).
  - **MD 파일 저장** (협업 결과 저장/다운로드).
- **사용자 권한**:
  - **각 라운드별 개입** (예: R1에서 답변 수정, R2에서 리뷰 수정).
  - **LLM 재질의** (특정 LLM 답변 재작성).
  - **최종 판단** (사용자가 최종 답변을 결정).

#### **2.1.4 산출물 및 저장**
- **협업 결과 저장**:
  - **MD 파일 형식**으로 협업 결과 저장/다운로드 가능.
  - **사용자별 저장 폴더**로 구분 (예: `user_<ID>/round_<R1/R2/R3>/output.md`).
- **로그 저장**:
  - 사용자 세션, 라운드 진행, 최종 결정 등 **시스템 로그** 저장 (안전성 및audit 용도).

## **4. 데이터 흐름**
### **4.1 사용자 데이터 흐름**
```
[사용자] → (로그인) → [세션 저장] → [LLM 목록 선택] → [R1: Blind Response] → [사용자 개입] → [R2: Peer Review] → [사용자 개입] → [R3: Final Decision] → [MD 저장/다운로드]
```

### **4.2 시스템 데이터 흐름**
| **단계**               | **데이터 이동**                                                                 |
|-------------------------|---------------------------------------------------------------------------------|
| **접속**               | 사용자 세션 → LLM API (세션 기반 접근)                                      |
| **R1 (Blind Response)** | LLM1, LLM2, LLM3 → 각 독립 답변 → 사용자 확인/수정 가능                   |
| **R2 (Peer Review)**    | LLM1 → LLM2 답변 리뷰 → 사용자 개입 가능                                    |
| **R3 (Final Decision)** | 사용자 → 최종 답변 저장 (MD 파일)                                          |
