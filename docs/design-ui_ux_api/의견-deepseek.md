#### **1. **핵심 10문서 구성** (깊이 가이드 포함)  
| 문서 종류 | 깊이 | 재사용 포인트 | 검증 기준 |  
|-----------|------|--------------|-----------|  
| **1. 아키텍처** | ★★★★★ | 컴포넌트 다이어그램 + 확장 프로그램 생명주기 | "의존성 설치 방법 질문 없음" |

| **3. LLM 어댑터** | ★★★★★ | ChatGPT/Claude/Gemini별 연결 표준 | 새 LLM 추가 시 1시간 내 적용 가능 |  
| **4. 상태 관리** | ★★★★ | 라운드 전환 로직 상태 머신 | QA가 시나리오 100% 커버 가능 |  
| **5. DB 스키마** | ★★★★ | IndexedDB 테이블 + 마이그레이션 전략 | `ALTER TABLE` 자동 생성 가능 |  
| **6. 에러 처리** | ★★★★ | 타임아웃/429/504 표준 응답 코드 | 모든 에러 케이스 UI 텍스트 포함 |

| **8. 파일 구조** | ★★☆ | `src/core/` vs `src/adapters/` 구분 | 1명이 5분 내 프로젝트 탐색 가능 |  
| **9. 보안** | ★★☆ | `manifest.json` 권한 최소화 | Chrome 스토어 승인 1회 통과 |  
| **10. MD 포맷** | ★★☆ | 헤더/본문 구조 표준화 | 생성된 파일 Git diff 1줄 이내 |

#### **2. **템플릿 병행 작업 시스템**  
**모든 문서를 즉시 재사용 가능한 템플릿으로 생성**:  
```markdown
📂templates/  
├── 📄01-architecture.template.md  <!-- 주석으로 가이드 포함 -->  
│   └─ # [프로젝트명] 아키텍처  
│      ## 컴포넌트 다이어그램 (※Mermaid※)  
│      ## 크롬 API 호출 목록 (※변경불가※)  
├── 📄02-message-protocol.template.md  
│   └─ # 필수 포함 사항:  
│      - 요청/응답 샘플 3종  
│      - 에러 코드 매트릭스  
└── ... (10종)  
```  
**재사용 프로세스**:  
1. 신규 프로젝트 시 `templates/` 복사  
2. `[프로젝트명]` 검색 → 일괄 치환  
3. 차이점만 보완 (평균 2시간 내 완성)

#### **3. **공수 절감을 위한 결정 체크리스트**  
각 문서 끝에 **반드시 포함할 항목**:  
```markdown
## [✓] 결정 사항 체크리스트  
- [ ] 모든 LLM 어댑터 인터페이스 일치 여부 (✔️/✖️)  
- [ ] IndexedDB 버전 컨트롤 전략 (✔️ `v1 → v2` 마이그레이션 경로)  
- [ ] 429 Too Many Requests 시 재시도 횟수 (✔️ 3회)  
```  
> ⚠️ **미결정 항목 존재 시 구현 진입 불가** → No-Question 보장.
