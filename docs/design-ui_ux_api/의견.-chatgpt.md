# 왜 Gemini 단독은 위험한가

Gemini 안은 매우 효율적이지만,
너 목표인:

```text id="n6p4td"
"No Question Build"
+
재사용 가능한 프로젝트 자산화
```

에는 약간 부족 가능성 있음.

특히 부족한 부분:
- runtime state 정의
- orchestration failure
- browser automation lifecycle
- race/sync 정의
- edge-case 통제

이건 실제 구현에서 매우 중요.

즉:
```text id="s8w1ka"
types.ts 만으로는
런타임 ambiguity 제거 불충분
```

가능성 큼.

---

# 반대로 Claude 단독은?

Claude 안은:
- 구조 안정성
- 재사용성
- 누락 방지

매우 강함.

하지만 단점:
```text id="a1f9qx"
사용자 검토 공수 증가
```

실제로 커짐.

특히:
- 문서 많아짐
- 리뷰 포인트 많아짐
- "합의" 피로도 증가

발생 가능.
