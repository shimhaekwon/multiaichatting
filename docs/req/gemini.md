## [요구사항정의서 v1.0] 멀티 LLM 협업 시스템

### 1. 개요 및 목적
* **시스템명:** 멀티 LLM 협업 시스템 (가칭)
* **목적:** 사용자가 브라우저 세션을 통해 접속 가능한 여러 LLM들을 활용하여, 라운드 방식의 협업 및 검토 과정을 거쳐 최종 결론을 도출할 수 있도록 지원하는 시스템.

### 2. 기능 요구사항 (Functional Requirements)

| ID | 대분류 | 상세 요구사항 (협의 항목만 반영) | 비고 |
| :--- | :--- | :--- | :--- |
| **REQ-01** | **LLM 연동** | 사용자가 웹 브라우저에 로그인한 세션을 활용하여 LLM에 접속한다. (API Key 불필요) | |
| **REQ-02** | **LLM 선택** | 접속 가능한 LLM 목록을 화면에 표시한다. | |
| **REQ-03** | | 사용자는 표시된 LLM 중 일부 또는 전체를 선택할 수 있다. | |
| **REQ-04** | **라운드 제어** | 협업 프로세스는 최대 3 라운드로 제한하여 진행한다. | |
| **REQ-05** | | 모든 선택된 LLM의 응답이 완료된 후 다음 라운드로 진행한다. (동기화 방식) | |
| **REQ-06** | | **[R1]** 타 LLM의 답변을 보지 못하는 Blind 상태에서 독립 응답을 수집한다. | |
| **REQ-07** | | **[R2]** 타 LLM의 답변을 서로 검토하도록 프로세스를 진행한다. | |
| **REQ-08** | | **[R3]** 각 LLM의 최종 답변을 도출한다. | |
| **REQ-09** | **사용자 권한** | 사용자는 각 라운드 단계별로 개입할 수 있다. | |
| **REQ-10** | | 사용자는 특정 LLM을 지정하여 1:1로 재질의를 할 수 있다. | |
| **REQ-11** | | 시스템이 제안한 결과에 대한 최종 판단 및 결정은 사용자가 수행한다. | |
| **REQ-12** | **부가 기능** | LLM별 전체 대화 내용을 확인할 수 있는 팝업 뷰 기능을 제공한다. | |
| **REQ-13** | | 대화 및 협업 결과를 Markdown(.md) 파일로 저장하는 기능을 제공한다. | |
