## IT-01 · 기본 3라운드 협업 (해피패스) ★핵심

- **목적**: 라운드 협업 전체 흐름(전송→동기화→교차전달→DONE→저장) 검증
- **사전조건**: ChatGPT·Claude·Gemini 로그인
- **단계**:
  1. LLM 풀에서 **ChatGPT·Claude·Gemini** 선택 → 사이드바 3 체크
  2. 설정바: 라운드=**3**, 토큰=**1000**
  3. 입력 "시장 조사 아이디어를 내줘" → 전송
  4. status **WAITING** → 3 셀 동시 스트리밍 → 전원 완료 → **ROUND_COMPLETE**
  5. "다음 라운드" → 각 봇 발신 prompt에 **타 2종 응답 YAML(자기 제외)** 주입 확인(Console)
  6. 3라운드 도달 → **DONE** → 저장 다이얼로그 자동 오픈
  7. 4개 파일 저장(`chatgpt.md`/`claude.md`/`gemini.md`/`summary.md`)
- **기대/합격**:
  - 화면 동시 스트리밍 정상, 완료 순서 무관 수집
  - 2R+ prompt에 self-exclusion YAML(key=botId 소문자)
  - `summary.md` = 라운드순 + 전원 응답 **원문** 합본 / LLM별 md = 단일 봇 Req·Resp 흐름
- **결과**: ☐ Pass ☐ Fail ☐ Blocked

## IT-02 · STOP → 계속하기 / 종료 ★C1 핵심검증

- **목적**: 진행 중 수동 STOP 시 부분응답 보존·상태 전이·**hang 부재** 검증
- **사전조건**: **DeepSeek 포함** 3종 이상 선택(stopButton 없는 봇 포함이 핵심)
- **단계**:
  1. 라운드 전송 → WAITING 상태에서 응답 생성 중 **STOP** 클릭
  2. 상태 전이·완료/미완료 셀 표기 관찰
  3. **계속하기**(RESUME) → 다음 라운드 진행 / 또는 **종료**(DONE)
- **기대/합격**:
  - STOP 즉시 **STOPPED** 전이, 완료된 셀 응답 **유지**, 미완료 셀 **'stopped'** 표기
  - 중단 라운드가 `history`에 기록되고 스냅샷 저장됨(=hang 없이 마무리)
  - 자동 타임아웃으로 인한 stop 미발생
- **⚠ 검토주의(C1)**: 현재 구현은 STOP 시 `DONE` 통보가 누락돼(특히 **DeepSeek**) `Promise.all`이 **hang → 라운드 미기록** 가능. 본 시나리오가 **C1 수정의 직접 검증**이다.
- **결과**: ☐ Pass ☐ Fail ☐ Blocked

## IT-03 · 라운드 중 LLM swap (확장)

- **사전조건**: 2종 이상 선택, 1라운드 완료(ROUND_COMPLETE)
- **단계**:
  1. ROUND_COMPLETE에서 한 슬롯의 swap ▼ → 미사용 LLM으로 교체
  2. 교체 슬롯의 대화 상태 확인 → 다음 라운드 전송

## IT-04 · 비밀채팅 (휘발성)

- **목적**: 메인 세션과 격리된 1:1 휘발성 대화 검증
- **사전조건**: 임의 LLM 1종 로그인
- **단계**:
  1. 셀 🔒 클릭 → 오버레이로 해당 LLM과 1:1 대화
  2. 메시지 2~3회 송수신 → (N/1000) 카운터 관찰 → 한도 근처에서 입력 차단 확인
  3. 오버레이 **닫기** / 또는 세션 종료
  4. Application→Storage·다운로드 MD 확인
- **기대/합격**:
  - 1:1 대화 동작, 1000토큰 지시문 부가, 카운터 표시·한도 차단
  - 닫기/종료 시 `secretChatAtom` 소멸, **storage·MD 어디에도 저장 흔적 0**
- **⚠ 참고(N-2)**: 비밀채팅은 메인과 같은 사이트 탭/웹 대화 스레드를 공유(앱 레벨 저장만 격리). 메인 진행 중 같은 LLM 비밀채팅은 탭 경합 가능
