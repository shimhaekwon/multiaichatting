## 진단 (브라우저 실측으로 확정)
- **Grok 미작동 근본원인 = promptArea 오류**: config의 `textarea`는 데코이(hidden 프록시), 실제 에디터는 **TipTap/ProseMirror**(`div.tiptap.ProseMirror`). 그래서 입력 자체가 불가였음. sendButton(`chat-submit`)·responseArea(`div.response-content-markdown`)는 원래 정확.
- **에디터 프레임워크별 fill 차이**(실측): textarea=native setter(DeepSeek) / ProseMirror=execCommand(Grok) / **Lexical=paste**(Perplexity, textContent·execCommand 모두 실패) / ChatGPT ProseMirror=textContent 수용.
- **백그라운드 탭 throttle = "2R에서 ChatGPT/Gemini만 영구 진행중"의 근본원인**: hidden 탭에서 `requestAnimationFrame` **완전 정지(rAF=0)**, `setInterval`은 ~1초로 clamp. ChatGPT/Gemini는 스트리밍을 rAF로 렌더 → 숨겨진 탭에서 **DOM에 글자가 안 그려짐** → 어댑터 done(`!!lastText`) 영영 미발화. **같은 hidden 탭에서 Claude는 "4" 렌더·완료, ChatGPT는 innerText 빈 값**(HTML 골격만)으로 입증. 라운드는 `Promise.all`(전원완료)이라 한 셀이 hang하면 전체 미종료.
- **비밀채팅 1사이클**: `SecretChat.tokenLimit`(1000)이 응답 힌트+전체 예산 겸용 → 첫 응답이 예산 소진 → 2번째 전송 차단.
- **ERR_002("login required")**: 실제는 확장 리로드 후 기존 LLM 탭 content script 무효화 → background 연결 실패(오해 유발 메시지).
- **MCP 제약**: Claude-in-Chrome가 `chrome-extension://`(app.html) 접근 불가(3회 chrome-error) + `chrome://extensions` 불가 → **통합 시나리오(IT-01~08) 자율 실행 불가**, LLM 사이트 탭 직접구동은 가능.
