## 2. 핵심 학습 (구현 시 반영)
- **rich editor 입력**(contenteditable: ChatGPT/Claude/Gemini/Perplexity): textContent 주입이 안 먹을 수 있음 → `execCommand('insertText')`/beforeinput·paste 폴백(221 §5).
- **버튼 형태 상이**: 대부분 진짜 `<button>`(aria-label)이나 **DeepSeek은 `<div role="button">`** → `querySelectorAll('button')`로 못 잡힘(content script가 `[role=button]`도 처리해야).
- **stop aria-label은 한글**(언어 종속): Claude "응답 중단", Gemini "대답 생성 중지" → data-testid/아이콘 있으면 우선(언어무관). Perplexity는 아이콘(`pplx-icon-player-stop-filled`) 기반.
- **responseArea는 사이트마다 제각각**: data-testid(ChatGPT/Grok) > 의미있는 id/class(Perplexity `markdown-content`, DeepSeek `ds-assistant-message-main-content`) 우선. 해시·framework class 회피.
- done 판정: stopButton 잡힌 곳(ChatGPT/Claude/Gemini/Perplexity)은 "stopButton 소멸", 못 잡은 곳(DeepSeek/Grok)은 "텍스트 안정화 debounce 1500ms".

## 3. 잔여(비차단, 구현 때 처리)
- Gemini sendButton 빈값 → **Enter 전송 여부 확인**(아니면 send 화살표 selector).
- DeepSeek/Grok stopButton 미확보 → debounce로 완료감지 OK, STOP 클릭은 best-effort.
- Perplexity stopButton 아이콘 기반 **미검증**(생성중 `querySelectorAll(...).length`=1 확인 권장).
- contenteditable 4종 입력 주입 폴백 구현.
