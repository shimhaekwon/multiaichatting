## 2. 진단
### opencode 구독 연동
- 설치: opencode **1.2.26** (최신 1.15.11 — 구버전), 전역 npm.
- 증상: `auth login --method "Claude Pro/Max"` OAuth가 `Failed to authorize`; 모델 추론은 `Not Found`.
- **근본 원인**: 환경변수 `ANTHROPIC_BASE_URL=https://api.anthropic.com` (**/v1 누락**).
  opencode `@ai-sdk/anthropic`가 뒤에 `/messages`만 붙여 → `api.anthropic.com/messages` (404).
- 구독 토큰 자체는 정상. `auth.json`에 유효한 anthropic **oauth** 자격증명 이미 존재(만료 전).
- `Failed to authorize`는 구버전/플러그인 깨짐 부산물(=재로그인만 실패, 기존 토큰은 유효).
- 플러그인 3개 로드 실패: `opencode-user-launch-plugin`(npm 404), `oh-my-openagent`(`fn5 is not a function`), `@ex-machina/opencode-anthropic-auth`(`Invalid Version: latest`).

### ultrawork
- `ultrawork`는 **oh-my-opencode/oh-my-openagent 플러그인 기능**(키워드 트리거 + 멀티에이전트 + ralph-loop + 모델 오버라이드). opencode 전용 → Claude Code엔 플러그인 이식 불가.
- Claude Desktop엔 Claude Code 내장(동일 엔진/plugin parity)이나, **opencode 플러그인 ≠ Claude Code 플러그인** → 네이티브 재구현만 가능.
- Agent Skill은 범용(웹/앱/Code/API 공유)이나, hook·subagent·/loop는 Claude Code 전용.

## 3. 처방

- `C:\Users\PC\.config\opencode\opencode.json` provider에 추가:
  `"anthropic": { "options": { "baseURL": "https://api.anthropic.com/v1" } }`
  (전역 env는 미변경 → Claude Code 등 타 도구 영향 0)
- 결과: 인증 성공, 남은 차단은 **Pro 사용량 소진**("out of extra usage") 뿐 = 연동 정상.

### Claude Code ultrawork 세트 (전역 `C:\Users\PC\.claude\`)
- `skills\ultrawork\SKILL.md` — 작업 규율(교본)
- `hooks\ultrawork-detect.js` — 키워드 감지 훅 (생성 → **이후 등록 해제**, 명시 `/ultrawork` 전용으로 전환)
- `agents\uw-explorer.md` / `uw-implementer.md` / `uw-critic.md` — 정찰/구현/검수 서브에이전트
- SKILL.md를 **업무처리원칙 v4 + 코드생성원칙**과 화해(v3): "자율 주도 + 에스컬레이션 펜스",
  하드스톱 2개(위험/모순, 파괴적/보안). uw-critic 독립검수로 10건(H1~L10) 보완.
