## 2. 발견 이슈

### 🔴 C1 — STOP/abort 시 DONE 미통보 → 라운드 영구 hang (CRITICAL)

**경로(확정):**
1. `AbstractBot.doSendMessageGenerator`(`abstract-bot.ts` L47-75): ReadableStream은 **`onEvent(DONE)`→close / `onEvent(ERROR)`→error 로만 종료**. abort signal로는 스트림을 닫지 않음.
2. `DomAdapterBot`(`dom-adapter-bot.ts` L25-31): abort 시 `port.postMessage({action:'STOP'})`만. **`onEvent(DONE)`/cleanup 호출 안 함.**
3. content script(`dom-adapter.ts` L113-117): STOP 수신 시 `findStop()?.click()`만. **observe timer 중단·`DONE` 통보 누락**(222 §8/§174 명세 위반).

**영향:**
- **deepseek**(`stopButton:''`): click 불가 → 생성 지속 시 debounce done 안 옴 → `for await` 미종료 → `Promise.all` **hang** → 중단 라운드 `history.push`/`persist` 미실행 → "중단됨 — 받은 응답까지 저장" 동작 불가.
- stopButton 보유 5종도 click 미반영/DOM 미변경 시 동일 hang. 정상 동작은 "click→사이트 중단→stop 소멸→done" 우연 경로에만 의존.

**수정 방향(222 §8 명세대로):** content script가 STOP에서 timer를 끊고 즉시 DONE을 통보.
- `dom-adapter.ts`: SEND_PROMPT 시 `observeResponse` 반환 cleanup을 변수로 보관 → STOP 핸들러에서 `findStop()?.click()` **+ `stopObserve?.()` + `port.postMessage({action:'DONE'})`**.
- (방어적 보강) `dom-adapter-bot.ts` abort 리스너에서도 `params.onEvent({type:'DONE'})` + `cleanup()` 호출해 CS 무응답 시에도 스트림이 닫히도록.

### 🟡 M1 — connectToCSWithRetry 재시도가 실효 없음 → 거짓 ERR_002

`background/index.ts` L37-64: `for(retries=3)` 루프지만 **첫 시도에서 `return new Promise(...)`로 즉시 탈출**. 100ms 내 `onDisconnect` 발생 시 `resolve(null)` → 재시도 0회. 탭 신규 오픈 직후 CS 주입이 100ms를 넘으면 거짓 "로그인 필요(ERR_002)".

**수정 방향:** disconnect를 `await` 직렬 판정으로 바꿔 실패 시 `delay`(300ms) 후 다음 이터레이션 진행, 모두 실패해야 null 반환.

### 🟡 M2 — proxy-fetch.ts 미삭제 (222 §1.2 위반)

`src/services/proxy-fetch.ts` 잔존(전 봇 DOM 전환으로 dead code 추정, import 0). 명세상 삭제 대상.

**수정 방향:** 파일 삭제 + `proxyFetch`/`proxy-fetch` import 잔재 grep 0 확인. (참고: `000.AICOWORK/`는 OpenCode 분석 산출물 폴더로 코드와 무관 — 제외.)

### 🟢 LOW (권장)

- **L1** `secretChatAtom = atomWithImmer`(`state/round.ts` L15) — 222는 `atom()` 명시. **단 atomWithImmer도 휘발성(스토리지 미사용)이라 저장 금지 요구는 충족** — 기능 무방, 명세 문구만 상이.
- **L2** `generating = useMemo(..., [slots])`(`use-round-engine.ts` L59) — 의존성에 `s.chat.generating` 누락. atom 리렌더로 대개 갱신되나 stale 여지.
- **L3** 자동 테스트 0 — 220 TC-01~14 다수 정의됐으나 미자동화(219 Phase는 수동 검증 위주라 필수는 아니나 회귀 위험).
