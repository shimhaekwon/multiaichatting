## 2. 항목별 상세

### ERR_002 — `promptArea` 렌더 폴링 대기
**원인**: `isLoginPage`가 `SEND_PROMPT` 수신 즉시 실행 → `background.ensureTab`은 `status='complete'`까지만 기다리지 SPA 렌더까지는 안 기다림 → 새로 연 탭에서 `promptArea`가 아직 없으면 ERR_002 오발("Login required or site not ready").

```js
// AFTER (dom-adapter.ts, SEND_PROMPT)
currentSelectors = s
// [ERR_002] promptArea 렌더 대기: status=complete여도 SPA 렌더 전일 수 있음 (최대 ~3초)
for (let i = 0; i < 30 && !document.querySelector(s.promptArea); i++) {
  await new Promise((r) => setTimeout(r, 100))
}
log('promptArea?', !!document.querySelector(s.promptArea), 'isLoginPage?', isLoginPage(s))
if (isLoginPage(s)) { /* ERR_002 */ }
```

### RT-1 — done 판정 안정화 fallback
**원인**: `observeResponse`의 done이 **stopButton 소멸에만** 의존(`done = !stop && !!lastText`). 응답 완료해도 stop 버튼이 안 사라지거나 selector가 어긋나면 자동 종료 안 됨 → STOP 수동 필요. (런타임 테스트 증상: 1R 응답 끝났는데 STOP 눌러야 종료, 2R 무한 진행)

```js
// AFTER (dom-adapter.ts, observeResponse)
const stable = Date.now() - lastChange > debounce
...
} else if (s.stopButton) {
  // [RT-1] stop 소멸 OR 텍스트 안정화 — selector 노후 시에도 종료되도록 안전망
  done = !!lastText && (!stop || stable)
}
```
> 주의: `lastText`(responseArea)가 아예 안 잡히면 이 fallback도 무력 → 재테스트의 `observe` 로그로 확인.

### 진단 — `[MC]` 디버그 로그
`DEBUG=true`일 때 각 LLM 탭 콘솔에 출력. 단계: `SEND_PROMPT` 수신 → `promptArea?`/`isLoginPage?` → `observe {resp,len,stop,stable,done}`. 안정화 후 `const DEBUG = false`로.

### U-1 — SecretChatOverlay Hooks 규칙 위반 (🔒 크래시)
**원인**: `if(!secretData) return null`(L19) **뒤**에 `useCallback`(L29). 오버레이 항상 마운트+atom 초기 null → 🔒 클릭 시 훅 수 증가 → React 크래시 + `react-hooks/rules-of-hooks` 빌드 에러.
**수정**: `useCallback` → 일반 async 함수(훅 아님 → early-return 뒤 허용). `import`에서 `useCallback` 제거.

### U-5 — 🔒 이중표기
**원인**: `title={`🔒 ${t('secretChatTitle')}`}` 인데 키 값도 `🔒 비밀채팅 — {{name}}` → "🔒 🔒".
**수정**: `title={t('secretChatTitle', { name })}` (코드의 🔒 제거).

### U-3 — swap 미결선
**원인**: `MultiBotChatPanel`이 `ConversationPanel`에 `onSwitchBot` 미전달 → swap ▼ 미렌더 → `engine.swap()` 도달 불가.
**수정**: `onSwitchBot={(b) => engine.swap(s.slot, b)}` 결선.

### U-2 — i18n 기존 키 소실
**원인**: ko/en이 신규 32키만 남고 ChatHub 원본 키 삭제 → `t('Stop')`/`t('Share conversation')`/`t('Settings')` 등 원시 키 노출(222 §17 위반).
**수정**: ko/en.json에 `Stop·Send·Share conversation·Clear conversation·View history·GitHub·Feedback·Display·Settings` 9개 추가.
