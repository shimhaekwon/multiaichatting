## 3. E2E 시나리오

### 3.1 메인 흐름 (해피패스)
1. LLM 풀에서 **ChatGPT·Claude·Gemini** 선택 → 사이드바 3 체크.
2. 설정바: 라운드=**3**, 토큰=**1000** → 입력 "시장 조사 아이디어를 내줘" → 전송.
3. status **WAITING** → 3 셀 동시 스트리밍 → 전원 완료 → **ROUND_COMPLETE**.
4. "다음 라운드" → 각 봇 prompt에 **타 2종 응답 YAML(자기 제외)** 주입 확인(개발 로그/네트워크).
5. 3라운드 도달 → **DONE** → 저장 다이얼로그 → 4개 파일 저장.
   - `chatgpt.md`: R1 Req/Resp, R2 Req(+타 LLM)/Resp 흐름 (TC-12)
   - `summary.md`: 라운드순 Req + 3종 전원 응답 원문 합본 (TC-13)
6. **합격**: 화면 스트리밍 정상 + 4파일 내용 정합 + self-exclusion 확인.

### 3.2 STOP 분기
- WAITING 중 **STOP** → 완료된 셀 응답 유지, 미완료 셀 `stopped` 표기 → **계속하기**(RESUME→ROUND_ACTIVE) 또는 **종료**(DONE). 자동 stop 발생 없음. (TC-04b/11)

### 3.3 비밀채팅
- 셀 🔒 → Claude와 1:1 → 메시지 송수신 → (N/1000) 카운터 → **닫기** → 저장물·storage 어디에도 흔적 없음. (TC-09a/b)

### 3.4 최소/경계
- 참여 **1종만** 선택 → 전송 가능(최소 1). / 6종 초과 선택 시 toast 차단. (TC-06b)
