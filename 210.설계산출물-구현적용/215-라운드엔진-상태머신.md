# 215. 라운드 엔진 · 상태머신

> **자기완결**. 구현: `src/app/hooks/use-round-engine.ts`(신규). 엔진은 **앱 컨텍스트**(213). 각 슬롯은 기존 `useChat`를 재사용해 스트리밍/중단을 처리한다. [대조용 초안: 200-초안/230]

## 1. 상태 / 이벤트

| 상태 | 의미 | 다음 |
|------|------|------|
| `IDLE` | 미시작 | START→ROUND_ACTIVE |
| `ROUND_ACTIVE` | prompt 입력 가능 | SEND_PROMPT→WAITING |
| `WAITING` | 응답 대기 | ALL_COMPLETE→ROUND_COMPLETE / STOP→STOPPED |
| `ROUND_COMPLETE` | 라운드 완료(roundNo<maxRounds) | NEXT_ROUND→WAITING(§4 sendPrompt 재호출, ROUND_ACTIVE 경유 생략) / END→DONE |
| `STOPPED` | 사용자 중단 | RESUME→ROUND_ACTIVE / END→DONE·IDLE |
| `DONE` | 최대 라운드 도달 | SAVE_MD→종료 |

## 2. 훅 시그니처

```ts
export interface RoundEngine {
  session: RoundSession | null
  slots: { slot: number; botId: BotId; chat: ReturnType<typeof useChat> }[]
  generating: boolean
  start(maxRounds?: number, tokenLimit?: number): void  // 생략 시 maxRoundsAtom/tokenLimitAtom 기본값(214 §2)
  sendPrompt(userPrompt: string): Promise<void>     // ROUND_ACTIVE → WAITING → ROUND_COMPLETE/DONE (세션을 store에서 읽음 → start 직후 호출 안전)
  nextRound(userPrompt: string): Promise<void>       // ROUND_COMPLETE → 다음 라운드 (= sendPrompt 재사용)
  stop(): void                                       // WAITING → STOPPED
  resume(): void                                     // STOPPED → ROUND_ACTIVE
  endSession(): void                                 // → DONE/IDLE (저장 다이얼로그 트리거)
  swap(slotIndex: number, newBotId: BotId): void     // FR-14★ (VER-002 확장) — 중복 botId·WAITING 중 차단
}
export function useRoundEngine(): RoundEngine
```

## 3. 슬롯 구성 (React 훅 규칙 준수)

훅은 조건부 호출 불가 → **고정 6회 `useChat` 호출 후 활성 슬롯만 사용**(ChatHub `SixBotChatPanel` 패턴, `MultiBotChatPanel.tsx:167`).

```ts
import { useStore, useAtomValue } from 'jotai'
const store = useStore()                                   // [C-1] Provider-safe jotai store. 응답 수집/세션 판정에 "최신" atom 값을 읽는 데 사용
const selected = useAtomValue(selectedLLMsAtom)            // 최대 6
const ids = padTo6(selected)                              // [G4] 빈 슬롯은 selected[0](없으면 'chatgpt')로 패딩 — 더미(유효 BotId 필수)
const c0 = useChat(ids[0]); /* ... */ const c5 = useChat(ids[5])
const slots = selected.map((botId, i) => ({ slot: i, botId, chat: [c0,c1,c2,c3,c4,c5][i] }))
```

> **[C-1] 중요 — 응답은 캡처된 `s.chat.messages`가 아니라 store에서 읽는다.** `chatFamily`는 `atomWithImmer`(불변 갱신)라, 비동기 `sendPrompt` 클로저가 잡은 `s.chat.messages`는 **호출 시점 스냅샷(빈 값)** 이다. 스트리밍 완료 후 최신 응답은 반드시 `store.get(chatFamily({botId, page:'singleton'})).messages`로 읽어야 한다(아래 §4). 세션 상태(STOP 여부)도 같은 이유로 `store.get(roundSessionAtom)`로 판정한다. (jotai `<Provider>` 미사용 시 `useStore()`는 기본 store를 반환 — ChatHub 기본값과 일치)

> 슬롯 page 키: `useChat(botId)`는 내부적으로 `chatFamily({botId, page:'singleton'})` 사용. 동일 botId가 두 슬롯에 오지 않도록 LLM풀에서 중복 선택 차단(218).
>
> **[G4] 빈 슬롯 처리**: `padTo6(selected) = [...selected, ...Array(6 - selected.length).fill(selected[0] ?? 'chatgpt')]`. 더미로 채운 인덱스(`i >= selected.length`)는 **`slots` 배열에서 제외**하여 렌더·전송 모두 안 함(React 훅 규칙상 `useChat`은 6회 고정 호출, 결과만 미사용). `selected`는 1~6개 보장(최소 1 — VER-002). `createBotInstance`는 6종 BotId만 처리하므로 패딩값도 유효 BotId여야 함.

## 4. 핵심 동작 의사코드

> 📌 `genId()` 구현체 → [222 §6]. `padTo6`은 §3에 정의. 아래에서 헬퍼가 미정의로 보이면 222 참조.

```ts
// setRoundSession = useSetAtom(roundSessionAtom) — atomWithImmer setter(recipe (draft)=>void).
// jotai set은 store에 "동기" 반영되므로, start() 직후 같은 tick의 sendPrompt가 store.get으로 즉시 읽을 수 있다. [C-2]

function start(maxRounds = store.get(maxRoundsAtom), tokenLimit = store.get(tokenLimitAtom)) {
  setRoundSession({ id: genId(), createdAt: Date.now(), status: 'ROUND_ACTIVE',
    currentRound: 0, maxRounds, tokenLimit, slots: slots.map(s=>({slot:s.slot,botId:s.botId})), history: [] })
}

async function sendPrompt(userPrompt) {                    // FR-01: 라운드당 1회
  const s0 = store.get(roundSessionAtom)                   // [C-1/C-2] 세션은 closure가 아니라 store에서 최신값
  if (!s0 || (s0.status !== 'ROUND_ACTIVE' && s0.status !== 'ROUND_COMPLETE')) return   // [H-1] ROUND_ACTIVE(R1)+ROUND_COMPLETE(다음 라운드) 허용. ROUND_COMPLETE는 ROUND_ACTIVE 경유 생략하고 WAITING 직행 (start 직후에도 store 반영됨)
  const roundNo = s0.currentRound + 1
  const tokenLimit = s0.tokenLimit
  const prev = s0.history[s0.history.length - 1]
  setRoundSession(d => { d.status = 'WAITING'; d.currentRound = roundNo })
  // FR-07: 순서 무관 — 병렬 발사. [M2] 각 슬롯이 "STOP 전에" 끝났는지 기록 = 자연 완료 판정
  //   (useChat.sendMessage는 완료·에러·abort 어느 경우든 resolve하므로, resolve 시점의 status로 구분)
  const completedNaturally = new Set()
  await Promise.all(slots.map(async (s) => {
    const ctx  = buildContextForLlm(s.botId, prev, roundNo, userPrompt, slots.map(x=>x.botId))   // 216 (1라운드는 prev 없음, participants로 빈 키 생성)
    const full = composePrompt(userPrompt, ctx, tokenLimit)               // 216 (토큰지시문 FR-02 포함)
    await s.chat.sendMessage(full)   // useChat 래퍼(string): 스트리밍 셀 갱신 + 완료/에러/abort 시 resolve (내부 AbortController, STOP은 stopGenerating)
    if (store.get(roundSessionAtom)?.status !== 'STOPPED') completedNaturally.add(s.botId)   // STOP 눌리기 전 종료 = 자연 완료
  }))
  // [C-1] 응답 수집 — 반드시 store에서 "최신" 메시지를 읽는다(캡처된 s.chat.messages는 stale).
  const responses = {}
  for (const s of slots) {
    const msgs = store.get(chatFamily({ botId: s.botId, page: 'singleton' })).messages   // ← 최신
    const last = msgs.filter(m => m.author === s.botId).at(-1)
    const errored = !!last?.error
    const completed = !errored && completedNaturally.has(s.botId)         // [M2] STOP 전 자연 완료만 completed
    responses[s.botId] = {
      content: completed ? (last?.text ?? '') : '',                       // [M2] 중단/에러의 부분응답은 폐기(VER-001 3.3 "대기 중 응답 폐기"), self-exclusion에서도 자동 제외
      tokens: approxTokens(completed ? (last?.text ?? '') : ''),
      status: errored ? 'error' : (completed ? 'completed' : 'stopped'),  // 에러/완료/중단 구분
      timestamp: Date.now(),
    }
  }
  const stopped = store.get(roundSessionAtom)?.status === 'STOPPED'
  setRoundSession(d => {
    d.history.push({ round: roundNo, userPrompt, responses })             // STOP 라운드도 부분 기록(완료분 completed + 나머지 stopped)
    if (!stopped) {                                                       // STOP이면 STOPPED 유지(전이 덮어쓰기 금지)
      d.status = roundNo >= d.maxRounds ? 'DONE' : 'ROUND_COMPLETE'
    }
  })
  await persistRoundSnapshot(store.get(roundSessionAtom))                 // 214 §5 (최신 스냅샷)
}

const nextRound = sendPrompt   // [H-1] sendPrompt 가드가 ROUND_COMPLETE를 허용 → 다음 라운드 진입(ROUND_COMPLETE→WAITING 직행, roundNo=currentRound+1로 증가). UI는 현재 입력 텍스트를 인자로 전달(218 §2/§5)

function stop() {                                           // FR-11: 수동 STOP
  slots.forEach(s => s.chat.stopGenerating())               // abortController.abort()
  setRoundSession(d => { d.status = 'STOPPED' })  // 진행 중 sendPrompt는 abort로 Promise.all이 곧 resolve → store에서 STOPPED 감지 → 부분 응답 수집·history 기록·STOPPED 유지
}

function resume() { setRoundSession(d => { if (d.status==='STOPPED') d.status = 'ROUND_ACTIVE' }) }

function endSession() {
  const id = store.get(roundSessionAtom)?.id
  setRoundSession(d => { if (d) d.status = 'DONE' })  // 저장 다이얼로그(218)에서 saveSessionMarkdown 호출
  if (id) clearRoundSnapshot(id)
  setSecretChat(null)        // 비밀채팅 휘발 소멸
}

function swap(slotIndex, newBotId) {                        // FR-14★ (VER-002 확장)
  if (store.get(roundSessionAtom)?.status === 'WAITING') return         // 진행 중(응답 대기)이면 차단
  setSelectedLLMs(prev => {
    if (prev.includes(newBotId)) return prev                            // [중복 차단] 이미 다른 슬롯이 쓰는 봇이면 무시(동일 chatFamily 공유 방지)
    return prev.map((b,i) => i===slotIndex ? newBotId : b)
  })
  // [N-3] session.slots도 동기 갱신 — 안 하면 summary/per-LLM md/snapshot이 start() 시점 stale 슬롯을 써서 교체된 봇 응답이 누락/오귀속된다.
  setRoundSession(d => { if (d && !d.slots.some(s => s.botId === newBotId)) { const sl = d.slots.find(s => s.slot === slotIndex); if (sl) sl.botId = newBotId } })
  // [222 §12 확정] 파일 봇 집합 = session.slots(정본, swap도 본 함수 N-3로 동기). history 합집합안은 미채택.
  // 새 봇 슬롯은 새 conversation(빈 messages)
}
```

## 5. 전 슬롯 완료 판정

- `useChat.sendMessage`는 스트림 종료/에러 처리 후 resolve → `Promise.all`이 곧 ALL_COMPLETE.
- 보조 표시용 `generating = slots.some(s => s.chat.generating)`.
- 자동 타임아웃 **없음**(설계 — ERR_003은 STOP 수동 처리). 장시간 미응답은 사용자 STOP.
