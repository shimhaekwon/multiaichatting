## 1. 문제 (근본원인 재확인)

- 숨겨진(백그라운드) 탭에서 브라우저는 `requestAnimationFrame`을 **완전 정지**, `setInterval`을 ~1초로 clamp.
- ChatGPT/Gemini 등은 스트리밍 응답을 **rAF로 렌더** → 숨겨진 탭에선 **DOM에 글자가 안 그려짐**.
- 현재 어댑터(`dom-adapter.ts`)의 done 판정은 `els[last].innerText` 폴링 기반 → 글자가 안 그려지면 **done 영영 미발화** → 라운드(`Promise.all`) hang.
- **현 안전망**(`observeResponse`): MutationObserver + 60s 타임아웃 + 포그라운드 힌트 + 180s 하드상한.
  → "영구 hang"은 막았지만, **여전히 탭을 앞으로 가져와야** 응답을 받는다(throttle 자체는 미해결).

## 2. 해결 아키텍처 — MAIN world 네트워크 인터셉트

완성 스트림(SSE/청크)을 **네트워크 레벨에서 캡처**하면 렌더·탭 가시성과 무관하게 토큰을 받는다(throttle 완전 우회).

**주입 방식 결정 = `chrome.scripting.executeScript({ world:'MAIN', func })`** (background에서 전송 직전 호출)

| 후보 | 판정 | 이유 |
|------|------|------|
| crxjs `content_scripts` + `world:'MAIN'` | ✗ | crxjs(beta.21)는 `world` 미지원 + CS를 **async ESM 로더**로 래핑(`runtime.getURL` dynamic import). 이 로더는 MAIN world에서 `chrome.runtime` 부재로 **작동 불가**. |
| 페이지에 `<script>textContent>` 인라인 주입 | ✗ | ChatGPT/Claude/Gemini는 **CSP(script-src)**가 인라인 스크립트 차단. |
| `<script src=runtime.getURL()>` + WAR | △ | 가능하나 **독립 IIFE 별도 빌드** 필요(crxjs 로더 비경유) → 빌드 복잡. |
| **`executeScript({world:'MAIN', func})`** | **✅** | Chrome 주입이라 **CSP 면제**, 2차 빌드/WAR 불요, vite target(chrome87+)이라 async 네이티브 유지 → `func.toString()` 직렬화 안전. |

**타이밍**: 완성 fetch는 "프롬프트 전송 시" 발생(페이지 로드 한참 후) → `SEND_PROMPT` 처리 중(전송 직전) 주입이면 **항상 선행 패치**. document_start 불요. (전체 fetch 커버가 필요하면 Phase 3에서 `registerContentScripts({world:'MAIN', runAt:'document_start'})`로 승격 가능.)

### 안전 불변식 (Phase 1 = "추가만", 기존 경로 무손상)
1. 페이지 응답 **절대 무변형** — 원본 Promise/Response 그대로 반환, **`clone()`만** 별도로 읽음. fetch 원본 먼저 호출, 캡처는 try/catch best-effort.
2. `window.__mcNetInstalled` 가드로 **idempotent**(매 전송 호출해도 이중 패치 없음).
3. 캡처는 `window.__mcNet` 링버퍼(상한 40엔트리/200KB)에만 적재. **done 판정·라운드 로직 미사용**.
4. `func`는 **100% 자기완결**(외부 import/모듈전역/헬퍼 참조 0). 표준 web 전역만 사용 → MAIN world 직렬화 안전(빌드 산출물에서 esbuild 헬퍼 참조 0 확인).
