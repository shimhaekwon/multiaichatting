## 1. 제품 한 줄 정의

여러 LLM(6종)과 **API 키 없이** 브라우저 로그인 세션으로 대화하며, **라운드 기반**으로 의견을 주고받고(라운드 종료 후 **타 LLM 응답만** 다음 라운드에 주입), 결과를 **MD로 산출**하는 브레인스토밍용 Chrome 확장.

## 2. ChatHub와의 관계 (무엇을 가져오나)

ChatHub는 "여러 봇에 같은 질문을 던져 답을 나란히 비교"하는 확장이다. 본 제품은 그 위에 **라운드 엔진·교차전달·비밀채팅·MD 산출**을 얹는다.

| 가져오는 것(재사용) | 새로 만드는 것 |
|---------------------|----------------|
| MV3 스캐폴딩·Vite 빌드·앱 셸(app.html 탭) | 라운드 상태머신 엔진 |
| React+Jotai+Tailwind UI 셸·2×3 그리드·입력창·사이드바 | self-exclusion YAML context 빌더 |
| 마크다운 렌더러·공유·기록 컴포넌트 | **6종 전부 DOM 어댑터**(`DomAdapterBot`) + 공통 content script |
| `AbstractBot` 계약·`useChat` 스트리밍·`chatFamily` | 비밀채팅(휘발성)·MD exporter·세션 복구 |
| host permission·storage·i18n·테마 | LLM풀 선택·설정바(라운드/토큰) |

## 6. 코딩 지침 (④ 다름없이 — 기존 패턴 미러링)

- **TypeScript strict 유지**. 경로 alias `~* → src/*`, `~app/*`, `~services/*`, `~utils/*` 사용(tsconfig.json paths).
- **봇**: 반드시 `AbstractBot`(또는 `AsyncAbstractBot`) 상속, `doSendMessage(params: SendMessageParams)`에서 `params.onEvent({type:'UPDATE_ANSWER'|'DONE'|'ERROR', ...})` 이벤트 계약 준수(`src/app/bots/abstract-bot.ts`).
- **상태**: Jotai. 지속 필요하면 `atomWithStorage`, 가변 드래프트는 `atomWithImmer`(`jotai-immer`) — 기존 `chatFamily` 패턴 따름.
- **에러**: `throw`/이벤트에 `ChatError(message, ErrorCode)` 사용(`src/utils/errors.ts`). 신규 에러는 `ErrorCode`에 추가(217 §5, ERR_001~005 매핑).
- **권한**: 크로스오리진 호출 전 `requestHostPermission(host)`(`src/app/utils/permissions.ts`).
- **i18n**: 사용자 노출 문구는 `useTranslation()`의 `t()` 사용, 한글 하드코딩 금지. ko/en 키 동시 관리.
- **스타일**: Tailwind 유틸 + `cx()`(`src/utils`), CSS 변수(`--primary-background` 등) 사용.
- **프로덕션**: `vite.config.ts`가 console/debugger 제거(esbuild.drop) — 디버그 로그는 자유롭게.

## 7. 위험 (Step 1.5) [2026-05-29 갱신]

> Phase 0 실증: 1.45.7 내부 API 봇 전부 실패 확인 → **6종 전부 DOM으로 전환**(하이브리드 폐기).

1. **selector 노후**: 6종 DOM이라 사이트 개편 시 selector 갱신 필요 → `adapter-config.json`(version·**단일 출처**) 분리로 **로직 재작성 없이** config 수정+재빌드로 대응(런타임 fetch 채택 시 무재빌드, 217 §3.3 [M-2]).
2. **사이트별 DOM 난이도 상이** → Phase 2에서 6종 각 PoC 실측.
3. **탭 의존**: 각 사이트 탭 오픈+로그인 필요 → 미로그인 시 "로그인 필요" 표시(설계 session 체크 제거).
4. **ToS 회색지대**: 무키 웹 자동화는 각 사 ToS 회색지대 — 단일 사용자 내부용 전제. / 원안 SW엔진→앱 컨텍스트 엔진(213, 사용자 인지).
