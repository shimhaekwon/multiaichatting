## 3. DOM 어댑터 (6종 공통)

### 3.1 공통 베이스 `src/app/bots/dom-adapter-bot.ts` (신규)

```ts
import { AbstractBot, SendMessageParams } from './abstract-bot'
import { ChatError, ErrorCode } from '~utils/errors'
import { BotId } from './index'

// AdapterSelectors = adapter-config.json 한 항목의 타입(콘텐츠 스크립트가 로드해 사용). 봇 클래스엔 두지 않는다. [M-2]
export interface AdapterSelectors {
  version: number
  url?: string
  promptArea: string          // textarea | contenteditable
  promptIsContentEditable?: boolean
  sendButton?: string         // 빈 문자열/생략 시 Enter keydown
  responseArea: string        // 마지막 요소가 응답
  stopButton?: string
  generatingFlag?: string      // 존재하면 생성 중 (예: stop 버튼/스피너)
  doneDebounceMs?: number      // 텍스트 안정화 done 판정용(기본 1500) [G14]
  loginIndicator?: string      // [222 §8] 로그인 폼 감지용 selector(옵션). 미정의 시 promptArea 부재로 미로그인 간주
}

export abstract class DomAdapterBot extends AbstractBot {
  abstract llm: BotId          // [M-2] 봇은 llm 식별자만 보유 — selector는 content script가 adapter-config.json에서 조회

  async doSendMessage(params: SendMessageParams) {
    const port = chrome.runtime.connect({ name: `dom:${this.llm}` })
    const cleanup = () => port.disconnect()
    params.signal?.addEventListener('abort', () => { port.postMessage({ action: 'STOP' }); })
    port.onMessage.addListener((msg) => {
      if (msg.action === 'RESPONSE_CHUNK') params.onEvent({ type: 'UPDATE_ANSWER', data: { text: msg.content } })
      else if (msg.action === 'DONE') { params.onEvent({ type: 'DONE' }); cleanup() }
      else if (msg.action === 'ERROR') { params.onEvent({ type: 'ERROR', error: new ChatError(msg.message, MAP[msg.code] ?? ErrorCode.UNKOWN_ERROR) }); cleanup() }
    })
    port.postMessage({ action: 'SEND_PROMPT', llm: this.llm, prompt: params.prompt })   // [M-2] selector 미전송 — CS가 config에서 조회
  }
  resetConversation() { /* 선택: 새 대화 시작 메시지(탭 navigate) */ }
}
// [M-2] ERR_001~005 전수 매핑(§5). LLM_TAB_NOT_FOUND/DOM_SELECTOR_FAILED/TOKEN_LIMIT_EXCEEDED는 errors.ts ErrorCode에 신규 추가.
const MAP: Record<string, ErrorCode> = {
  ERR_001: ErrorCode.LLM_TAB_NOT_FOUND, ERR_002: ErrorCode.MISSING_HOST_PERMISSION,
  ERR_003: ErrorCode.NETWORK_ERROR,     ERR_004: ErrorCode.DOM_SELECTOR_FAILED,
  ERR_005: ErrorCode.TOKEN_LIMIT_EXCEEDED,
}
```

### 3.2 개별 어댑터 (selector)

> **selector 단일 출처(authoritative) = `src/content-script/adapter-config.json`** (아래 [G2 명세]·실측값은 221 §7). **[M-2 확정] 봇 클래스는 selector를 보유·전송하지 않는다** — content script가 `adapter-config.json`을 로드해 `llm` 키로 조회한다(§3.3 `loadConfig`). 따라서 6종 봇 클래스는 `llm` 식별자만 갖는 trivial 클래스이며, selector 변경은 `adapter-config.json` 수정으로 끝난다(로직 재작성 없음).

```ts
// 6종 봇 클래스 = llm 식별자만. selector는 adapter-config.json(§3.2 [G2]/221 §7). [M-2]
// bots/chatgpt/index.ts (내부 API → DOM 재작성), claude/perplexity/grok 동일하게 재작성, gemini/deepseek 신규
export class ChatGPTDomBot    extends DomAdapterBot { llm = 'chatgpt'    as const }
export class ClaudeDomBot     extends DomAdapterBot { llm = 'claude'     as const }
export class PerplexityDomBot extends DomAdapterBot { llm = 'perplexity' as const }
export class GeminiDomBot     extends DomAdapterBot { llm = 'gemini'     as const }
export class DeepSeekDomBot   extends DomAdapterBot { llm = 'deepseek'   as const }
export class GrokDomBot       extends DomAdapterBot { llm = 'grok'       as const }
```
> **selector 실측값은 221 §7(2026-05-29 확정)** 을 `adapter-config.json`에 그대로 반영. 아래 [G2] json은 초안 기본값이며 **221 §7 통합 json으로 대체**한다.
> **selector 확보·검증 절차 [G2 명세]** — selector는 **초안 추정치**이므로 구현 시 반드시 검증:
> 1. 대상 사이트 로그인 후 DevTools(F12) → Elements에서 입력창/전송버튼/응답영역/정지버튼 inspect. 안정적 속성 우선순위: `data-testid` > `aria-label` > 역할/태그 > `class`.
> 2. selector는 하드코딩 대신 **`src/content-script/adapter-config.json`**(version 필드 포함)으로 분리:
>    **→ [M-1] 초안 추정 JSON 삭제됨.** selector **단일 출처 = [221 §7 "통합 `adapter-config.json`"]** 의 실측 확정값(2026-05-29)을 그대로 `src/content-script/adapter-config.json`에 복사한다. (초안 추정치는 실측과 달랐다: claude `div.standard-markdown`, deepseek `div.ds-assistant-message-main-content`, grok `div.response-content-markdown`, perplexity promptArea `#ask-input`·responseArea `div[id^='markdown-content']`·send `button[aria-label='제출']`.) 217 문서는 selector 값을 보유하지 않는다.
> 3. **fallback**: content script가 `promptArea`/`responseArea` 미발견 시 `ERROR(code:'ERR_004')` → 셀에 "사이트 구조 변경 — selector 업데이트 필요" 표시(자동 복구 안 함, 운영자가 adapter-config.json 수정·version++).
> 4. Phase 2 PoC에서 6종 각 1회 왕복 실증 후 version=1 확정.

### 3.3 콘텐츠 스크립트 `src/content-script/dom-adapter.ts` (신규)

> 📌 `fillPrompt`/`pressEnterOrClick`/`observeResponse`/`isLoginPage`의 **실구현 본문은 [222 §8]** (221 §2 실증 코드 기반, 복사 가능). 아래는 흐름 의사코드. `ensureTab` 본문은 222 §9.

**6종 사이트 전부**(`chatgpt.com`/`claude.ai`/`*.perplexity.ai`/`gemini.google.com`/`chat.deepseek.com`/`grok.com`)에 주입.
```ts
// [M-2] selector는 adapter-config.json에서 로드(봇은 llm/prompt만 보냄). resolveJsonModule:true 라 import 가능.
import cfg from './adapter-config.json'   // 번들 인라인(빌드 시 확정). ‘무재빌드 갱신’이 필요하면 fetch(chrome.runtime.getURL(...)) + web_accessible_resources로 교체
async function loadConfig() { return cfg as Record<string, AdapterSelectors> }

chrome.runtime.onConnect.addListener((port) => {            // background 경유 relay
  let selectors: AdapterSelectors | undefined              // 이 port의 활성 selector(STOP 핸들러에서도 사용) [scope 버그 수정]
  port.onMessage.addListener(async (msg) => {
    if (msg.action === 'SEND_PROMPT') {
      selectors = (await loadConfig())[msg.llm]
      if (!selectors?.promptArea || !selectors?.responseArea)
        return port.postMessage({ action: 'ERROR', code: 'ERR_004' })   // selector 부재/사이트 구조 변경
      if (isLoginPage(selectors)) return port.postMessage({ action: 'ERROR', code: 'ERR_002' })  // 로그인 폼 감지 → "로그인 필요"
      fillPrompt(selectors, msg.prompt)     // textarea.value 또는 contenteditable(+ execCommand/paste 폴백, 221 §5)
      pressEnterOrClick(selectors)          // sendButton.click() 또는 Enter keydown(빈 sendButton)
      observeResponse(selectors, (text, done) =>
        port.postMessage(done ? { action: 'DONE' } : { action: 'RESPONSE_CHUNK', content: text }))
      // observeResponse: MutationObserver(responseArea 마지막 요소.innerText) 누적.
      // ── done 판정 (우선순위, [G14]) ── (실측 확정값은 221 §7)
      //   1) generatingFlag 정의됨: 존재→생성중, 사라지면 done. (※ 221 §7 기준 6종 모두 미사용 — 향후 대비용 분기)
      //   2) stopButton 정의됨(chatgpt/claude/gemini/grok/perplexity): 표시→생성중, 사라지면 done
      //   3) stopButton 없음(deepseek): responseArea 마지막 요소 innerText가
      //      (selectors.doneDebounceMs||1500)ms 동안 변화 없으면 done (텍스트 안정화)
      //   + 사용자 STOP(abort) 수신 시 즉시 done. 자동 타임아웃 없음(FR-11 — 장시간은 수동 STOP)
    } else if (msg.action === 'STOP') {
      findStop(selectors)?.click()   // [222 §8 CRITICAL-1] perplexity ':has(use[*|href])' 는 querySelector에서 throw → findStop() 필수(naive document.querySelector(stopButton) 금지). findStop 정의=222 §8
    }
  })
})
```

## 6. 비밀채팅 봇 호출 (FR-09)

비밀채팅은 동일 봇 인스턴스를 쓰되 **별도 대화**가 필요 → 비밀채팅 전용 봇 인스턴스(`createBotInstance(targetLlm)`)를 메모리에 만들고, 1000토큰 지시문 부가하여 `sendMessage`. 메시지/인스턴스는 `secretChatAtom`(휘발) 외부 저장 금지.

> **⚠️ [N-2] all-DOM 한계 (사용자 확정: 한계 수용 2026-06-01)**: 6종 모두 LLM당 **단일 탭을 재사용**하므로(§3.3 `ensureTab`), 비밀채팅은 메인 세션과 **같은 사이트 탭·같은 웹 대화 스레드를 공유**한다. 즉 `secretChatAtom`(메모리)은 **앱 저장만 격리**할 뿐, LLM의 웹 대화 맥락은 메인과 분리되지 않는다 — FR-09 "전체 맥락과 분리"는 **앱 레벨에 한함**. 비밀 발화가 메인 라운드의 사이트 컨텍스트에 영향을 줄 수 있음에 유의(같은 LLM이 메인 세션에 참여 중일 때 특히). 진짜 웹 대화 격리(별도 탭/사이트 "새 대화" 스레드)는 현 범위 밖.
