## 3. LLM 간 응답 전달 포맷 (YAML) [v2] · [v3 환원]

> **LLM 전달용 context는 YAML로 직렬화.** 선정 사유 = **토큰 최소화 + 시각적 시인성** (minified JSON은 시인성↓, 평문 라벨은 구조 모호 → YAML이 균형 최적). 전달 내용도 최소화(타 LLM 응답만).
> [v3 정정] 설계세션의 "JSON 전환"은 오독 — 원 결정은 "**LLM 전달 시 YAML 사용**"(transcript 20260528 msg262). 내부 저장/메시징은 JSON, **LLM에 주입하는 context만 YAML**.
> Session 체크 제거: prompt 실패 시 "로그인 필요" 처리.

### Self-Exclusion 규칙 [v2]

라운드 종료 후 **각 LLM마다 context를 개별 구성**하여 전달. 자기 자신의 응답은 제외.
아래는 **ChatGPT에 전달**되는 예시 (ChatGPT 응답 제외, 최소 context):

```yaml
roundNo: 3
userPrompt: "시장 조사 아이디어를 내줘"
others:
  claude: "경쟁사 분석부터..."
  gemini: "소비자 인터뷰를..."
```

**의도**: 자기 응답을 다시 보면 self-reinforcement / echo chamber 현상 발생. 타 AI의 의견만 참고하여 독립적 사고 유도.

> `maxRounds`, `tokenLimit` 등은 context에서 제외 (라운드 시작 시 이미 전달됨) [v2]

### Context 구성 로직 [v2]
```
function buildContextForLlm(targetLlm, roundData):
    others = {}
    for each (llm, response) in roundData.responses:
        if llm != targetLlm and response.status == "completed":
            others[llm] = response.content  // 최소화: LLM명 + 내용만
    return {
        roundNo: roundData.currentRound + 1,
        userPrompt: roundData.userPrompt,
        others: others
    }
```

## 5. 세션 유지 (Service Worker 복구 대비) [v2]

MV3 Service Worker는 30초 비활성 시 종료될 수 있음. **매 Round 완료 시점**에 `chrome.storage.session`에 저장:

```js
chrome.storage.session.set({
  "lastRound_20260528-1430-abc123": {
    "sessionId": "20260528-1430-abc123",
    "completedRound": 3,
    "gridSlots": [{ "slot": 0, "llm": "chatgpt" }, ...],
    "roundHistory": [
      { "round": 1, "userPrompt": "...", "responses": { "chatgpt": "...", ... } },
      { "round": 2, ... }
    ]
  }
})
```

SW 재시작 시 `chrome.storage.session`에서 마지막 저장 시점을 읽어 세션 재개.
