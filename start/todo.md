# 사용자 요구사항 v2.0 — 확정 (2026-05-27)

## 1. LLM 접속 및 탐색

| 항목 | 내용 |
|------|------|
| 접속 방식 | 크롬 브라우저 로그인 세션 기반 (API 키 불필요) |
| LLM 탐색 | 지원 LLM 3종(Claude, ChatGPT, Gemini) 로그인 상태 자동 탐색 |
| 탐색 실패 시 | ①크롬 로그인 유도 → ②유효 LLM 목록 부재 시 default LLM 계정 생성 요청 안내 |
| UI 참고 | ChatHub (크롬 확장 플러그인) |
| 구현 베이스 | E:\workspace\chathub 개선 |

## 2. LLM 선택

| 항목 | 내용 |
|------|------|
| 선택 방식 | 목록 표시 → 사용자가 일부 또는 전체 선택 |
| 최소 선택 | 2개 이상 (단독 실행 불가) |
| 선택 상태 | 세션 간 유지 |
| 채팅 중 변경 | LLM 추가/제외 가능 (라운드 사이에만) |

## 3. 라운드 규칙

| 항목 | 내용 |
|------|------|
| 기본 회차 | 3회 |
| 최대 회차 | 9회 (채팅 중 1회씩 증감 가능, 마지막 회차에서도 증설 가능) |
| 동기화 | 모든 선택 LLM 응답 완료 후 다음 라운드 진행 |
| R1 | Blind 독립 응답 (타 LLM 응답 미공유) |
| R2 | 타 LLM의 R1 응답 전체 텍스트 검토 후 의견 제시 |
| R3 | 최종 답변 |
| 조기 종료 | 사용자 판단에 의해 가능 |
| 연장 | 지정 회차 완료 시 자동 종료, 연장 가능 |

## 4. 사용자 개입

| 항목 | 내용 |
|------|------|
| 개입 시점 | Request 후 **전체 Response 완료 전까지 개입 불가** |
| 재질의 | 특정 LLM 1:1, **라운드 도중 불가** (라운드 사이에만) |
| 재질의 결과 | 다음 라운드 컨텍스트에 포함 |
| 최종 판단 | 사용자 |
| 사전 입력 | 다음 회차 request 미리 입력 가능 (send 불가) |
| 몰래 채팅 | 선응답 LLM과 개별 채팅 *(MVP 이후)* |

## 5. 타임아웃

| 항목 | 내용 |
|------|------|
| 기본 | 5분 |
| 조정 | 분 단위 사용자 설정 가능 |
| 재시도 | 3~9회 사용자 조정 가능 |
| LLM 전달 | 제한시간 + 현재차수/제한차수를 request 시 LLM에 전달 |

## 6. 저장

| 항목 | 내용 |
|------|------|
| 자동 저장 | IndexedDB, 라운드 완료 시마다 자동 저장 |
| MD 저장 | 전체 내용 + LLM별 개별 MD 파일 저장 |
| 이력 조회 | 년월일시 / LLM / 라운드 검색 가능 |

## 7. 응답

| 항목 | 내용 |
|------|------|
| 출력 형식 | 강제 (구조화) |
| 컨텍스트 제한 | Response context 1500 token 제한 *(설계 검토 필요)* |
| 응답 간소화 | 장황한 출력 회피 |
