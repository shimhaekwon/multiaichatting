# 멀티 LLM 협업 시스템 요구사항정의서 v1.0

## 1. 목적
브라우저 로그인 세션 기반으로 여러 LLM에게 동일 요청을 보내고, 결과를 한 화면에서 비교·토론시켜 사용자가 최종 판단하는 시스템.

## 2. 기능 요구사항

**FR-1. LLM 탐지 및 선택**
- FR-1.1 브라우저에 로그인된 LLM 자동 탐지 후 목록 표시
- FR-1.2 사용자는 목록에서 1개 이상 선택 또는 전체 선택 가능

**FR-2. 요청 전달**
- FR-2.1 사용자는 하나의 요청을 입력
- FR-2.2 선택된 모든 LLM에 동시 전달

**FR-3. 라운드 진행**
- FR-3.1 최대 3 라운드
- FR-3.2 R1: 각 LLM 독립 응답 (타 LLM 답변 비공개)
- FR-3.3 R2: R1 결과 전체를 각 LLM에 전달, 검토 응답
- FR-3.4 R3: R2 결과 전체를 각 LLM에 전달, 최종 답변
- FR-3.5 모든 LLM 응답 완료 후에만 다음 라운드 진행 (동기화)

**FR-4. 사용자 개입**
- FR-4.1 각 라운드 종료 시점에 사용자 개입 가능
- FR-4.2 특정 LLM에 1:1 단독 재질의 가능 (타 LLM 컨텍스트 미포함)
- FR-4.3 조기 종료 가능

**FR-5. 결과 표시**
- FR-5.1 라운드별·LLM별 응답을 한 화면에 비교 표시
- FR-5.2 재질의 로그 별도 표시

**FR-6. 최종 판단**
- FR-6.1 시스템은 자동 합의하지 않음
- FR-6.2 최종 선택은 사용자가 수행

## 3. 비기능 요구사항
- NFR-1 브라우저 자동화 기반 (API 키 불필요)
- NFR-2 LLM 응답 타임아웃 처리 필요 (시간값 TBD)
- NFR-3 세션 영속화 (저장 방식 TBD)

## 4. 제약사항
- C-1 각 LLM 서비스 약관 준수, 봇 탐지 우회 금지
- C-2 브라우저 로그아웃 시 동작 불가
- C-3 무료 한도/레이트 리밋은 각 서비스 정책 따름
