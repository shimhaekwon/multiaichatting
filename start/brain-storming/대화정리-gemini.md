# Multi-LLM Ensemble & Debate Platform — 대화 정리

## 시스템 정의

> 크롬 브라우저 로그인 세션을 활용해 여러 LLM을 선택하고, 동기식 3라운드 토론을 통해 사용자가 최종 판단을 내리는 **Human-in-the-Loop AI 협업 플랫폼**

---

## 핵심 동작 규칙

| 항목 | 내용 |
|---|---|
| LLM 접속 | 브라우저 로그인 세션 활용 (API Key 불필요) |
| LLM 선택 | 탐지된 목록에서 일부 또는 전체 선택 |
| Round 1 | 독립(Blind) 응답 — 서로 미공유 |
| Round 2 | 상호 검토 — 타 LLM 결과 기반 비판/보완 |
| Round 3 | 최종 제안 |
| 동기화 | 모든 LLM 응답 완료 후 다음 라운드 진행 |
| 최종 판단 | 사용자 직접 판단 (AI 자동 합의 없음) |

---

## 사용자 권한

- 각 라운드마다 개입 가능
- 특정 LLM에게 1:1 근거 재질의 (Deep Dive)
- 특정 LLM 제외 또는 결과 선택 채택
- 최종 Commit은 사용자가 수행

---

## 핵심 설계 원칙

- **Context 폭발 방지** — 압축된 핵심만 다음 라운드에 전달
- **Timeout/Drop** — 무응답 LLM은 15초 후 DROP, 나머지로 진행
- **응답 간소화** — 기본 응답은 짧게, 상세는 Deep Dive 옵션으로
- **상태 머신** — IDLE → ROUND_RUNNING → BARRIER_WAIT → HITL_WAIT → FINAL

---

## 각 AI 성향 (사용자 체감)

| AI | 특성 |
|---|---|
| ChatGPT | 확장적, 맥락 연결, 중심에서 약간 벗어남 |
| Gemini | 대화 주도, 문서 구조화, 제품 감각 |
| Perplexity | 중립, 사실 정리, 객관 비교 |
| Claude | 분석적, 누락 지적, 구조 검증 |
